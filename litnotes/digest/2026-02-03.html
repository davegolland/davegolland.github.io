1. VisionTrim: Unified Vision Token Compression for Training-Free MLLM Acceleration

Source: HuggingFace | Tags: Embeddings, LLM Code: https://github.com/hanxunyu/VisionTrim Deep dive: Full analysis

Summary: Preserve essential visual tokens by combining global and local importance scores, eliminating redundant or non-salient tokens without training. Vision transformers in MLLMs generate excessive tokens that increase computational cost, especially for high-resolution images and videos. Useful for: High-resolution image understanding tasks (4K/8K images)

2. ORCH: many analyses, one merge-a deterministic multi-agent orchestrator for discrete-choice reasoning with EMA-guided routing

Source: arXiv:cs.AI | Tags: LLM Deep dive: Full analysis

Summary: Coordinate multiple heterogeneous LLMs through deterministic, rule-based task decomposition and answer aggregation rather than stochastic routing, ensuring reproducibility and interpretability for discrete-choice reasoning. Existing multi-agent systems rely on stochastic routing or ad-hoc heuristics that make behavior difficult to reproduce and decision processes hard to interpret. Useful for: Multiple-choice exams and standardized reasoning benchmarks (MMLU, MMLU-Pro, GSM8K)

Vault connections: Agentic Reasoning Pipeline, Synergistic Tool Integration in Agentic Reasoning, Agentic Reasoning Design Patterns

4. PretrainRL: Alleviating Factuality Hallucination of Large Language Models at the Beginning

Source: arXiv:cs.CL | Tags: RL, LLM, RAG Deep dive: Full analysis

Summary: Integrate reinforcement learning during pretraining to actively reshape the model’s probability distribution by down-weighting high-probability falsehoods, creating capacity for low-probability truths to be learned effectively. Large language models suffer from factual hallucinations due to imbalanced data distribution in pretraining corpora, where low-probability truths compete against high-probability falsehoods. Useful for: Pretraining large language models to reduce factual hallucinations

Vault connections: Early-Exit Large Language Models, Language Generation vs. Identification in the Limit Outline, Lengthy Context Retrieval via LLM Pretraining

5. Geometric Analysis of Token Selection in Multi-Head Attention

Source: arXiv:cs.AI | Tags: LLM, RAG Deep dive: Full analysis

Summary: Quantify multi-head attention’s token selection behavior using geometric metrics (Precision, Recall, F-score) that measure separability between selected and non-selected tokens directly in value-state space. Existing attention analysis treats the mechanism as a black box. Useful for: Interpreting which attention heads perform retrieval vs. mixing vs. resetting

Vault connections: Dilated Attention, Attention Head Specialization for Matching Detection, Infinite Retrieval: Attention Enhanced LLMs in Long-Context Processing

6. On the Power of (Approximate) Reward Models for Inference-Time Scaling

Source: arXiv:cs.CL | Tags: RL, LLM Deep dive: Full analysis

Summary: Identify Bellman error as the fundamental quantity that determines whether an approximate reward model can effectively guide Sequential Monte Carlo inference-time scaling, with exponential improvements in computational complexity achievable when Bellman error is bounded by O(1/T). In practice, deployed inference-time scaling systems cannot access true reward models and must use approximations. Useful for: Language model reasoning tasks requiring multi-step inference (mathematics, code generation, complex planning)

Vault connections: Scaling LLM Test-Time Compute Optimally: Summary and Implications, Reasoning Beyond Limits: Advances and Open Problems for LLMs, Monte Carlo Tree Search

7. Constrained Process Maps for Multi-Agent Generative AI Workflows

Source: arXiv:cs.AI | Tags: LLM, RAG Deep dive: Full analysis

Summary: Formalize multi-agent LLM workflows as finite-horizon Markov Decision Processes with a directed acyclic graph (DAG) structure, where each agent represents a distinct decision stage with predefined state transitions. Existing agentic architectures rely heavily on single-agent prompt engineering, making it difficult to observe, compare, and control how agents handle uncertainty across decision stages or coordinate with human oversight in regulated workflows. Useful for: Compliance and regulatory workflows (AML, KYC, content moderation)

Vault connections: Reasoning Beyond Limits: Advances and Open Problems for LLMs, Language Agents by Yu Su, Agentic RAG: Taxonomy of Architectures

8. Fat-Cat: Document-Driven Metacognitive Multi-Agent System for Complex Reasoning

Source: arXiv:cs.LG | Tags: LLM, RAG Code: https://github.com/answeryt/Fat-Cat Deep dive: Full analysis

Summary: Represent agent state as Markdown documents rather than nested JSON to reduce syntactic overhead and align with pre-training corpora, improving the model’s ability to focus on semantic reasoning. Existing agent frameworks force LLMs to devote attention to parsing rigid JSON syntax, creating a signal-to-noise ratio problem where syntactic processing competes with semantic reasoning for limited context. Useful for: Multi-step reasoning tasks requiring frequent state updates

Vault connections: Mind-Map Agent, Agentic Reasoning Pipeline, Multi-Agent Agentic RAG

9. SLIME: Stabilized Likelihood Implicit Margin Enforcement for Preference Optimization

Source: arXiv:cs.LG | Tags: RL, LLM, RAG Deep dive: Full analysis

Summary: A reference-free preference alignment objective that decouples preference learning from generation quality by combining likelihood anchoring, probability stabilization, and dual-margin constraints. Direct preference optimization (DPO) methods suffer from objective mismatch where optimizing relative margins between chosen and rejected responses doesn’t preserve absolute likelihood of preferred outputs, leading to “unlearning” (degradation of high-quality outputs) and “formatting collapse” (over-penalization of rejected sequences). Useful for: LLM alignment and fine-tuning for preference-based learning

Vault connections: Obsidian Literature Review: Direct Preference Optimization (DPO) in Finetuning Language Models, Test-Time Preference Optimization (TPO): A Breakthrough in LLM Alignment, LLM Alignment as Retriever Optimization: An Information Retrieval Perspective

Summary: Train separate encoder towers for queries (reviews) and documents (item metadata) to map both into a shared semantic space, optimized via Multiple Negatives Ranking Loss to maximize similarity between matched pairs while minimizing similarity to other items in the batch. - Sparse keyword matching (BM25) fails under vocabulary mismatch between user intent and product metadata

  • Single-tower models are computationally expensive for retrieval at catalog scale
  • Standard contrastive learning doesn’t leverage the full negative information available in training batches Useful for: E-commerce recommendation and search under vocabulary mismatch

3. RedVisor: Reasoning-Aware Prompt Injection Defense via Zero-Copy KV Cache Reuse

Summary: A lightweight, removable adapter positioned above a frozen LLM backbone that generates explainable reasoning paths during a gating phase, then remains “muted” during response generation, enabling both attack detection and guided safe response synthesis. Existing defenses face a critical trade-off—prevention-based fine-tuning degrades general utility (alignment tax), while detection-based filtering incurs prohibitive latency and memory costs. Useful for: Defending RAG (Retrieval-Augmented Generation) systems against adversarial context injection

Deep dive: Full analysis

7. Hierarchical Adaptive Eviction for KV Cache Management in Multimodal Language Models

Summary: Selectively prune visual tokens during KV cache pre-filling by identifying and retaining only tokens that receive high attention weights across both visual and textual attention patterns. Multimodal LLMs allocate equal KV cache space to all visual tokens regardless of their importance to the task. Useful for: Vision-language understanding tasks (VQA, image captioning) where image regions outside the region-of-interest are redundant

8. Data Distribution Matters: A Data-Centric Perspective on Context Compression for Large Language Model

Summary: Systematically evaluate context compression quality through an autoencoder-based framework that measures semantic integrity by analyzing how data distribution dimensions (input data and intrinsic model knowledge) affect compression. Existing context compression research focuses exclusively on model-side improvements while ignoring how the data distribution itself impacts compression effectiveness. Useful for: Diagnosing why context compression fails on specific domains or datasets

9. OneMall: One Architecture, More Scenarios — End-to-End Generative Recommender Family at Kuaishou E-Commerce

E-commerce recommendation systems struggle with unified representation across heterogeneous item distribution channels (product cards, short-video, live-streaming). Useful for: Multi-scenario e-commerce recommendation (product discovery, short-video feeds, live-shopping)

10. Read As Human: Compressing Context via Parallelizable Close Reading and Skimming

Summary: Partition context into segments and adaptively apply close reading (full retention) to high-relevance segments while compressing low-relevance segments into compact summary vectors based on query relevance. Large Language Models suffer from computational inefficiency and information redundancy when processing long contexts. Useful for: Long-context question answering on documents (16K-32K tokens)

11. PRISM: Parametrically Refactoring Inference for Speculative Sampling Draft Models

Summary: Disaggregate the computation of each predictive step across different parameter sets to decouple model capacity from inference cost in draft models. Existing speculative decoding methods require parametrically larger draft models to achieve better prediction accuracy, but larger models incur substantial computational overhead during inference. Useful for: Speculative decoding for LLM acceleration

Deep dive: Full analysis

12. ClueTracer: Question-to-Vision Clue Tracing for Training-Free Hallucination Suppression in Multimodal Reasoning

Summary: A metric for quantitatively assessing whether a multimodal reasoning model correctly retrieves task-relevant visual clues from images during inference. Existing metrics for evaluating hallucination focus on output correctness but don’t diagnose why hallucinations occur—specifically, they fail to measure whether the model gathered the right visual evidence before generating text.
