1. TSR: Trajectory-Search Rollouts for Multi-Turn RL of LLM Agents

Source: arXiv:cs.CL | Tags: Search, RL, LLM Deep dive: Full analysis

Summary: Move test-time search techniques (beam search, lookahead) into the training-time rollout generation phase to construct higher-quality trajectories for multi-turn RL of LLM agents. Multi-turn RL suffers from sparse/delayed rewards and stochastic environments, where naive trajectory sampling leads to poor exploitation, mode collapse, and unstable learning. Useful for: Multi-turn agent tasks with sparse rewards (planning, navigation, puzzle-solving)

Vault connections: Reinforcement learning in LLM – recent turbulence, Sergey Levine — Multi-Turn Reinforcement Learning for LLM Agents, Proximal policy optimization (PPO)

2. DMAP: A Distribution Map for Text

Source: arXiv:cs.CL | Tags: RAG, Search, LLM, RecSys Deep dive: Full analysis

Summary: Maps text through a language model into a set of samples in the unit interval [0,1] that jointly encode both rank and probability information from next-token distributions, enabling model-agnostic statistical analysis. Traditional metrics like perplexity fail to account for context—interpreting a next-token probability requires understanding the shape of the conditional distribution (i.e., how many reasonable choices exist). Useful for: Generation parameter validation: Detecting contaminated or improperly generated training data

Vault connections: LLM Query → BM25 Hard Negative Pipeline, Consolidating Ranking and Relevance Predictions of Large Language Models, LLM in dense retrieval with query likelihood Modeling

3. Learning to Configure Agentic AI Systems

Source: arXiv:cs.AI | Tags: Search, RL, LLM, RAG Deep dive: Full analysis

Summary: Learn a hierarchical reinforcement learning policy that dynamically configures LLM agent systems on a per-query basis—selecting workflows, tools, token budgets, and prompts—rather than applying fixed “one-size-fits-all” configurations. Static agent configurations waste compute on simple queries and underperform on complex ones; hand-tuned heuristics are brittle across diverse inputs. Useful for: Multi-turn reasoning systems with variable query complexity (e.g., SWE-bench, HotpotQA, tool-augmented QA)

Vault connections: Synergistic Tool Integration in Agentic Reasoning, Automated Multi-Agent Architecture Search (ADAS, AFlow, EvoFlow), Agentic Reasoning Pipeline

4. Think like a Scientist: Physics-guided LLM Agent for Equation Discovery

Source: arXiv:cs.AI | Tags: LLM, RAG Deep dive: Full analysis

Summary: An agentic framework that mimics scientific reasoning by first extracting physical properties (symmetries, units, conservation laws) from data using physics-based tools, then using these properties as structured priors to guide symbolic regression engines toward interpretable equations. Existing LLM-based equation discovery systems try to guess formulas directly from data without modeling the multi-step reasoning process scientists use, leading to poor symbolic accuracy and brittleness to noise. Useful for: Scientific discovery from experimental physics data (mechanics, electromagnetism, fluid dynamics)

Vault connections: Language Agents by Yu Su, Coding Agent for Tool-Augmented Reasoning, Pocket Flow

5. Scene-Aware Memory Discrimination: Deciding Which Personal Knowledge Stays

Source: arXiv:cs.CL | Tags: RAG, Search, LLM, RecSys Deep dive: Full analysis

Summary: A two-stage method that filters non-memorable user interactions via a lightweight gating unit and establishes adaptive memory standards through clustered prompting, enabling efficient personal knowledge organization at scale. Addresses the dual challenge of filtering irrelevant information from large-scale user interactions and managing rising computational costs in LLM-based memory systems, while adapting to diverse memory standards across different application contexts. Useful for: Personal assistant systems that accumulate user interaction history over time

Vault connections: Hierarchical Agent Memory for Contextual Recommendation, MemoryBank, Augmented Multimodal Memory

6. Extending Puzzle for Mixture-of-Experts Reasoning Models with Application to GPT-OSS Acceleration

Source: arXiv:cs.LG | Tags: Search, RL, LLM, RAG Deep dive: Full analysis

Summary: Apply post-training neural architecture search to reasoning-focused LLMs by combining heterogeneous MoE expert pruning, selective window attention replacement, FP8 KV-cache quantization, and reinforcement learning recovery to reduce inference cost while maintaining accuracy across varying reasoning efforts. Reasoning models generate long traces that increase serving cost dramatically; traditional per-token throughput metrics are misleading because reasoning token counts vary with model architecture changes, making it unclear whether optimizations actually improve end-to-end efficiency. Useful for: Deploying reasoning-focused LLMs (o1-style models) in production where long traces dominate cost

Vault connections: Reasoning Beyond Limits: Advances and Open Problems for LLMs, Hierarchical Reasoning Model, RaDeR: Reasoning-aware Dense Retrieval Model

7. Temperature as a Meta-Policy: Adaptive Temperature in LLM Reinforcement Learning

Source: arXiv:cs.LG | Tags: RL, LLM, RAG Deep dive: Full analysis

Summary: Treat temperature selection as a learnable meta-policy that adaptively chooses exploration/exploitation trade-offs during LLM reinforcement learning by rewarding temperatures that maximize the likelihood of high-advantage trajectories. Static or heuristic temperature schedules fail to adapt to the dynamic exploration needs throughout RL training—high temperatures generate noisy exploration when exploitation is needed, while low temperatures cause premature convergence when exploration is required, limiting policy improvement. Useful for: Mathematical reasoning tasks requiring adaptive exploration (GSM8K, MATH, ARC)

Vault connections: Learnable Temperature Parameter in InfoNCE Contrastive Learning, Pignal, Population-Based Evolution of LLMs (GENOME)

8. PrefillShare: A Shared Prefill Module for KV Reuse in Multi-LLM Disaggregated Serving

Source: arXiv:cs.LG | Tags: LLM, RAG Deep dive: Full analysis

Summary: Share a frozen prefill module and its generated KV cache across multiple specialized models in disaggregated serving, eliminating redundant computation when multiple LLMs process the same prompt prefix. Multi-agent systems that invoke multiple specialized models over a shared context suffer from redundant prefill computation and KV cache storage for identical prompts, leading to high aggregate prefill load, worse tail latency due to prefill-decode interference, and wasted GPU memory—even in disaggregated architectures. Useful for: Multi-agent orchestration systems (e.g., ReAct, AutoGPT) where specialized models collaborate on shared context

Vault connections: LLM Inference Performance: The Team Mental Model + Experiment Playbook, Chunked prefill in LLMs, SGLang Architecture - PM Perspective

9. Towards On-Policy SFT: Distribution Discriminant Theory and its Applications in LLM Training

Source: arXiv:cs.AI | Tags: RL, LLM, RAG Code: https://github.com/zhangmiaosen2000/Towards-On-Policy-SFT Deep dive: Full analysis

Summary: Enable supervised fine-tuning to achieve RL-level generalization by quantifying and correcting the mismatch between training data distribution and the model’s own output distribution, without requiring expensive online sampling. SFT is efficient but generalizes poorly because it trains on off-policy data (static datasets that don’t reflect what the model actually generates). Useful for: Domains where RL is infeasible (limited compute, reward modeling is expensive, unstable training)

Vault connections: GRPO outline, Policy Gradients, Reinforcement learning in LLM – recent turbulence

10. Improving Neural Retrieval with Attribution-Guided Query Rewriting

Source: arXiv:cs.IR | Tags: RAG, Search, LLM, RecSys Deep dive: Full analysis

Summary: Use gradient-based token attributions from a neural retriever to identify weak or misleading query terms, then feed those attribution scores as soft guidance into an LLM prompt that rewrites the query to improve retrieval effectiveness while preserving user intent. Neural retrievers are brittle to underspecified or ambiguous queries, and existing solutions operate in isolation—LLMs rewrite queries without retriever feedback, while explainability methods (gradient attributions, attention) are used only for post-hoc analysis rather than actively improving retrieval. Useful for: Open-domain search with implicit or ambiguous information needs (BEIR benchmarks)

Vault connections: Leveraging No-Retrieval as a Routing Option, Multi-step Agentic Retrieval and Rewriting for Massive GraphRAG, RANK1: Test-Time Compute for Reranking in Information Retrieval

Multi-modal LLMs for autonomous driving require processing thousands of visual tokens from camera inputs, creating computational bottlenecks unsuitable for resource-constrained vehicle platforms. Useful for: End-to-end autonomous driving systems requiring multi-modal understanding (vision + language instructions)

4. End-to-End Semantic ID Generation for Generative Advertisement Recommendation

Summary: End-to-end learnable Semantic ID generation that jointly optimizes item embeddings and discrete SIDs directly from raw advertising data, eliminating the two-stage compression bottleneck of traditional Residual Quantization-based approaches. Addresses objective misalignment, semantic degradation, and error accumulation inherent in cascading two-stage SID generation (embedding learning → quantization) used by existing Generative Recommendation systems. Useful for: Large-scale advertisement recommendation systems requiring semantic item tokenization

5. Hi-SAM: A Hierarchical Structure-Aware Multi-modal Framework for Large-Scale Recommendation

Summary: A multi-modal recommendation framework that disentangles cross-modal semantics from modality-specific details during tokenization, then preserves user interaction hierarchy through memory-compressed attention over semantic ID sequences. Existing semantic ID approaches suffer from (1) tokenization redundancy/collapse due to lack of disentanglement between shared and modality-specific information, and (2) flattened attention that ignores the hierarchical structure of user sequences (user → items → tokens), causing bias toward local token details over holistic item semantics. Useful for: Large-scale multi-modal recommendation systems with text, image, and video content

Deep dive: Full analysis

6. LASER: An Efficient Target-Aware Segmented Attention Framework for End-to-End Long Sequence Modeling

Summary: A full-stack framework combining hybrid DRAM-SSD storage infrastructure (SeqVault) with a sparse attention mechanism (Segmented Target Attention) to enable efficient modeling of ultra-long user behavior sequences in real-time recommendation systems. Deploying long-sequence models in production faces a dual “Latency Wall”: (1) high I/O latency retrieving massive user histories from storage, and (2) quadratic computational complexity of attention mechanisms over sequences with millions of interactions. Useful for: Large-scale recommendation systems requiring lifelong user modeling (e-commerce, social media, streaming)

Deep dive: Full analysis

7. Recurrent Preference Memory for Efficient Long-Sequence Generative Recommendation

Summary: Compresses long user interaction sequences into compact learnable memory tokens via self-referential teacher-forcing, enabling parallel training of recurrent models while maintaining O(1) inference memory. Full-attention generative recommendation models face quadratic computational cost and cannot scale to lifelong user histories; traditional recurrent models require serial training and accumulate noise from stochastic interactions. Useful for: Sequential recommendation with extremely long user histories (thousands+ interactions)

Deep dive: Full analysis

8. Thinking with Drafting: Optical Decompression via Logical Reconstruction

Summary: Reconceptualizes visual reasoning as optical decompression by forcing models to draft their mental model into executable Domain-Specific Language (DSL) code, which then generates deterministic visual proofs for self-verification rather than directly hallucinating answers. Addresses the precision paradox in multimodal LLMs where optical perception transcribes symbols without capturing logical topology, and pixel-based generation produces visual artifacts lacking mathematical exactness—resulting in reasoning failures on complex visual tasks requiring precise logical structure. Useful for: Visual algebra and symbolic mathematics problems requiring exact symbolic manipulation

Deep dive: Full analysis

9. Deep Kernel Fusion for Transformers

Summary: A deeply fused CUDA kernel that co-optimizes the entire SwiGLU MLP forward pass (gate projection → SiLU activation → elementwise multiply → down projection) to minimize HBM traffic and maximize L1/L2 cache reuse during memory-bound LLM inference. In long-context agentic LLM inference, token generation is memory-bandwidth bottleneck rather than compute-bound. Useful for: Long-context agentic inference (RAG, code generation, multi-turn dialogue) where batch size is small (1-8) and generation length is high

Deep dive: Full analysis

10. Differentially Private and Communication Efficient Large Language Model Split Inference via Stochastic Quantization and Soft Prompt

Summary: Reduces communication overhead and preserves privacy in LLM split inference by projecting and quantizing token embeddings with differential privacy guarantees, while using server-side soft prompts to recover utility without requiring local denoising models. Existing privacy-preserving LLM split inference methods require transmitting high-dimensional token embeddings and deploying resource-intensive local denoising models, creating excessive communication and computation overhead that prevents deployment on resource-constrained devices. Useful for: Resource-constrained edge devices requiring private LLM inference (mobile, IoT)

Deep dive: Full analysis

11. Towards Personalized Bangla Book Recommendation: A Large-Scale Multi-Entity Book Graph Dataset

Summary: A large-scale heterogeneous knowledge graph dataset for Bangla book recommendations that connects books, users, authors, categories, publishers, and reviews through eight relation types to enable graph-based and hybrid recommendation models in a low-resource language setting. Personalized book recommendation in Bangla literature has been severely limited by the lack of structured, large-scale, publicly available datasets. Useful for: Low-resource language recommendation systems (cultural/regional content platforms)

Deep dive: Full analysis

12. Where Bits Matter in World Model Planning: A Paired Mixed-Bit Study for Efficient Spatial Reasoning

Summary: Allocate quantization bits strategically across world model modules (encoder, dynamics, decoder) rather than uniformly, preserving encoder precision to maintain spatial reasoning quality in the 4-bit transition regime. Uniform low-bit quantization (e.g., INT4) causes catastrophic collapse in world model planning performance, but blindly increasing bits everywhere is computationally wasteful. Useful for: Real-time robotics planning under memory/compute constraints (embedded vision-based navigation)
