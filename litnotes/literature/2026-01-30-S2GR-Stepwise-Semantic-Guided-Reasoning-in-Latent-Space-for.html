SGR: Stepwise Semantic-Guided Reasoning in Latent Space for Generative Recommendation

Source: arXiv:cs.IR Published: 2026-01-26 Rank: #11 (score: 0.85, max_sim: 0.88) Reading time: ~0 min Tags: Uncategorized

Abstract

Generative Recommendation (GR) has emerged as a transformative paradigm with its end-to-end generation advantages. However, existing GR methods primarily focus on direct Semantic ID (SID) generation from interaction sequences, failing to activate deeper reasoning capabilities analogous to those in large language models and thus limiting performance potential. We identify two critical limitations in current reasoning-enhanced GR approaches: (1) Strict sequential separation between reasoning and generation steps creates imbalanced computational focus across hierarchical SID codes, degrading quality for SID codes; (2) Generated reasoning vectors lack interpretable semantics, while reasoning paths suffer from unverifiable supervision. In this paper, we propose stepwise semantic-guided reasoning in latent space (SGR), a novel reasoning enhanced GR framework. First, we establish a robust semantic foundation via codebook optimization, integrating item co-occurrence relationship to capture behavioral patterns, and load balancing and uniformity objectives that maximize codebook utilization while reinforcing coarse-to-fine semantic hierarchies. Our core innovation introduces the stepwise reasoning mechanism inserting thinking tokens before each SID generation step, where each token explicitly represents coarse-grained semantics supervised via contrastive learning against ground-truth codebook cluster distributions ensuring physically grounded reasoning paths and balanced computational focus across all SID codes. Extensive experiments demonstrate the superiority of SGR, and online A/B test confirms efficacy on large-scale industrial short video platform.

Key Techniques

  • Codebook optimization with co-occurrence relationships
  • Stepwise reasoning mechanism with thinking tokens
  • Contrastive learning for semantic supervision
  • Load balancing and uniformity objectives

Stepwise Semantic-Guided Reasoning with Thinking Tokens (S²GR)

Core Idea: Insert interpretable “thinking tokens” before each Semantic ID generation step to perform explicit coarse-grained semantic reasoning in latent space, with each token supervised via contrastive learning against ground-truth codebook cluster distributions.

Problem It Solves: Previous reasoning-enhanced GR methods suffer from (1) strict sequential separation between reasoning and generation creating imbalanced computational focus (early SID codes get more reasoning, later ones less), and (2) generated reasoning vectors lack interpretability and have no verifiable supervision signal, leading to ungrounded reasoning paths.

How It Works:

  1. For each SID generation step (coarse-to-fine hierarchy), insert a learnable thinking token into the sequence before generating SID code
  2. Each thinking token is trained via contrastive learning: push closer to the ground-truth codebook cluster distribution (from step labels), and push away from other cluster distributions
  3. Explicitly represent coarse-grained semantics through : token embeddings directly correspond to expected semantic clusters at hierarchy level
  4. Generate SID code using both interaction context and the semantically-grounded thinking token:
  5. Distribute computational focus evenly across all hierarchy levels by ensuring every step has equally-weighted reasoning supervision

Where It’s Useful:

  • Generative recommendation with hierarchical SID codes (coarse-to-fine item categories)
  • Any sequential generation task requiring interpretable intermediate reasoning
  • Recommendation systems needing balanced computational allocation across decoding steps
  • Short-form video platforms with multi-level item hierarchies (as validated in paper’s A/B test)

Limitations / Caveats:

  • Requires ground-truth codebook cluster labels during training (supervision overhead)
  • Thinking token insertion adds inference latency (one additional forward pass per SID generation step)
  • Contrastive learning hyperparameters (temperature, margin) require careful tuning
  • Assumes codebook cluster distributions are well-defined; fails if semantics are ambiguous

Internal Use Cases:

  • Ranking systems: thinking tokens could represent intermediate ranking signals (e.g., relevance→diversity→freshness) before final score generation
  • LLM chains-of-thought: analogous to explicit reasoning steps in language models, applicable to multi-stage retrieval-ranking pipelines
  • RecSys hierarchies: e-commerce (category→subcategory→SKU) or video platforms (topic→creator→video)

Balanced Hierarchical Supervision via Contrastive Learning

Core Idea: Enforce interpretable semantics in reasoning paths by using contrastive learning to align thinking tokens with ground-truth codebook cluster distributions at each generation step, ensuring physically grounded and verifiable reasoning.

Problem It Solves: Prior reasoning-enhanced GR methods generate reasoning vectors with uninterpretable semantics and no verifiable supervision, making it impossible to validate whether reasoning is meaningful or just memorizing spurious patterns.

How It Works:

  1. For each SID generation step , obtain ground-truth codebook cluster distribution representing the true semantic cluster assignment for items at hierarchy level
  2. Define contrastive loss: for thinking token , treat its cluster distribution (via softmax of similarities to codebook) as positive anchor
  3. Apply InfoNCE-style loss:
  4. Backpropagate contrastive loss to update thinking token embeddings, explicitly aligning them with ground-truth semantics
  5. Ensures reasoning paths are not just plausible but grounded in verified item-cluster relationships from training data

Where It’s Useful:

  • Any multi-step generation requiring interpretable intermediate outputs
  • Supervised learning scenarios with ground-truth hierarchical labels
  • Recommendation systems with explicit item taxonomy/categorization

Limitations / Caveats:

  • Requires annotated ground-truth cluster labels (cannot be used in fully unsupervised settings)
  • Contrastive learning is sensitive to negative sampling and temperature hyperparameters
  • May overfit to training cluster distributions if taxonomy is biased or incomplete

These three techniques form the integrated S²GR framework: codebook optimization provides semantic grounding, thinking tokens implement stepwise reasoning, and contrastive supervision ensures those tokens are interpretable and verifiable.
