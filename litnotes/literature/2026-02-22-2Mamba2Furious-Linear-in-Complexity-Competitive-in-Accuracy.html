2Mamba2Furious: Linear in Complexity, Competitive in Accuracy

2Mamba2Furious: Linear in Complexity, Competitive in Accuracy

Source: arXiv:cs.LG Published: 2026-02-19 Rank: #7 (score: 0.70, max_sim: 0.81) Reading time: ~0 min Tags: Uncategorized

Abstract

Linear attention transformers have become a strong alternative to softmax attention due to their efficiency. However, linear attention tends to be less expressive and results in reduced accuracy compared to softmax attention. To bridge the accuracy gap between softmax attention and linear attention, we manipulate Mamba-2, a very strong linear attention variant. We first simplify Mamba-2 down to its most fundamental and important components, evaluating which specific choices make it most accurate. From this simplified Mamba variant (Mamba-2S), we improve the A-mask and increase the order of the hidden state, resulting in a method, which we call 2Mamba, that is nearly as accurate as softmax attention, yet much more memory efficient for long context lengths. We also investigate elements to Mamba-2 that help surpass softmax attention accuracy. Code is provided for all our experiments

Key Techniques

  • Mamba-2 simplification (Mamba-2S)
  • A-mask improvement
  • hidden state order increase
  • 2Mamba method development

2Mamba

Core Idea: A simplified and enhanced linear attention architecture derived from Mamba-2 that improves the A-mask and increases hidden state order to close the accuracy gap with softmax attention while retaining linear complexity.

Problem It Solves: Linear attention variants like Mamba-2 are memory-efficient but trail softmax attention in accuracy due to limited expressivity in their state-transition masks and low-order hidden states; 2Mamba targets this accuracy gap without sacrificing linear-time scaling.

How It Works:

  1. Simplify Mamba-2 → Mamba-2S: Strip Mamba-2 down to its most impactful components, ablating away elements that don’t contribute to accuracy, yielding a clean baseline (Mamba-2 Simplified).
  2. Improve the A-mask: Replace the original decay/selection mask in Mamba-2S with a more expressive A-mask that better controls how information flows and decays across the recurrent hidden state.
  3. Increase hidden state order: Expand the order of the recurrent hidden state (analogous to increasing state-space model (SSM) rank/degree), giving the model greater capacity to encode long-range dependencies within its fixed-size memory before each recurrent step.

Architecture:

flowchart LR
    A[Token Sequence] --> B[Mamba-2S\nStripped Core]
    B --> C[Enhanced A-mask\nExpressive Decay]
    C --> D[Higher-Order\nHidden State]
    D --> E[Linear Recurrence\nO-n Memory]
    E --> F[Output\nLogits]

Where It’s Useful:

  • Long-context language modeling where quadratic softmax attention is memory-prohibitive
  • Deployment-constrained inference (edge, streaming) requiring fixed memory footprint per token
  • Sequence modeling tasks (document understanding, code, genomics) with sequences exceeding typical transformer context budgets

Limitations / Caveats:

  • Still trails softmax attention slightly in accuracy on some benchmarks; the gap narrows but is not fully closed
  • Higher hidden state order increases per-step compute/memory relative to vanilla Mamba-2, partially trading recurrent efficiency for expressivity
  • Architectural improvements are empirically validated but the theoretical justification for the improved A-mask form may not be fully characterized

Internal Use Cases:

  • Long-context ranking/retrieval: Replace softmax attention in rerankers processing long documents or session histories with 2Mamba to enable O(n) memory scaling without major accuracy loss
  • Sequential recommendation: Model long user interaction histories recurrently with higher-order hidden states, capturing richer temporal patterns than standard SSMs
  • LLM serving with extended context: Use 2Mamba layers in place of attention in production LLMs to reduce KV-cache memory, enabling larger batch sizes or longer contexts at fixed GPU memory
