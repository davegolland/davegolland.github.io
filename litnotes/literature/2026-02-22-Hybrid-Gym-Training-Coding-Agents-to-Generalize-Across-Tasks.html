Hybrid-Gym: Training Coding Agents to Generalize Across Tasks

Source: arXiv:cs.CL Published: 2026-02-18 Rank: #2 (score: 1.00, max_sim: 0.82) Reading time: ~20 min Tags: LLM, Search Code: https://github.com/yiqingxyq/Hybrid-Gym

Abstract

When assessing the quality of coding agents, predominant benchmarks focus on solving single issues on GitHub, such as SWE-Bench. In contrast, in real use, these agents solve more various and complex tasks that involve other skills such as exploring codebases, testing software, and designing architecture. In this paper, we first characterize some transferable skills that are shared across diverse tasks by decomposing trajectories into fine-grained components, and derive a set of principles for designing auxiliary training tasks to teach language models these skills. Guided by these principles, we propose a training environment, Hybrid-Gym, consisting of a set of scalable synthetic tasks, such as function localization and dependency search. Experiments show that agents trained on our synthetic tasks effectively generalize to diverse real-world tasks that are not present in training, improving a base model by 25.4% absolute gain on SWE-Bench Verified, 7.9% on SWT-Bench Verified, and 5.1% on Commit-0 Lite. Hybrid-Gym also complements datasets built for the downstream tasks (e.g., improving SWE-Play by 4.9% on SWT-Bench Verified). Code available at: https://github.com/yiqingxyq/Hybrid-Gym.

Hybrid-Gym

Core Idea: A synthetic training environment composed of fine-grained, scalable auxiliary tasks derived from decomposed agent trajectories, designed to teach transferable coding-agent skills that generalize to unseen real-world software engineering benchmarks.

Problem It Solves: Predominant benchmarks (SWE-Bench) train and evaluate agents on single-issue GitHub tasks, leaving agents brittle to the broader skill set required in practice—codebase exploration, dependency analysis, test design, and architectural reasoning.

How It Works:

  1. Trajectory Decomposition: Real coding-agent trajectories are broken into fine-grained components (e.g., file localization, dependency tracing, test execution), revealing transferable sub-skills shared across diverse tasks.
  2. Principle Derivation: From these components, design principles for synthetic auxiliary tasks are extracted—tasks must be scalable, automatically verifiable, and representative of the identified sub-skills.
  3. Synthetic Task Construction: A gym of tasks is built (e.g., function localization, dependency search) using existing codebases as substrates, enabling unlimited generation of labeled training examples without human annotation.
  4. Agent Training: A base language model is fine-tuned on Hybrid-Gym tasks; the acquired skills transfer zero-shot to downstream benchmarks not seen during training.

Architecture:

flowchart LR
    A[Real Agent Trajectories] --> B[Trajectory Decomposition]
    B --> C[Transferable Sub-skill Identification]
    C --> D[Synthetic Task Design Principles]
    D --> E[Hybrid-Gym Task Suite\nfunction localization\ndependency search etc.]
    E --> F[Fine-tuned Coding Agent]
    F --> G[Generalization to\nSWE-Bench / SWT-Bench / Commit-0]

Where It’s Useful:

  • Training coding agents that must operate across heterogeneous software repositories without task-specific fine-tuning
  • Bootstrapping agent capabilities cheaply when labeled real-world task data is scarce or expensive

Limitations / Caveats:

  • Synthetic tasks may not fully capture the ambiguity and underspecification of real-world engineering requests
  • Gains are additive but moderate on some benchmarks (5.1% on Commit-0 Lite), suggesting synthetic coverage is still incomplete
  • Task design principles require initial investment in trajectory analysis from a capable base agent

Internal Use Cases:

  • Retrieval/ranking pipeline agents: Train sub-skills like dependency graph traversal and relevance localization on synthetic corpora before deploying agents on live ranking codebases
  • LLM self-improvement loops: Combine with Self-Challenging Agent (SCA) to let the agent generate new Hybrid-Gym-style tasks from its own trajectories, closing the data flywheel
  • Agent Symbolic Learning: Use trajectory decomposition as a signal source for Agent Symbolic Learning Framework to auto-optimize which sub-skill prompts and tools to activate per task type
