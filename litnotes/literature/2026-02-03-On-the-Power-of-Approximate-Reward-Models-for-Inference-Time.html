Abstract

Inference-time scaling has recently emerged as a powerful paradigm for improving the reasoning capability of large language models. Among various approaches, Sequential Monte Carlo (SMC) has become a particularly important framework, enabling iterative generation, evaluation, rejection, and resampling of intermediate reasoning trajectories. A central component in this process is the reward model, which evaluates partial solutions and guides the allocation of computation during inference. However, in practice, true reward models are never available. All deployed systems rely on approximate reward models, raising a fundamental question: Why and when do approximate reward models suffice for effective inference-time scaling? In this work, we provide a theoretical answer. We identify the Bellman error of the approximate reward model as the key quantity governing the effectiveness of SMC-based inference-time scaling. For a reasoning process of length , we show that if the Bellman error of the approximate reward model is bounded by , then combining this reward model with SMC reduces the computational complexity of reasoning from exponential in to polynomial in . This yields an exponential improvement in inference efficiency despite using only approximate rewards.

Sequential Monte Carlo (SMC) for Inference-Time Scaling

Core Idea: Use SMC to iteratively generate, evaluate, reject, and resample reasoning trajectories during inference, enabling adaptive computation allocation that asymptotically improves solution quality without retraining the base model.

Problem It Solves: Language models generate solutions in a single forward pass with no opportunity to backtrack or allocate more compute to promising partial solutions. SMC enables post-hoc reasoning refinement using a budget of additional inference calls and an approximate reward model to guide trajectory selection.

How It Works:

  1. Initialize a population of N parallel reasoning trajectories by sampling from the base LLM
  2. At each step t ∈ {1, …, T}, evaluate all trajectories using the approximate reward model V̂(s_t)
  3. Compute resampling weights based on relative trajectory quality: w_i ∝ V̂(s_t^i) / (1/N Σ_j V̂(s_t^j))
  4. Resample trajectories proportionally to weights (low-quality trajectories are pruned, high-quality ones duplicated)
  5. Continue generation for each trajectory to step t+1, then repeat evaluation-resampling cycle
  6. Return the highest-reward trajectory from the final population

Architecture:

flowchart TD
    A["Initialize N trajectories<br/>from base LLM"] --> B["Generate step t<br/>for all trajectories"]
    B --> C["Evaluate all trajectories<br/>with reward model V̂"]
    C --> D["Compute resampling weights<br/>w_i ∝ V̂(s_i)"]
    D --> E["Resample trajectories<br/>according to weights"]
    E --> F{Step t < T?}
    F -->|Yes| B
    F -->|No| G["Return best trajectory"]

Where It’s Useful:

  • Mathematical problem solving where intermediate correctness can be verified
  • Code generation with automated testing as reward signal
  • Multi-step reasoning tasks where some partial solutions are provably worse than others
  • Any domain with tractable intermediate evaluation (planning, dialogue, search)

Limitations / Caveats:

  • Requires a learned reward model to score intermediate states—poor reward models can misdirect the search
  • Resampling introduces variance in the particle population; small populations may collapse to single trajectories
  • Computational cost scales with N × T × (cost of generation + cost of evaluation), requiring careful budgeting
  • The exponential-to-polynomial complexity reduction only holds in theory; empirical speedup depends on reward model quality

Internal Use Cases:

  • Ranking systems could use SMC to iteratively refine candidate lists: generate initial ranking → evaluate subsets with expensive metrics → resample → refine
  • Recommendation refinement: initial broad retrieval → evaluate with user-aligned reward model → focus compute on promising items

Analysis: Theoretical Foundations for Approximate Reward Models in Inference-Time Scaling

This paper provides crucial theoretical grounding for the practical inference-time scaling approaches discussed in the matched notes. While Scaling LLM Test-Time Compute Optimally and Reasoning Beyond Limits: Advances and Open Problems for LLMs focus on empirical demonstrations of how test-time compute allocation improves reasoning performance, this work directly addresses a fundamental gap: why approximate reward models—which are the only tools available in practice—are sufficient for effectiveness. The identification of Bellman error as the key governing quantity transforms what could be seen as a practical limitation (using imperfect reward models) into a theoretically justified approach. By showing that O(1/T) Bellman error bounds enable polynomial-time complexity instead of exponential, this paper provides the mathematical justification for why the process reward models and self-evaluation mechanisms referenced in the Monte Carlo Tree Search notes can successfully guide SMC-based search despite being approximate.

The connection to SMC-based inference-time scaling directly extends the Monte Carlo Tree Search framework discussed in the vault. MCTS traditionally relies on accurate value estimates from simulation or learned models; however, this paper shows that the iterative evaluation, rejection, and resampling cycle inherent to SMC can tolerate substantial approximation errors in the reward model. This is particularly relevant to the “selection, expansion, simulation, and backpropagation phases” of MCTS—the paper’s Bellman error analysis essentially quantifies how much error can propagate through these phases while still achieving exponential speedup. This bridges the gap between the idealized MCTS algorithms (which assume perfect reward signals) and the deployed systems that must use imperfect process reward models or self-evaluation heuristics.

Finally, this work adds a quantitative framework for evaluating reward model quality in inference-time scaling systems. Rather than treating reward model approximation as an engineering inconvenience, the paper establishes that reward models with O(1/T) Bellman error form a theoretically principled sweet spot—they’re loose enough to be practically implementable, yet tight enough to preserve exponential computational benefits. This insight helps contextualize why contemporary approaches like those in DeepSeek-R1 and o1/o3 (mentioned in Reasoning Beyond Limits) succeed despite not having ground-truth reward signals. The paper thus provides both a lower bound on reward model quality (explaining why some approximate models fail) and an upper bound on acceptable error (showing diminishing returns beyond a certain threshold), creating a theoretical roadmap for future research on reward model design and training.
