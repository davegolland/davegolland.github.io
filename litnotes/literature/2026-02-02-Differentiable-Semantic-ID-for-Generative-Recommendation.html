Differentiable Semantic ID for Generative Recommendation

Differentiable Semantic ID for Generative Recommendation

Source: arXiv:cs.IR Published: 2026-01-27 Rank: #11 (score: 0.73, max_sim: 0.86) Reading time: ~0 min Tags: Uncategorized Code: https://arxiv.org/pdf/2601.19711v1

Abstract

Generative recommendation provides a novel paradigm in which each item is represented by a discrete semantic ID (SID) learned from rich content. Most existing methods treat SIDs as predefined and train recommenders under static indexing. In practice, SIDs are typically optimized only for content reconstruction rather than recommendation accuracy. This leads to an objective mismatch: the system optimizes an indexing loss to learn the SID and a recommendation loss for interaction prediction, but because the tokenizer is trained independently, the recommendation loss cannot update it. A natural approach is to make semantic indexing differentiable so that recommendation gradients can directly influence SID learning, but this often causes codebook collapse, where only a few codes are used. We attribute this issue to early deterministic assignments that limit codebook exploration, resulting in imbalance and unstable optimization. In this paper, we propose DIGER (Differentiable Semantic ID for Generative Recommendation), a first step toward effective differentiable semantic IDs for generative recommendation. DIGER introduces Gumbel noise to explicitly encourage early-stage exploration over codes, mitigating codebook collapse and improving code utilization. To balance exploration and convergence, we further design two uncertainty decay strategies that gradually reduce the Gumbel noise, enabling a smooth transition from early exploration to exploitation of learned SIDs. Extensive experiments on multiple public datasets demonstrate consistent improvements from differentiable semantic IDs. These results confirm the effectiveness of aligning indexing and recommendation objectives through differentiable SIDs and highlight differentiable semantic indexing as a promising research direction.

Key Techniques

  • Gumbel noise for codebook exploration
  • uncertainty decay strategies
  • differentiable semantic indexing
  • end-to-end SID learning

Uncertainty Decay Strategy (UDS)

Core Idea: Dynamically reduce Gumbel noise (temperature) over training epochs to progressively transition from broad codebook exploration to confident exploitation of learned semantic IDs, achieving stability without codebook collapse.

Problem It Solves: Constant Gumbel noise enables exploration but prevents convergence to sharp code assignments needed for stable recommendation inference. A static temperature cannot simultaneously support early exploration and late-stage exploitation—this technique balances both phases by explicitly scheduling noise decay.

How It Works:

  1. Initialize temperature: τ_0 = τ_init (high value, e.g., 1.0 to 5.0)
  2. Define epoch-based decay schedule—two strategies proposed:
    • Linear Decay: τ_t = τ_init · (1 - t/T) where t is current epoch, T is total epochs
    • Exponential Decay: τ_t = τ_init · λ^t where λ ∈ (0,1) is decay rate (e.g., λ=0.9995 per epoch)
  3. At each epoch t, use τ_t in Gumbel-Softmax: p_i^(t) = softmax((logits_i + G(0,1))/τ_t)
  4. As τ_t → 0, Gumbel-Softmax converges to hard argmax, producing deterministic code assignments
  5. Monitor codebook usage metrics and recommendation performance; adjust decay schedule if codes remain unused

Architecture:

Where It’s Useful:

  • Any discrete latent variable model requiring transition from exploration to exploitation
  • Curriculum learning strategies where early training benefits from stochasticity
  • Reinforcement learning with discrete action spaces using Gumbel-max trick
  • Multi-stage information retrieval where indexing changes gradually during training

Limitations / Caveats:

  • Decay schedule is a new hyperparameter requiring tuning (init temperature τ_0 and decay rate λ); poor choices perpetuate collapse or prevent convergence
  • Linear decay may be too aggressive early (premature convergence); exponential decay may decay too slowly in late stages
  • No principled method to set decay schedule a priori—may require dataset-specific tuning
  • If recommendation loss provides weak gradients, decay alone cannot prevent collapse; exploration benefit is limited

Internal Use Cases:

  • Annealing strategies for learned item embeddings in ranking systems where SIDs should adapt over epochs
  • Scheduled softening/hardening of assignment probabilities in hierarchical recommendation (e.g., category → subcategory → item)
