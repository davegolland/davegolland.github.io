Breaking Flat: A Generalised Query Performance Prediction Evaluation Framework

Source: arXiv:cs.IR Published: 2026-01-24 Rank: #3 (score: 0.99, max_sim: 0.82) Reading time: ~15 min Tags: Search, RAG, RecSys

Abstract

The traditional use-case of query performance prediction (QPP) is to identify which queries perform well and which perform poorly for a given ranking model. A more fine-grained and arguably more challenging extension of this task is to determine which ranking models are most effective for a given query. In this work, we generalize the QPP task and its evaluation into three settings: (i) SingleRanker MultiQuery (SRMQ-PP), corresponding to the standard use case; (ii) MultiRanker SingleQuery (MRSQ-PP), which evaluates a QPP model’s ability to select the most effective ranker for a query; and (iii) MultiRanker MultiQuery (MRMQ-PP), which considers predictions jointly across all query ranker pairs. Our results show that (a) the relative effectiveness of QPP models varies substantially across tasks (SRMQ-PP vs. MRSQ-PP), and (b) predicting the best ranker for a query is considerably more difficult than predicting the relative difficulty of queries for a given ranker.

Analysis

Extending Ranking Beyond Single-Stage Retrieval

The matched notes focus primarily on improving ranking within a fixed retrieval context—whether through selection-then-rank pipelines, reranking with query context, or stochastic diversification. “Breaking Flat” fundamentally challenges this single-ranker assumption by introducing MultiRanker scenarios (MRSQ-PP and MRMQ-PP). Where Selection-Then-Rank for Mathematical Retrieval assumes a dense retriever feeds into a fixed re-ranker, this paper asks a more foundational question: which ranking model should we use in the first place? This represents a shift from optimizing within a ranking architecture to selecting between ranking architectures—a meta-ranking problem that none of the matched notes address.

Prediction Complexity and the Ranker Selection Gap

A critical finding—that predicting the best ranker for a query (MRSQ-PP) is substantially harder than predicting query difficulty under a fixed ranker (SRMQ-PP)—reveals a blind spot in existing retrieval frameworks. The matched notes assume ranking models are interchangeable or that once selected, their performance remains stable across queries. This paper exposes that ranker effectiveness varies dramatically by query characteristics, suggesting that approaches like Reranking with Original Query Context may oversimplify by treating all queries similarly with respect to reranker selection. The paper implicitly argues that query-specific ranker adaptation—knowing when a BGE-style reranker is better than alternatives—requires new prediction signals beyond traditional QPP features.

New Directions in Query-Aware Model Selection

This work introduces a hierarchical prediction problem absent from prior notes: predicting not just document relevance but model suitability. This complements Stochastic Reranking’s focus on diversity by suggesting that even probabilistic ranking models benefit from upstream knowledge about which model family suits a given query. The MRMQ-PP setting—jointly predicting across all query-ranker pairs—hints at potential solutions: learning cross-domain features that characterize when retrieval/ranking architectures succeed or fail, potentially informed by query representation and ranker properties rather than document pools alone.
