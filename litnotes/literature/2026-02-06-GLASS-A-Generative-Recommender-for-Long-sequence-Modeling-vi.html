Abstract

Leveraging long-term user behavioral patterns is a key trajectory for enhancing the accuracy of modern recommender systems. While generative recommender systems have emerged as a transformative paradigm, they face hurdles in effectively modeling extensive historical sequences. To address this challenge, we propose GLASS, a novel framework that integrates long-term user interests into the generative process via SID-Tier and Semantic Search. We first introduce SID-Tier, a module that maps long-term interactions into a unified interest vector to enhance the prediction of the initial SID token. Unlike traditional retrieval models that struggle with massive item spaces, SID-Tier leverages the compact nature of the semantic codebook to incorporate cross features between the user’s long-term history and candidate semantic codes. Furthermore, we present semantic hard search, which utilizes generated coarse-grained semantic ID as dynamic keys to extract relevant historical behaviors, which are then fused via an adaptive gated fusion module to recalibrate the trajectory of subsequent fine-grained tokens. To address the inherent data sparsity in semantic hard search, we propose two strategies: semantic neighbor augmentation and codebook resizing. Extensive experiments on two large-scale real-world datasets, TAOBAO-MM and KuaiRec, demonstrate that GLASS outperforms state-of-the-art baselines, achieving significant gains in recommendation quality. Our codes are made publicly available to facilitate further research in generative recommendation.

Key Techniques

  • SID-Tier interest mapping
  • semantic hard search with dynamic keys
  • semantic neighbor augmentation
  • codebook resizing

GLASS (Generative Long-sequence Modeling via SID-Tier and Semantic Search)

Core Idea: A generative recommender framework that integrates long-term user interaction sequences into semantic ID (SID) generation through a two-stage process: SID-Tier for predicting initial semantic tokens using compact codebook features, and semantic hard search for dynamically retrieving relevant history based on generated coarse-grained tokens.

Problem It Solves: Generative recommender systems struggle to effectively leverage extensive historical user sequences (hundreds to thousands of interactions) when generating recommendations. Traditional approaches either truncate history (losing long-term patterns) or face scalability issues when modeling cross-features between long sequences and massive item spaces.

How It Works:

  1. SID-Tier Module: Maps long-term interaction sequences into a unified interest vector, then computes cross-features between this vector and all candidate semantic codes in the compact codebook (much smaller than item space) to predict the initial SID token with full historical context
  2. Semantic Hard Search: Uses the generated coarse-grained semantic ID as a dynamic key to retrieve relevant historical behaviors from the user’s long sequence, filtering thousands of interactions down to pertinent subset
  3. Adaptive Gated Fusion: Fuses retrieved historical behaviors to recalibrate the generation trajectory for subsequent fine-grained SID tokens, with semantic neighbor augmentation and codebook resizing to address data sparsity
  4. Generation Process: Autoregressively generates hierarchical semantic IDs (coarse-to-fine) with history-aware refinement at each token position

Architecture:

flowchart LR
    A[Long-term History] --> B[SID-Tier: Interest Vector]
    B --> C[Cross-features with Codebook]
    C --> D[Initial SID Token]
    D --> E[Semantic Hard Search]
    A --> E
    E --> F[Retrieved Relevant History]
    F --> G[Adaptive Gated Fusion]
    G --> H[Fine-grained SID Tokens]
    H --> I[Recommended Items]

Where It’s Useful:

  • E-commerce platforms with extensive user browsing/purchase histories (TAOBAO-MM dataset)
  • Video/content recommendation systems with long watch histories (KuaiRec dataset)
  • Any generative recommender requiring long-sequence modeling (>100 historical items)
  • Applications using hierarchical semantic ID representations (RQ-VAE, residual quantization)

Limitations / Caveats:

  • Requires pre-trained semantic codebook via methods like RQ-VAE; quality depends on codebook construction
  • Semantic hard search depends on quality of coarse-grained semantic ID generation; early errors cascade
  • Data sparsity in semantic codes requires augmentation strategies (neighbor expansion, resizing)
  • Computational overhead from cross-feature computation in SID-Tier, though mitigated by compact codebook size

Internal Use Cases:

  • Modeling long-term user engagement patterns in feed ranking while maintaining generative framework efficiency
  • Dynamic context retrieval for LLM-based recommenders: use coarse predictions to filter relevant history before fine-grained generation
  • Hierarchical token generation for any semantic ID-based recommendation system requiring historical context
  • Gated fusion strategies for combining retrieved context with generation state in sequential models
