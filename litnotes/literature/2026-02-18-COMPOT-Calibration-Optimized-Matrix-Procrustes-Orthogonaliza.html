COMPOT: Calibration-Optimized Matrix Procrustes Orthogonalization for Transformers Compression

COMPOT: Calibration-Optimized Matrix Procrustes Orthogonalization for Transformers Compression

Source: arXiv:cs.LG Published: 2026-02-16 Rank: #6 (score: 0.25, max_sim: 0.77) Reading time: ~0 min Tags: Uncategorized

Abstract

Post-training compression of Transformer models commonly relies on truncated singular value decomposition (SVD). However, enforcing a single shared subspace can degrade accuracy even at moderate compression. Sparse dictionary learning provides a more flexible union-of-subspaces representation, but existing approaches often suffer from iterative dictionary and coefficient updates. We propose COMPOT (Calibration-Optimized Matrix Procrustes Orthogonalization for Transformers), a training-free compression framework that uses a small calibration dataset to estimate a sparse weight factorization. COMPOT employs orthogonal dictionaries that enable closed-form Procrustes updates for the dictionary and analytical single-step sparse coding for the coefficients, eliminating iterative optimization. To handle heterogeneous layer sensitivity under a global compression budget, COMPOT further introduces a one-shot dynamic allocation strategy that adaptively redistributes layer-wise compression rates. Extensive experiments across diverse architectures and tasks show that COMPOT consistently delivers a superior quality-compression trade-off over strong low-rank and sparse baselines, while remaining fully compatible with post-training quantization for extreme compression. COMPOT (Calibration-Optimized Matrix Procrustes Orthogonalization for Transformers)

Core Idea: Replace truncated SVD with sparse dictionary learning using orthogonal dictionaries that admit closed-form Procrustes solutions and single-step sparse coding, enabling training-free Transformer weight compression without iterative optimization.

Problem It Solves: Standard low-rank SVD compression forces all weight structure into a single shared subspace, degrading accuracy at moderate compression ratios. Existing sparse dictionary learning approaches require expensive alternating iterative updates (dictionary update + sparse coding) that are slow and unstable. COMPOT eliminates both limitations simultaneously.

How It Works:

  1. Calibration-driven estimation: Pass a small calibration dataset through the model to collect layer-wise activation statistics, producing an empirical covariance estimate used to weight the importance of each weight component.
  2. Orthogonal dictionary factorization: Decompose each weight matrix W ≈ D·A where D is an orthogonal dictionary (D^T D = I). The orthogonality constraint converts the dictionary update into a Procrustes problem — solved in closed form via SVD of a single cross-product matrix, with no iterative refinement needed.
  3. Analytical sparse coding: Given a fixed orthogonal D, coefficients A are computed in a single step (A = D^T W projected/thresholded), exploiting the orthogonality to decouple coefficient estimation from dictionary optimization entirely.
  4. One-shot dynamic layer allocation: A global compression budget is redistributed across layers by estimating each layer’s sensitivity (via calibration loss proxy) and solving a one-shot allocation problem — layers that are more sensitive receive higher rank/sparsity budgets, heterogeneous layer importance is handled without per-layer tuning.

Architecture:

flowchart LR
    A[Pretrained\nTransformer] --> B[Calibration\nDataset Pass]
    B --> C[Layer Sensitivity\nEstimation]
    C --> D[One-Shot Budget\nAllocation]
    D --> E[Per-Layer\nOrthogonal Dictionary\nInit]
    E --> F[Closed-Form\nProcrustes Update\nfor D]
    F --> G[Single-Step\nSparse Coding\nfor A]
    G --> H[Compressed\nWeight: W ≈ D·A]
    H --> I[Optional:\nPost-Training\nQuantization]

Where It’s Useful:

  • Compressing large language models (LLMs) for inference on memory-constrained hardware without fine-tuning
  • Stacking with post-training quantization (PTQ) for extreme compression (e.g., 4-bit weights + low-rank factorization)
  • Any setting where calibration data is available but retraining is infeasible (proprietary models, fast deployment cycles)
  • Architectures with heterogeneous layer sensitivity (encoder-decoder models, MoE layers) where uniform compression rates are suboptimal

Limitations / Caveats:

  • Orthogonality constraint on D restricts the representational family compared to unconstrained dictionary learning — may underperform in highly anisotropic weight distributions
  • Calibration data quality and domain match affect the sensitivity estimates; out-of-distribution calibration can misallocate budget
  • Closed-form sparse coding (hard thresholding of D^T W) may be suboptimal versus pursuit algorithms (OMP, LASSO) for highly non-uniform sparsity patterns
  • Compression gains are bounded by the union-of-subspaces assumption; extremely low bitwidth targets may still require fine-tuning recovery

Internal Use Cases:

  • Ranking/RecSys: Compress embedding projection layers and interaction MLP blocks in two-tower or cross-attention retrieval models for lower-latency inference without retraining on private interaction data
  • LLM serving: Apply to KV-cache projection matrices (K/V heads) or FFN layers in serving infrastructure to reduce memory bandwidth and fit larger batch sizes
  • Distillation alternative: Use as a cheaper substitute to knowledge distillation when labeled data is unavailable — calibration only requires unlabeled forward passes
  • Quantization pipeline integration: Stack COMPOT + PTQ as a two-stage compression pipeline to push models below 4-bit effective precision on edge deployments
