Source: arXiv:cs.CL Published: 2026-02-05 Rank: #1 (score: 0.98, max_sim: 0.83) Reading time: ~0 min Tags: Early-Exit, Speculative-Decoding

Abstract

Existing techniques for accelerating language model inference, such as speculative decoding, require training auxiliary speculator models and building and deploying complex inference pipelines. We consider a new approach for converting a pretrained autoregressive language model from a slow single next token prediction model into a fast standalone multi-token prediction model using a simple online distillation objective. The final model retains the exact same implementation as the pretrained initial checkpoint and is deployable without the addition of any auxiliary verifier or other specialized inference code. On GSM8K, our method produces models that can decode more than faster on average at drop in accuracy relative to single token decoding performance.

Key Techniques

  • Speculative decoding
  • online distillation
  • multi-token prediction
  • single-to-multi-token conversion

Multi-Token Prediction via Self-Distillation (MTP-SD)

Core Idea: Convert a pretrained autoregressive LM into a multi-token predictor by training it to simultaneously predict multiple future tokens at each position, using the original model’s own single-token predictions as soft targets for distillation.

Problem It Solves: Traditional speculative decoding requires training separate draft models and complex inference pipelines; this approach achieves multi-token generation with a single standalone model that retains the exact same architecture and deployment footprint as the original.

How It Works:

  1. Take a pretrained autoregressive LM and add multiple prediction heads (one per future token position)
  2. During training, generate soft target distributions for positions t+1, t+2, …, t+n using the frozen original model
  3. Train the model to predict token t+1 autoregressively (hard labels) while simultaneously predicting t+2, t+3, etc. using KL-divergence against the teacher’s distributions
  4. At inference, greedily decode multiple tokens per forward pass by using the multi-token predictions, verifying correctness, and recovering from errors

Architecture:

flowchart LR
    A[Input tokens] --> B[Shared Transformer Layers]
    B --> C[Head 1: predict t+1]
    B --> D[Head 2: predict t+2]
    B --> E[Head 3: predict t+3]
    C --> F[Primary token]
    D --> G[Speculative tokens]
    E --> G

Where It’s Useful:

  • Accelerating inference for math reasoning tasks (GSM8K: 3× speedup at <5% accuracy drop)
  • Deployment-constrained environments where auxiliary models are impractical
  • Applications requiring low-latency generation without infrastructure changes

Limitations / Caveats:

  • Accuracy-speed tradeoff: higher speedups come with larger accuracy degradation
  • Requires retraining/fine-tuning the base model (not a zero-shot technique)
  • Effectiveness varies by domain; speculative tokens may be less accurate for creative/diverse generation
  • Greedy decoding only; unclear how to extend to sampling-based generation

Internal Use Cases:

  • Fast structured output generation (JSON, code) where early tokens strongly constrain later ones
  • Low-latency ranking model reranking where candidate diversity is limited
  • Accelerating multi-turn dialogue systems where context creates predictable continuations
