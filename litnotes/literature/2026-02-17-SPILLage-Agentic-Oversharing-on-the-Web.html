SPILLage: Agentic Oversharing on the Web

Source: HuggingFace Published: 2026-02-13 Rank: #2 (score: 0.77, max_sim: 0.81) Reading time: ~30 min Tags: RecSys, RAG, LLM Code: https://github.com/jrohsc/SPILLage

Abstract

LLM-powered agents are beginning to automate user’s tasks across the open web, often with access to user resources such as emails and calendars. Unlike standard LLMs answering questions in a controlled ChatBot setting, web agents act “in the wild”, interacting with third parties and leaving behind an action trace. Therefore, we ask the question: how do web agents handle user resources when accomplishing tasks on their behalf across live websites? In this paper, we formalize Natural Agentic Oversharing — the unintentional disclosure of task-irrelevant user information through an agent trace of actions on the web. We introduce SPILLage, a framework that characterizes oversharing along two dimensions: channel (content vs. behavior) and directness (explicit vs. implicit). This taxonomy reveals a critical blind spot: while prior work focuses on text leakage, web agents also overshare behaviorally through clicks, scrolls, and navigation patterns that can be monitored. We benchmark 180 tasks on live e-commerce sites with ground-truth annotations separating task-relevant from task-irrelevant attributes. Across 1,080 runs spanning two agentic frameworks and three backbone LLMs, we demonstrate that oversharing is pervasive with behavioral oversharing dominates content oversharing by 5x. This effect persists — and can even worsen — under prompt-level mitigation. However, removing task-irrelevant information before execution improves task success by up to 17.9%, demonstrating that reducing oversharing improves task success. Our findings underscore that protecting privacy in web agents is a fundamental challenge, requiring a broader view of “output” that accounts for what agents do on the web, not just what they type. Our datasets and code are available at https://github.com/jrohsc/SPILLage.

SPILLage: Agentic Oversharing Taxonomy and Benchmarking Framework

Core Idea: A two-dimensional taxonomy (channel × directness) that formally characterizes how LLM web agents unintentionally disclose task-irrelevant user information through both textual outputs and observable behavioral traces.

Problem It Solves: Prior privacy work on LLM agents focused exclusively on text leakage (what agents type), missing a larger and more insidious class of privacy violation: behavioral signals (clicks, scrolls, navigation) that third parties can monitor to infer sensitive user attributes even when no private text is transmitted.

How It Works:

  1. Taxonomy definition: Classify oversharing along two axes — channel (content: text/data entered vs. behavior: interaction patterns) and directness (explicit: directly stated sensitive info vs. implicit: inferable from context or patterns)
  2. Task construction: Define 180 tasks on live e-commerce sites with ground-truth annotations that partition user attributes into task-relevant vs. task-irrelevant; oversharing occurs when an agent exposes task-irrelevant attributes
  3. Benchmarking: Run 1,080 agent trajectories (2 frameworks × 3 LLMs × 180 tasks), scoring each trajectory for content oversharing (text entered into forms/search) and behavioral oversharing (pages visited, items clicked, filters applied that reveal private attributes)
  4. Mitigation testing: Compare prompt-level mitigations (instructions to be privacy-preserving) against context-stripping (removing task-irrelevant info from the agent’s input before execution)

Architecture:

flowchart LR
    A[User Task + Profile] --> B[Attribute Partitioner\ntask-relevant vs. irrelevant]
    B --> C[Web Agent\nFramework × LLM]
    C --> D[Action Trace\nclicks/scrolls/text]
    D --> E1[Content Oversharing\nScorer]
    D --> E2[Behavioral Oversharing\nScorer]
    E1 --> F[SPILLage Score]
    E2 --> F
    G[Mitigation: Context Stripping] --> B

Where It’s Useful:

  • Auditing LLM agent deployments that access personal data (email, calendar, health) before acting on the open web
  • Designing privacy-preserving agent architectures with minimal-disclosure task planning
  • Evaluating agentic frameworks (e.g., browser-use, OpenHands) for privacy compliance

Limitations / Caveats:

  • Behavioral oversharing requires a threat model where the website itself is the adversary — less applicable when the site is trusted
  • Prompt-level mitigation not only fails but can worsen oversharing, suggesting the problem is architectural, not instructable
  • Benchmark is scoped to e-commerce; generalization to other domains (healthcare, finance) is unvalidated

Internal Use Cases:

  • Agentic recommender systems: agents fetching user context to personalize recommendations may behaviorally signal preference signals to third-party sites — SPILLage’s taxonomy can audit what the agent’s navigation reveals about the user beyond what was explicitly shared
  • LLM-powered ranking pipelines: if agents query external catalogs or APIs on behalf of users, behavioral traces (query sequences, filter selections) could leak ranking signals or user segment membership
  • Context stripping as a design pattern: the finding that removing task-irrelevant attributes improves task success by 17.9% suggests that minimal-context prompting is both a privacy and performance optimization for agentic RAG pipelines
