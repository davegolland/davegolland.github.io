Abstract

Quantization has significantly improved the compute and memory efficiency of Large Language Model (LLM) training. However, existing approaches still rely on accumulating their updates in high-precision: concretely, gradient updates must be applied to a high-precision weight buffer, known as . This buffer introduces substantial memory overhead, particularly for Sparse Mixture of Experts (SMoE) models, where model parameters and optimizer states dominate memory usage. To address this, we introduce the Error-Compensating Optimizer (ECO), which eliminates master weights by applying updates directly to quantized parameters. ECO quantizes weights after each step and carefully injects the resulting quantization error into the optimizer momentum, forming an error-feedback loop with no additional memory. We prove that, under standard assumptions and a decaying learning rate, ECO converges to a constant-radius neighborhood of the optimum, while naive master-weight removal can incur an error that is inversely proportional to the learning rate. We show empirical results for pretraining small Transformers (30-800M), a Gemma-3 1B model, and a 2.1B parameter Sparse MoE model with FP8 quantization, and fine-tuning DeepSeek-MoE-16B in INT4 precision. Throughout, ECO matches baselines with master weights up to near-lossless accuracy, significantly shifting the static memory vs validation loss Pareto frontier.

Master Weight Elimination with Direct Quantized Parameter Updates

Core Idea: Remove the high-precision master weight buffer entirely by applying optimizer updates directly to quantized weights, reducing memory overhead proportional to the number of model parameters.

Problem It Solves: Traditional quantized training maintains two weight representations—quantized weights for computation and high-precision master weights for gradient accumulation—doubling memory cost. This is particularly acute in models with billions of parameters and complex optimizer states.

How It Works:

  1. Initialize weights in target quantization format (e.g., FP8):
  2. Cast quantized weights to full-precision only temporarily for the gradient update computation
  3. Apply the optimizer update step directly: (in full-precision)
  4. Immediately quantize the updated weights back to target format:
  5. Discard the full-precision intermediate value; no master weight buffer is retained between steps
  6. The quantization error from step 4 is captured and fed back through the error-feedback mechanism to the next update

Where It’s Useful:

  • SMoE model training where parameter count × optimizer state memory is prohibitive
  • Single-GPU fine-tuning of billion-parameter models
  • Memory-constrained distributed training scenarios
  • Production training pipelines requiring maximum memory efficiency

Limitations / Caveats:

  • Without the error-feedback mechanism (ECO), naive master weight removal causes error accumulation inversely proportional to learning rate, degrading convergence
  • Requires hardware support for efficient FP8/INT4 operations to avoid compute overhead from repeated quantization
  • May introduce slightly higher numerical instability if quantization precision is too aggressive
  • Decaying learning rate schedule is necessary for theoretical convergence guarantees

Internal Use Cases:

  • Training dense ranking models for web search with FP8 quantization, reducing memory cost by ~50%
  • Fine-tuning embedding models (e.g., 1B-16B parameters) in INT4 to fit on single GPUs
  • Iterative refinement of recommendation rankers where multiple training runs are needed
