Bridging Graph Structure and Knowledge-Guided Editing for Interpretable Temporal Knowledge Graph Reasoning

Source: arXiv:cs.LG Published: 2026-01-29 Rank: #6 (score: 0.99, max_sim: 0.83) Reading time: ~20 min Tags: RAG, LLM, Graph

Abstract

Temporal knowledge graph reasoning (TKGR) aims to predict future events by inferring missing entities with dynamic knowledge structures. Existing LLM-based reasoning methods prioritize contextual over structural relations, struggling to extract relevant subgraphs from dynamic graphs. This limits structural information understanding, leading to unstructured, hallucination-prone inferences especially with temporal inconsistencies. To address this problem, we propose IGETR (Integration of Graph and Editing-enhanced Temporal Reasoning), a hybrid reasoning framework that combines the structured temporal modeling capabilities of Graph Neural Networks (GNNs) with the contextual understanding of LLMs. IGETR operates through a three-stage pipeline. The first stage aims to ground the reasoning process in the actual data by identifying structurally and temporally coherent candidate paths through a temporal GNN, ensuring that inference starts from reliable graph-based evidence. The second stage introduces LLM-guided path editing to address logical and semantic inconsistencies, leveraging external knowledge to refine and enhance the initial paths. The final stage focuses on integrating the refined reasoning paths to produce predictions that are both accurate and interpretable. Experiments on standard TKG benchmarks show that IGETR achieves state-of-the-art performance, outperforming strong baselines with relative improvements of up to 5.6% on Hits@1 and 8.1% on Hits@3 on the challenging ICEWS datasets. Additionally, we execute ablation studies and additional analyses confirm the effectiveness of each component.

Key Techniques

  • Temporal GNN for path identification
  • LLM-guided path editing
  • Hybrid GNN-LLM reasoning framework

LLM-Guided Path Editing and Refinement

Core Idea: Use LLM reasoning to detect and correct logical, semantic, and temporal inconsistencies in candidate paths extracted from GNNs, leveraging external knowledge to enhance path validity before final inference.

Problem It Solves: Graph-extracted paths may be structurally valid but logically or semantically flawed (e.g., violating commonsense, containing contradictory relations, or having conceptual misalignment). Raw GNN paths lack contextual reasoning to catch these issues. LLM-guided refinement bridges structural and semantic validity.

How It Works:

  1. Input: Candidate paths from temporal GNN (sequences of entities and relations: e₁ —r₁→ e₂ —r₂→ … —rₙ→ eₙ₊₁)
  2. Prompt LLM with: path sequence, entity/relation definitions, external knowledge context, and query goal
  3. LLM performs path validation: checks for semantic consistency (do relations make logical sense?), temporal consistency (do timestamps align with real-world causality?), and factual accuracy (do facts contradict known knowledge?)
  4. LLM generates refinements: reorder path hops if needed, substitute entities/relations with more semantically appropriate alternatives, insert missing intermediate steps, or prune invalid segments
  5. Output: Edited paths with improved logical coherence, semantic alignment, and temporal validity

Architecture:

flowchart LR
    A["GNN Candidate<br/>Paths"] --> B["Serialize Paths<br/>+ Context"]
    B --> C["LLM Validation<br/>& Refinement"]
    C --> D["Check: Semantic<br/>Consistency?"]
    D -->|Issues Found| E["Generate<br/>Edits"]
    D -->|Valid| F["Refined Paths"]
    E --> F

Where It’s Useful:

  • Knowledge graph completion with semantic verification
  • Event prediction in domains requiring commonsense reasoning
  • Fact-checking and knowledge base cleaning
  • Temporal event chains (e.g., detecting anachronisms)

Limitations / Caveats:

  • LLM refinement adds latency and computational cost per path
  • LLMs can hallucinate even when “guided”—corrections may introduce new errors
  • Requires careful prompt engineering to balance over-aggressive editing vs. under-editing
  • May struggle with domain-specific semantics if not well-represented in training data

Internal Use Cases:

  • In LLM-based ranking: using path editing to refine retrieved candidate ranking lists by checking semantic coherence with user query intent
  • For recommendation explanations: editing user-item reasoning paths to ensure they’re both structurally sound (from interaction graphs) and semantically sensible (validated by LLM knowledge)

Path Integration and Ensemble Prediction

Core Idea: Synthesize multiple refined reasoning paths into a unified prediction by aggregating evidence from structurally validated, semantically refined, and temporally consistent paths.

Problem It Solves: Single-path reasoning is brittle and prone to errors. Multiple candidate paths capture different valid reasoning chains; integrating them improves robustness and allows confidence scoring. Naive averaging ignores path quality and relevance differences.

How It Works:

  1. Input: Set of K refined paths from LLM-guided editing, each with confidence scores or quality metrics (path length, consistency score, LLM validation confidence)
  2. Rank paths by quality: combine structural confidence (GNN scores), semantic validity (LLM refinement success), and temporal coherence metrics
  3. Aggregate predictions:
    • For each refined path, extract predicted tail entity eₙ₊₁ and confidence cᵢ
    • Compute weighted ensemble: P(e|head, relation, time) = Σᵢ αᵢ · P(eᵢ|pathᵢ) where weights αᵢ reflect path quality
    • Alternative: use path voting with confidence thresholds
  4. Output: Final predicted entity with confidence score and interpretable reasoning trace (top-k contributing paths)

Architecture:

flowchart LR
    A["Refined Paths<br/>with Scores"] --> B["Rank by Quality<br/>GNN + LLM Metrics"]
    B --> C["Extract Predictions<br/>& Confidences"]
    C --> D["Weighted<br/>Aggregation"]
    D --> E["Final Prediction<br/>+ Confidence"]
    E --> F["Interpretable Trace<br/>Top-k Paths"]

Where It’s Useful:

  • Temporal knowledge graph link prediction (multi-path evidence fusion)
  • Multi-hop reasoning systems requiring interpretability
  • Ensemble methods for improving prediction robustness
  • Confidence calibration in structured reasoning

Limitations / Caveats:

  • Weight aggregation strategy (how to combine path scores?) requires tuning and may be domain-sensitive
  • Interpretability benefit decreases if too many low-quality paths are included
  • Doesn’t account for path correlations—redundant paths may inflate confidence artificially
  • Ensemble overhead: storing and aggregating multiple paths increases memory and computation

Internal Use Cases:

  • In ranking systems: combining signals from multiple retrieval paths (e.g., collaborative filtering paths + content-based paths) with learned path quality weights
  • For LLM-based recommendation: ensemble multiple reasoning chains to ground predictions in diverse evidence while maintaining interpretability via contributing path display
