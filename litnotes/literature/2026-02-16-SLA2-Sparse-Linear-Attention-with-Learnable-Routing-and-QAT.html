SLA2: Sparse-Linear Attention with Learnable Routing and QAT

SLA2: Sparse-Linear Attention with Learnable Routing and QAT

Source: arXiv:cs.AI Published: 2026-02-13 Rank: #8 (score: 0.89, max_sim: 0.83) Reading time: ~0 min Tags: Uncategorized

Abstract

Sparse-Linear Attention (SLA) combines sparse and linear attention to accelerate diffusion models and has shown strong performance in video generation. However, (i) SLA relies on a heuristic split that assigns computations to the sparse or linear branch based on attention-weight magnitude, which can be suboptimal. Additionally, (ii) after formally analyzing the attention error in SLA, we identify a mismatch between SLA and a direct decomposition into sparse and linear attention. We propose SLA2, which introduces (I) a learnable router that dynamically selects whether each attention computation should use sparse or linear attention, (II) a more faithful and direct sparse-linear attention formulation that uses a learnable ratio to combine the sparse and linear attention branches, and (III) a sparse + low-bit attention design, where low-bit attention is introduced via quantization-aware fine-tuning to reduce quantization error. Experiments show that on video diffusion models, SLA2 can achieve 97% attention sparsity and deliver an 18.6x attention speedup while preserving generation quality.

Key Techniques

  • Learnable router for dynamic attention selection
  • Faithful sparse-linear attention formulation with learnable ratio
  • Sparse + low-bit attention design via quantization-aware fine-tuning

SLA2 (Sparse-Linear Attention with Learnable Routing and QAT)

Core Idea: A hybrid attention mechanism that uses a learnable router to dynamically assign each attention computation to either sparse or linear attention branches, combined with a learnable mixing ratio and quantization-aware training to achieve extreme sparsity (97%) while maintaining generation quality in diffusion models.

Problem It Solves: Original SLA uses a suboptimal heuristic (attention-weight magnitude) to split computations between sparse and linear branches, and exhibits a mathematical mismatch between its formulation and the true sparse-linear decomposition; this leads to inefficient routing and degraded quality under high sparsity.

How It Works:

  1. Learnable Router: A lightweight trainable network examines Q and K representations and predicts whether each attention computation should use the sparse attention branch or the linear attention branch (replacing magnitude-based heuristics)
  2. Direct Sparse-Linear Formulation: Computes attention as Attention = α × Sparse_Attention + (1-α) × Linear_Attention where α is a learnable scalar parameter that directly combines the two branches with proper weighting
  3. Quantization-Aware Fine-Tuning (QAT): Applies low-bit quantization to the linear attention branch during fine-tuning, reducing memory bandwidth and computation while minimizing quantization error through QAT
  4. Output: Produces the same attention output shape with 18.6× speedup and 97% sparsity on video diffusion tasks

Architecture:

flowchart LR
    A[Q, K, V] --> B[Learnable Router]
    A --> C[Sparse Attention Branch]
    A --> D[Linear Attention Branch + QAT]
    B --> E{Route Decision}
    E -->|Sparse| C
    E -->|Linear| D
    C --> F[Weighted Combiner]
    D --> F
    F -->|α sparse + 1-α linear| G[Attention Output]

Where It’s Useful:

  • Video generation models requiring long-context attention (e.g., text-to-video diffusion)
  • Large-scale diffusion models where attention is the computational bottleneck
  • Real-time or latency-sensitive generative applications needing inference speedup
  • Multi-modal transformers processing high-resolution spatial-temporal data

Limitations / Caveats:

  • Requires fine-tuning with the learnable router and mixing ratio (not a drop-in replacement)
  • Optimal router architecture and α initialization may be task-dependent
  • Quantization benefits depend on hardware support for low-bit operations
  • May not generalize to non-diffusion attention patterns (e.g., causal LLM decoding) without adaptation

Internal Use Cases:

  • RecSys ranking models: Apply learnable routing to decide per-query whether to use sparse (top-k candidate) or linear (LSH/kernel) attention over large item catalogs
  • Long-context LLMs: Route between sparse (attending to retrieved or top-k tokens) and linear (approximating global context) for efficient multi-document reasoning
  • Multimodal embeddings: Use QAT on cross-modal attention (image-text) to reduce memory footprint during inference while maintaining alignment quality
  • Real-time recommendation: Deploy quantized linear attention branch for low-latency scoring while using sparse branch for high-value personalization signals
