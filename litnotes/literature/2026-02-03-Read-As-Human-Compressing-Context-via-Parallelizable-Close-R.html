Read As Human: Compressing Context via Parallelizable Close Reading and Skimming

Read As Human: Compressing Context via Parallelizable Close Reading and Skimming

Source: arXiv:cs.CL Published: 2026-02-02 Rank: #10 (score: 0.93, max_sim: 0.86) Reading time: ~0 min Tags: Uncategorized Code: https://arxiv.org/pdf/2602.01840v1

Abstract

Large Language Models (LLMs) demonstrate exceptional capability across diverse tasks. However, their deployment in long-context scenarios is hindered by two challenges: computational inefficiency and redundant information. We propose RAM (Read As HuMan), a context compression framework that adopts an adaptive hybrid reading strategy, to address these challenges. Inspired by human reading behavior (i.e., close reading important content while skimming less relevant content), RAM partitions the context into segments and encodes them with the input query in parallel. High-relevance segments are fully retained (close reading), while low-relevance ones are query-guided compressed into compact summary vectors (skimming). Both explicit textual segments and implicit summary vectors are concatenated and fed into decoder to achieve both superior performance and natural language format interpretability. To refine the decision boundary between close reading and skimming, we further introduce a contrastive learning objective based on positive and negative query-segment pairs. Experiments demonstrate that RAM outperforms existing baselines on multiple question answering and summarization benchmarks across two backbones, while delivering up to a 12x end-to-end speedup on long inputs (average length 16K; maximum length 32K).

Key Techniques

  • Adaptive hybrid reading strategy
  • Query-guided compression
  • Contrastive learning objective

Contrastive Learning Objective for Relevance Refinement

Core Idea: Train the relevance scoring function via contrastive learning on positive query-segment pairs (truly relevant) and negative pairs (irrelevant) to sharpen the decision boundary between close reading and skimming.

Problem It Solves: A naive relevance scorer may conflate partial relevance with true relevance, leading to incorrect routing decisions. The contrastive objective ensures the model learns a discriminative boundary that reliably distinguishes segments worth fully retaining from those safe to compress.

How It Works:

  1. Positive Pair Construction: For a given query, identify segments that are semantically aligned or answer-bearing (ground truth relevant)
  2. Negative Pair Construction: Sample segments that are semantically misaligned or irrelevant to the query
  3. Contrastive Loss Formulation: Apply a contrastive loss (e.g., NT-Xent, InfoNCE) to push positive segment encodings close to the query encoding and negative encodings far:
    • Loss = -log[ exp(sim(q, seg+) / τ) / (exp(sim(q, seg+) / τ) + Σ exp(sim(q, seg−) / τ)) ]
    • where τ is temperature, sim() is cosine similarity or dot product
  4. Joint Training: Train the segment encoder and relevance scorer end-to-end alongside the main task loss
  5. Threshold Calibration: Use validation data to find the optimal relevance threshold that maximizes task performance

Where It’s Useful:

  • Refining relevance estimation in any adaptive compression scheme
  • Improving query-document matching in retrieval systems
  • Training discriminative encoders for selective processing tasks

Limitations / Caveats:

  • Requires labeled positive/negative segment pairs during training, increasing annotation overhead
  • Contrastive loss is sensitive to batch size and negative sampling strategy
  • The learned boundary may not generalize to out-of-domain queries or document types
  • Threshold calibration is task-dependent and may require retuning across different domains

Internal Use Cases:

  • Improving relevance signals in ranking models where some documents should be fully scored and others filtered early
  • Training efficient encoding models for real-time ranking that compress low-scoring candidates into placeholders

Analysis: RAM in Context of Existing Knowledge

Extending Context Compression Beyond Static Filtering

The RAM paper advances the Context Compression note by introducing adaptive and learned compression strategies rather than static filtering approaches. While the vault note describes compression as applying content filtering and salience scoring, RAM proposes a more sophisticated hybrid model: parallel query-segment encoding where relevance is dynamically determined relative to the input query, not just measured by generic importance. This is a meaningful shift—instead of pre-computing which content is important (keyword extraction, summarization), RAM makes compression decisions contingent on what the user actually asks. The contrastive learning objective the paper introduces (positive/negative query-segment pairs) provides a trainable mechanism to refine the close reading/skimming boundary, moving beyond heuristic filtering toward learned relevance judgment. This directly addresses the information utility problem in the vault note while adding a query-adaptive dimension.

Bridging Long-Context Reliability and Practical Performance

The RAM approach implicitly tackles concerns raised in the Long Context Degradation in LLMs note about non-uniform model performance on complex, distractor-rich inputs. By selectively close-reading high-relevance segments while skimming others, RAM avoids the “Needle in a Haystack” pitfall—the model isn’t just finding needles in uniform hay, it’s actively distinguishing signal from noise before decoding. The 12x speedup on 16K-32K length inputs demonstrates that this compression strategy maintains performance while reducing the computational burden that typically compounds degradation in long contexts. Where the vault note warns against misplaced confidence in long-context reliability, RAM offers a concrete mechanism to improve both speed and semantic fidelity by mimicking human reading behavior.

New Technique: Parallelizable Close Reading as Efficiency Lever

RAM introduces a novel procedural insight: that context compression can be parallelized across segments without sequential processing bottlenecks. The parallel encoding of segments with the query, followed by selective retention versus compression, is computationally different from traditional hierarchical or sequential summarization pipelines. This addresses a gap in the vault notes—they discuss compression mechanisms but don’t explicitly address parallelizability constraints. By keeping high-relevance segments in natural language (not just as latent vectors) while compressing others into summary vectors, RAM also preserves interpretability, a practical advantage not highlighted in the more abstract compression frameworks described in the matched notes. This hybrid explicit/implicit representation bridges theoretical efficiency gains with real-world usability.
