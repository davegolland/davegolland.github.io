Source: S2:Proceedings of the Recommender Systems 2025 Published: 2025 Rank: #10 (score: 0.99, max_sim: 0.83) Reading time: ~20 min Tags: Embeddings, LLM, Search, RecSys Code: https://github.com/knife982000/RecSys2025Challenge

Abstract

Universal user representations hold the promise of reducing the need for task-specific modelling in personalized systems, enabling a single embedding to support a wide range of predictive tasks such as churn prediction, product propensity, and category-level recommendations. In this paper, we introduce BEHAV-E (Behavioural Embedding via Hybrid Action Variational Encoder), a self-supervised architecture that learns rich, 2048-dimensional user embeddings from multi-event behavioural data. BEHAV-E integrates both temporal and semantic signals by combining kernel density estimation (KDE) of user activity timelines, product and category embeddings via shared embedding bags, and an LSTM-based autoencoder for modelling search query semantics. We evaluate our approach within the RecSys Universal Behavioural modelling Challenge. Our results demonstrate that BEHAV-E effectively captures complex, multi-modal user behaviour in a compact and transferable embedding format. This paper provides an overview of the approach we used as team DArgk1 for the ACM RecSys Challenge 2025. Our submission achieved the 13th rank and sum of scores of 4.5713 (borda count 483) in the competition academia-track final results. We release our source code at: https://github.com/knife982000/RecSys2025Challenge

BEHAV-E (Behavioural Embedding via Hybrid Action Variational Encoder)

Core Idea: A self-supervised architecture that learns universal 2048-dimensional user embeddings from multi-event behavioral data by fusing temporal activity patterns (via kernel density estimation), categorical item/category embeddings, and semantic search query representations (via LSTM autoencoder).

Problem It Solves: Traditional user representations either require task-specific modeling for each downstream prediction task (churn, propensity, recommendations) or fail to capture the multi-modal nature of user behavior (temporal patterns, item interactions, search intent), limiting transferability and expressiveness.

How It Works:

  1. Temporal encoding: Apply kernel density estimation (KDE) to user activity timelines to capture when users are active and behavioral rhythm patterns
  2. Categorical encoding: Use shared embedding bags to encode product IDs and category IDs from user interaction history into dense vectors
  3. Semantic encoding: Feed user search queries into an LSTM-based autoencoder to extract latent representations of search intent and textual preferences
  4. Fusion: Combine temporal (KDE), categorical (embedding bags), and semantic (LSTM) signals into a single 2048-dimensional embedding vector per user
  5. Self-supervised training: Train the variational encoder to reconstruct behavioral signals without task-specific labels, enabling universal representation learning

Architecture:

flowchart LR
    A[User Behavioral Data] --> B[KDE Module<br/>Temporal Patterns]
    A --> C[Embedding Bags<br/>Products & Categories]
    A --> D[LSTM Autoencoder<br/>Search Queries]
    B --> E[Fusion Layer]
    C --> E
    D --> E
    E --> F[2048-dim<br/>User Embedding]

Where It’s Useful:

  • Cross-task personalization systems requiring a single user representation for multiple downstream tasks (churn, propensity scoring, recommendations)
  • Cold-start scenarios where transferable embeddings enable quick adaptation to new prediction tasks
  • Large-scale recommendation platforms needing compact yet expressive user representations

Limitations / Caveats:

  • Requires diverse behavioral signals (activity timestamps, item interactions, search queries) to be effective; degrades with sparse or single-modality data
  • Fixed 2048-dimensional embedding may be overparameterized for simpler tasks or underparameterized for extremely complex user behavior
  • Self-supervised training without explicit task supervision may not optimize for specific downstream metrics without fine-tuning

Internal Use Cases:

  • Foundation model user towers: BEHAV-E-style fusion could replace or augment static user embeddings in two-tower retrieval models by encoding temporal, categorical, and semantic signals into a single dense representation
  • Cross-domain knowledge transfer: Universal embeddings trained on one product vertical (e.g., e-commerce) could initialize models for related domains (e.g., content recommendations)
  • Session-aware ranking: KDE temporal encoding could complement session partitioning strategies by explicitly modeling user activity rhythms and engagement patterns
