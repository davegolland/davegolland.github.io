Nüwa: Mending the Spatial Integrity Torn by VLM Token Pruning

Source: arXiv:cs.CL Published: 2026-02-03 Rank: #9 (score: 0.91, max_sim: 0.81) Reading time: ~0 min Tags: NSA Code: https://arxiv.org/pdf/2602.02951v1

Abstract

Vision token pruning has proven to be an effective acceleration technique for the efficient Vision Language Model (VLM). However, existing pruning methods demonstrate excellent performance preservation in visual question answering (VQA) and suffer substantial degradation on visual grounding (VG) tasks. Our analysis of the VLM’s processing pipeline reveals that strategies utilizing global semantic similarity and attention scores lose the global spatial reference frame, which is derived from the interactions of tokens’ positional information. Motivated by these findings, we propose , a two-stage token pruning framework that enables efficient feature aggregation while maintaining spatial integrity. In the first stage, after the vision encoder, we apply three operations, namely separation, alignment, and aggregation, which are inspired by swarm intelligence algorithms to retain information-rich global spatial anchors. In the second stage, within the LLM, we perform text-guided pruning to retain task-relevant visual tokens. Extensive experiments demonstrate that achieves SOTA performance on multiple VQA benchmarks (from 94% to 95%) and yields substantial improvements on visual grounding tasks (from 7% to 47%).

Text-Guided Token Pruning (TGTP)

Core Idea: In the second stage (within the LLM), selectively prune vision tokens based on task-relevant guidance from the language model to retain only semantically useful visual information.

Problem It Solves: Even after preserving spatial anchors, an VLM still processes all tokens through the LLM. Text-guided pruning reduces the computational burden of LLM processing by removing vision tokens that don’t contribute to answering the specific query or task.

How It Works:

  1. Input task-specific text (question, instruction) and all spatial-anchor-preserving vision tokens into the LLM decoder
  2. Compute attention weights or relevance scores between the text input and each vision token based on LLM’s internal representations
  3. Select and retain only vision tokens with high task relevance scores
  4. Process remaining tokens through downstream LLM layers for final output generation

Where It’s Useful:

  • Visual question answering (VQA) where questions provide strong guidance on which image regions matter
  • Image captioning with specific focus instructions
  • Visual reasoning tasks where the query constrains relevant regions
  • Grounding tasks where text descriptions guide which image areas to attend to

Limitations / Caveats:

  • Text-guided approach may miss important spatial context not explicitly mentioned in the task text
  • Requires text prompt to generate guidance (may underperform on tasks without clear textual context)
  • Risk of over-pruning if relevance thresholds are too aggressive

Internal Use Cases:

  • Query-aware image retrieval where the search query guides which image regions to index
  • Contextualized visual ranking in search/recommendation where the user query determines spatial importance
  • Multi-modal information retrieval where linguistic cues determine visual region relevance
