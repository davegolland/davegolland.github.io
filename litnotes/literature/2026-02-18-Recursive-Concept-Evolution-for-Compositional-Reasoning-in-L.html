Recursive Concept Evolution for Compositional Reasoning in Large Language Models

Source: arXiv:cs.CL Published: 2026-02-17 Rank: #8 (score: 0.98, max_sim: 0.83) Reading time: ~20 min Tags: LLM, RL, Search, RAG

Abstract

Large language models achieve strong performance on many complex reasoning tasks, yet their accuracy degrades sharply on benchmarks that require compositional reasoning, including ARC-AGI-2, GPQA, MATH, BBH, and HLE. Existing methods improve reasoning by expanding token-level search through chain-of-thought prompting, self-consistency, or reinforcement learning, but they leave the model’s latent representation space fixed. When the required abstraction is not already encoded in this space, performance collapses. We propose Recursive Concept Evolution (RCE), a framework that enables pretrained language models to modify their internal representation geometry during inference. RCE introduces dynamically generated low-rank concept subspaces that are spawned when representational inadequacy is detected, selected through a minimum description length criterion, merged when synergistic, and consolidated via constrained optimization to preserve stability. This process allows the model to construct new abstractions rather than recombining existing ones. We integrate RCE with Mistral-7B and evaluate it across compositional reasoning benchmarks. RCE yields 12-18 point gains on ARC-AGI-2, 8-14 point improvements on GPQA and BBH, and consistent reductions in depth-induced error on MATH and HLE.

Recursive Concept Evolution (RCE)

Core Idea: RCE enables pretrained LLMs to dynamically construct new low-rank representational subspaces during inference when existing latent geometry is insufficient to support a required abstraction.

Problem It Solves: LLMs hit a hard ceiling on compositional reasoning tasks (ARC-AGI-2, GPQA, MATH, BBH, HLE) because their representation space is fixed at inference time — when a required abstraction isn’t already encoded, the model cannot construct it, causing sharp accuracy degradation that token-level search methods (chain-of-thought, self-consistency, RL) cannot fix.

How It Works:

  1. Detect representational inadequacy: Monitor intermediate activations during a forward pass; flag when the current latent geometry cannot faithfully encode the concept required by the current reasoning step (e.g., via high reconstruction error or entropy in the activation distribution).
  2. Spawn a low-rank concept subspace: Dynamically generate a new low-rank adapter (LoRA-style projection matrix) targeted at the detected gap; select it using a Minimum Description Length (MDL) criterion to avoid overfitting or redundancy.
  3. Merge synergistic subspaces: Identify newly spawned subspaces that are geometrically compatible with existing ones and merge them to reduce overhead and encourage generalization.
  4. Consolidate via constrained optimization: Update the merged representation using constrained optimization (e.g., projected gradient) to integrate the new abstraction while preserving stability of existing knowledge.

Architecture:

flowchart LR
    A[Input Tokens] --> B[Frozen Pretrained LLM Layers]
    B --> C{Representational\nAdequacy Check}
    C -- Adequate --> F[Continue Forward Pass]
    C -- Inadequate --> D[Spawn Low-Rank\nConcept Subspace\nMDL Selection]
    D --> E[Merge Synergistic\nSubspaces]
    E --> G[Constrained Optimization\nConsolidation]
    G --> F
    F --> H[Output / Next Layer]

Where It’s Useful:

  • Compositional and multi-step reasoning tasks that require building new abstractions from primitives (ARC-AGI-2, math olympiad problems)
  • Scientific question answering requiring cross-domain concept synthesis (GPQA)
  • Any inference-time scenario where the required concept distribution is out-of-distribution relative to pretraining

Limitations / Caveats:

  • Inference-time subspace spawning adds latency and memory overhead proportional to the number of detected inadequacies — cost is unpredictable per sample
  • MDL-based selection and constrained consolidation introduce hyperparameters that may need tuning per domain; sensitivity analysis is not yet established
  • Gains are demonstrated on Mistral-7B; generalization to larger models or different architectures is unverified
  • Dynamically modifying activations at inference time risks instability if consolidation constraints are too loose — the stability-plasticity tradeoff is not fully characterized
  • No persistent learning across inference calls — each session spawns subspaces independently, so improvements do not accumulate

Internal Use Cases:

  • LLM reranking: When a ranker LLM encounters a query type poorly covered by its training distribution, RCE could spawn subspaces to better encode novel relevance signals without retraining
  • Retrieval-augmented generation (RAG): For queries requiring compositional synthesis across multiple retrieved chunks, RCE could build transient abstractions bridging disparate concepts mid-inference
  • Cold-start feature synthesis: In ranking models using LLM-generated features, RCE-style dynamic subspace construction could handle novel item/user concept combinations not seen at train time
