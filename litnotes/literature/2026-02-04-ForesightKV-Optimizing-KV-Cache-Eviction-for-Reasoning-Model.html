ForesightKV: Optimizing KV Cache Eviction for Reasoning Models by Learning Long-Term Contribution

ForesightKV: Optimizing KV Cache Eviction for Reasoning Models by Learning Long-Term Contribution

Source: arXiv:cs.CL Published: 2026-02-03 Rank: #7 (score: 0.95, max_sim: 0.85) Reading time: ~0 min Tags: Uncategorized Code: https://arxiv.org/pdf/2602.03203v1

Abstract

Recently, large language models (LLMs) have shown remarkable reasoning abilities by producing long reasoning traces. However, as the sequence length grows, the key-value (KV) cache expands linearly, incurring significant memory and computation costs. Existing KV cache eviction methods mitigate this issue by discarding less important KV pairs, but often fail to capture complex KV dependencies, resulting in performance degradation. To better balance efficiency and performance, we introduce ForesightKV, a training-based KV cache eviction framework that learns to predict which KV pairs to evict during long-text generations. We first design the Golden Eviction algorithm, which identifies the optimal eviction KV pairs at each step using future attention scores. These traces and the scores at each step are then distilled via supervised training with a Pairwise Ranking Loss. Furthermore, we formulate cache eviction as a Markov Decision Process and apply the GRPO algorithm to mitigate the significant language modeling loss increase on low-entropy tokens. Experiments on AIME2024 and AIME2025 benchmarks of three reasoning models demonstrate that ForesightKV consistently outperforms prior methods under only half the cache budget, while benefiting synergistically from both supervised and reinforcement learning approaches.

Pairwise Ranking Loss for KV Cache Eviction

Core Idea: Trains a neural predictor to rank KV pairs by eviction priority using contrastive learning, where pairs marked for eviction are ranked lower than those to retain.

Problem It Solves: Multi-class classification (evict/keep) struggles with class imbalance in KV cache (only a few tokens should be evicted). Pairwise ranking naturally models the ordinal preference: “this pair is more evict-able than that pair.”

How It Works:

  1. Feature extraction: Encode each KV pair with contextual features (e.g., position embeddings, attention statistics, layer-wise cumulative attention scores)
  2. Ranking model: Pass features through a neural network to produce a scalar eviction score for each KV pair
  3. Pairwise loss: For each training example, construct pairs where one KV is marked for eviction (positive) and one is marked for retention (negative): where should be evicted and should be retained
  4. Training: Minimize pairwise ranking loss over all (evict, keep) pairs from Golden Eviction labels
  5. Inference: Rank all cached KV pairs by score and evict bottom- at each step

Where It’s Useful:

  • Learning to rank KV pairs under cache budget constraints
  • Any ordinal preference learning where absolute thresholds are unknown
  • Combining multiple weak signals (recency, attention pattern, position) into a unified eviction score

Limitations / Caveats:

  • Pairwise loss is quadratic in the number of KV pairs per sequence; can be expensive on very long contexts
  • Only captures relative preference, not calibrated eviction thresholds
  • Requires careful negative sampling for stable training on imbalanced datasets

Internal Use Cases:

  • Learning to rank for recommendations: train pairwise loss to rank items by relevance, similar to how KV pairs are ranked for eviction priority
  • Query ranking in sparse retrieval: learn which documents/KV-equivalents are most relevant without explicit thresholds

GRPO (Generalized Reinforcement Policy Optimization) for KV Cache Distillation

Core Idea: Applies group relative policy optimization (a reinforcement learning algorithm) to fine-tune eviction policies on language modeling loss, mitigating performance degradation on low-entropy (easy) tokens caused by supervised learning alone.

Problem It Solves: Supervised training on Golden Eviction labels optimizes for AIME reasoning benchmarks (high-entropy, complex reasoning steps) but causes performance collapse on low-entropy tokens (common words, templates). GRPO allows online adaptation to improve overall language modeling perplexity while respecting the cache budget.

How It Works:

  1. Policy initialization: Start with the supervised-trained eviction ranker from Pairwise Ranking Loss
  2. Rollout: Generate sequences using the current eviction policy, receiving reward signal (e.g., log probability of generated token, or task-specific accuracy)

  5. Repeat: Alternate between supervised loss (on AIME) and RL loss (on LM objectives) until convergence

Where It’s Useful:

  • Balancing multiple objectives in inference optimization (cache efficiency vs. generation quality)
  • Online adaptation during deployment when supervised labels don’t capture all failure modes
  • Fine-tuning learned policies to handle distribution shift in token difficulty

Limitations / Caveats:

  • RL training is sample-inefficient; requires many forward passes to converge
  • Reward signal design is critical; poorly chosen rewards can degrade performance
  • Risk of policy collapse to local optima or reward hacking if objective is misaligned
  • Computational overhead during training (multiple generations per batch)

Internal Use Cases:

  • Multi-objective optimization in ranking: use GRPO-like policy gradient to balance click-through rate (supervised from data) with long-term user engagement (RL signal)
  • Cache eviction for recommendation inference: supervised on relevance (pairwise loss) + RL on latency constraints
