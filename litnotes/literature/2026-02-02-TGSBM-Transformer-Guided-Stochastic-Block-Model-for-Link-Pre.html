TGSBM: Transformer-Guided Stochastic Block Model for Link Prediction

Source: arXiv:cs.IR Published: 2026-01-28 Rank: #10 (score: 0.98, max_sim: 0.81) Reading time: ~20 min Tags: Search, Transformer, LLM, RecSys, Graph Code: https://arxiv.org/pdf/2601.20646v1

Abstract

Link prediction is a cornerstone of the Web ecosystem, powering applications from recommendation and search to knowledge graph completion and collaboration forecasting. However, large-scale networks present unique challenges: they contain hundreds of thousands of nodes and edges with heterogeneous and overlapping community structures that evolve over time. Existing approaches face notable limitations: traditional graph neural networks struggle to capture global structural dependencies, while recent graph transformers achieve strong performance but incur quadratic complexity and lack interpretable latent structure. We propose \textbf{TGSBM} (Transformer-Guided Stochastic Block Model), a framework that integrates the principled generative structure of Overlapping Stochastic Block Models with the representational power of sparse Graph Transformers. TGSBM comprises three main components: (i) \emph{expander-augmented sparse attention} that enables near-linear complexity and efficient global mixing, (ii) a \emph{neural variational encoder} that infers structured posteriors over community memberships and strengths, and (iii) a \emph{neural edge decoder} that reconstructs links via OSBM’s generative process, preserving interpretability. Experiments across diverse benchmarks demonstrate competitive performance (mean rank 1.6 under HeaRT protocol), superior scalability (up to faster training), and interpretable community structures. These results position TGSBM as a practical approach that strikes a balance between accuracy, efficiency, and transparency for large-scale link prediction.

Key Techniques

Neural Variational Encoder for Community Membership Inference

Core Idea: Use a neural encoder with variational inference to learn a posterior distribution over overlapping community memberships and membership strengths in the network, conditioned on the observed graph structure.

Problem It Solves: Overlapping Stochastic Block Models (OSBMs) provide interpretable latent structure but typically use expectation-maximization for inference, which is slow and doesn’t leverage deep representations. Transformers are powerful but produce opaque embeddings without explicit latent structure. This component bridges the gap by inferring structured, interpretable community assignments from transformer representations.

How It Works:

  1. Encode observed edges and node features using the sparse graph transformer to produce contextual node embeddings: z = Transformer(Graph)
  2. Feed embeddings through a neural encoder network q_φ(·) parameterized by φ to produce variational parameters (e.g., means and variances)
  3. Encode each node’s community membership as a categorical latent variable: c_i ~ Cat(π_i) where π_i = softmax(μ_i), and membership strength as a continuous variable: s_i ~ N(μ_s, σ_s)
  4. Apply reparameterization trick to enable backpropagation through stochastic sampling
  5. Optimize encoder using variational lower bound (ELBO) with KL divergence between variational posterior q_φ and prior p(c, s)
  6. Output: posterior distributions q_φ(c, s | Graph) that assign each node to one or more communities with learned strength values

Architecture:

flowchart LR
    A["Observed Graph"] --> B["Sparse Graph<br/>Transformer"]
    B --> C["Node Embeddings"]
    C --> D["Neural Encoder<br/>q_φ"]
    D --> E["Variational Parameters<br/>μ, σ"]
    E --> F["Reparameterization"]
    F --> G["Community Samples<br/>c, s"]

Where It’s Useful:

  • Community detection with interpretability constraints
  • Hierarchical clustering in large networks
  • Soft membership modeling for overlapping communities
  • Generative modeling where explicit latent factors matter (vs. pure discrimination)

Limitations / Caveats:

  • Variational inference introduces additional hyperparameters (KL weight, prior specification)
  • Reparameterization works well for Gaussian posteriors but less natural for discrete communities; categorical reparameterization via Gumbel-Softmax adds approximation error
  • Inference cost is higher than point-estimate alternatives; trades off computational cost for uncertainty quantification
  • Posterior quality depends heavily on encoder architecture and KL annealing schedule

Internal Use Cases:

  • RecSys: Learning soft user segments from interaction graphs for more interpretable recommender systems
  • Ranking: Inferring query-intent communities to improve diversity and explainability in ranker outputs
  • LLM: Community structure in knowledge graphs for more interpretable retrieval-augmented generation

Neural Edge Decoder via OSBM Generative Process

Core Idea: Reconstruct links by explicitly modeling the Overlapping Stochastic Block Model’s generative process in a neural decoder, ensuring that predictions respect interpretable community structure and improving generalization through principled probabilistic modeling.

Problem It Solves: Standard neural decoders (e.g., MLP on concatenated embeddings) are opaque black boxes that don’t enforce community-structure-aware link formation. They lack inductive bias about how real networks are formed (through community interactions) and can produce predictions that violate the modular structure of real graphs. The OSBM generative process provides this structure but is traditionally not end-to-end differentiable; this component makes it so.

How It Works:

  1. Receive inferred community memberships and strengths from the variational encoder: c_i, s_i for each node i
  2. For each potential edge (i, j), compute the probability that the edge exists via the OSBM generative likelihood:
    • P(e_{ij} = 1 | c_i, c_j, s_i, s_j, θ) = 1 - ∏_{k,ℓ} (1 - θ_{kℓ} · s_i^k · s_j^ℓ)
    • where θ_{kℓ} is the block-to-block edge probability learned during training, and c_i, s_i encode which communities node i belongs to and with what strength
  3. Parameterize θ and other block-level parameters via a neural network decoder that learns from the data
  4. Optimize the decoder using link-prediction loss (binary cross-entropy or ranking loss) while maintaining differentiability through the generative model
  5. Output: edge predictions that decompose into contributions from each community pair, preserving interpretability

Architecture:

flowchart LR
    A["Community Samples<br/>c, s"] --> B["Block Parameters<br/>Neural Network"]
    B --> C["Block-wise Affinities<br/>θ_kℓ"]
    A --> D["OSBM Likelihood"]
    C --> D
    D --> E["Edge Probability<br/>P(e_ij=1)"]
    E --> F["Link Prediction"]

Where It’s Useful:

  • Link prediction in modular networks (social networks, collaboration graphs)
  • Knowledge graph completion where interpretability is required (biomedical, financial networks)
  • Temporal link prediction where community evolution needs to be tracked
  • Recommendation as implicit link prediction (user-item bipartite graphs)

Limitations / Caveats:

  • OSBM assumes assortative mixing and may underfit disassortative networks (e.g., bipartite graphs with opposite-community connections)
  • Computational cost of evaluating the generative likelihood scales with number of communities; becomes expensive for K >> 10 communities
  • Marginalizing over latent community assignments (if not directly observed) requires integration; full marginalization is intractable; the variational approximation introduces additional error
  • Requires careful calibration of the community inference step; poor community assignments propagate directly to decoder predictions

Internal Use Cases:

  • RecSys: Predicting user-item links while respecting inferred user segments and item categories for better diversity and trustworthiness
  • Ranking: Learning community-aware ranking models where diversity and topic coherence are goals
  • LLM Knowledge Graphs: Inferring missing facts in knowledge graphs while maintaining structural consistency and interpretability
