OSCAR: Optimization-Steered Agentic Planning for Composed Image Retrieval

Source: arXiv:cs.AI Published: 2026-02-09 Rank: #2 (score: 0.99, max_sim: 0.82) Reading time: ~20 min Tags: Search, Embeddings, RAG

Abstract

Composed image retrieval (CIR) requires complex reasoning over heterogeneous visual and textual constraints. Existing approaches largely fall into two paradigms: unified embedding retrieval, which suffers from single-model myopia, and heuristic agentic retrieval, which is limited by suboptimal, trial-and-error orchestration. To this end, we propose OSCAR, an optimization-steered agentic planning framework for composed image retrieval. We are the first to reformulate agentic CIR from a heuristic search process into a principled trajectory optimization problem. Instead of relying on heuristic trial-and-error exploration, OSCAR employs a novel offline-online paradigm. In the offline phase, we model CIR via atomic retrieval selection and composition as a two-stage mixed-integer programming problem, mathematically deriving optimal trajectories that maximize ground-truth coverage for training samples via rigorous boolean set operations. These trajectories are then stored in a golden library to serve as in-context demonstrations for online steering of VLM planner at online inference time. Extensive experiments on three public benchmarks and a private industrial benchmark show that OSCAR consistently outperforms SOTA baselines. Notably, it achieves superior performance using only 10% of training data, demonstrating strong generalization of planning logic rather than dataset-specific memorization.

Key Techniques

  • Offline-online trajectory optimization
  • Mixed-integer programming for retrieval composition
  • VLM-steered agentic planning with in-context demonstrations

OSCAR (Optimization-Steered Agentic Planning for Composed Image Retrieval)

Core Idea: Reformulates agentic composed image retrieval from heuristic trial-and-error into a principled trajectory optimization problem, using offline mixed-integer programming to derive optimal retrieval trajectories that are stored as in-context demonstrations for online VLM planner steering.

Problem It Solves: Addresses the limitations of both unified embedding retrieval (single-model myopia) and heuristic agentic retrieval (suboptimal trial-and-error orchestration) by replacing ad-hoc exploration with mathematically optimal planning trajectories.

How It Works:

  1. Offline Trajectory Optimization: Model composed image retrieval as a two-stage mixed-integer programming (MIP) problem over atomic retrieval operations (selection and composition), computing optimal trajectories that maximize ground-truth coverage using rigorous boolean set operations on training samples.
  2. Golden Library Construction: Store the derived optimal trajectories in a golden library indexed by query characteristics and constraint patterns.
  3. Online VLM Planning: At inference time, retrieve relevant optimal trajectories from the golden library and use them as in-context demonstrations to steer a Vision-Language Model planner toward optimal atomic retrieval selection and composition.
  4. Execution: Execute the planned atomic retrieval operations (visual search, text filtering, attribute matching, etc.) in the derived optimal order to retrieve final results.

Architecture:

flowchart LR
    A[Training Data] --> B[MIP Optimizer]
    B --> C[Golden Library<br/>Optimal Trajectories]
    D[Query at Inference] --> E[Trajectory Retrieval]
    C --> E
    E --> F[VLM Planner<br/>w/ In-Context Demo]
    F --> G[Atomic Retrieval<br/>Execution]
    G --> H[Retrieved Images]

Where It’s Useful:

  • Composed image retrieval tasks requiring multi-constraint reasoning (visual + textual modifications)
  • E-commerce visual search with complex user refinements (“find shoes similar to this image but in red leather”)
  • Fashion and product discovery platforms with attribute-based filtering on visual similarity
  • Any retrieval scenario where atomic operations (filters, embeddings, attributes) need optimal orchestration

Limitations / Caveats:

  • Offline MIP optimization requires ground-truth labels for training samples to compute optimal trajectories
  • Computational cost of solving mixed-integer programs scales with number of atomic operations and training set size
  • Assumes atomic retrieval operations are predefined and fixed (limited flexibility for novel operation types at inference)
  • Golden library quality depends on coverage and diversity of training query patterns

Internal Use Cases:

  • Visual Product Search: Apply to multi-modal product retrieval where users provide reference images plus textual refinements (color, brand, price range), optimizing the order of visual embedding search, attribute filters, and collaborative signals
  • Recommendation Ranking: Use trajectory optimization to orchestrate heterogeneous ranking signals (content-based, collaborative, contextual) rather than fixed ensemble weights—learn optimal signal composition paths per user segment
  • LLM Tool-Use Planning: Extend to general LLM agent planning by replacing heuristic ReAct/chain-of-thought with MIP-derived optimal tool invocation sequences, storing them as few-shot planning demonstrations for complex multi-step tasks
