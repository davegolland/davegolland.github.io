Abstract

Post-Training Quantization (PTQ) is essential for deploying Large Language Models (LLMs) on memory-constrained devices, yet it renders models static and difficult to fine-tune. Standard fine-tuning paradigms, including Reinforcement Learning (RL), fundamentally rely on backpropagation and high-precision weights to compute gradients. Thus they cannot be used on quantized models, where the parameter space is discrete and non-differentiable. While Evolution Strategies (ES) offer a backpropagation-free alternative, optimization of the quantized parameters can still fail due to vanishing or inaccurate gradient. This paper introduces Quantized Evolution Strategies (QES), an optimization paradigm that performs full-parameter fine-tuning directly in the quantized space. QES is based on two innovations: (1) it integrates accumulated error feedback to preserve high-precision gradient signals, and (2) it utilizes a stateless seed replay to reduce memory usage to low-precision inference levels. QES significantly outperforms the state-of-the-art zeroth-order fine-tuning method on arithmetic reasoning tasks, making direct fine-tuning for quantized models possible. It therefore opens up the possibility for scaling up LLMs entirely in the quantized space. The source code is available at https://github.com/dibbla/Quantized-Evolution-Strategies .

Quantized Evolution Strategies (QES)

Core Idea: A backpropagation-free optimization method that performs full-parameter fine-tuning directly on quantized LLMs by combining accumulated error feedback (to preserve high-precision gradient signals in discrete parameter space) with stateless seed replay (to maintain memory at low-precision inference levels).

Problem It Solves: Standard fine-tuning requires backpropagation through high-precision weights, making it incompatible with Post-Training Quantized (PTQ) models where parameters are discrete and non-differentiable. Naive zeroth-order methods like vanilla Evolution Strategies fail on quantized parameters due to vanishing or inaccurate gradients when the search space is discrete.

How It Works:

  1. Perturbation Generation: Sample parameter perturbations using a seed-based random number generator (enables stateless replay without storing perturbation vectors)
  2. Forward Evaluation: Evaluate fitness/loss of perturbed quantized models on task examples (e.g., arithmetic reasoning)
  3. Gradient Estimation: Compute zeroth-order gradient estimate from fitness differences across perturbations
  4. Error-Feedback Update: Apply gradient to a high-precision shadow accumulator, then quantize the result back to discrete weights—carrying forward quantization residuals to preserve update signal across iterations
  5. Seed Replay: Reconstruct perturbations on-demand using stored seeds rather than caching full parameter tensors, keeping memory at inference levels

Architecture:

flowchart LR
    A[Quantized Model θ_q] --> B[Seed-based Perturbation]
    B --> C[Forward Pass + Fitness]
    C --> D[ES Gradient Estimate ∇f]
    D --> E[High-Precision Accumulator θ_hp]
    E --> F[Quantize with Error Feedback]
    F --> A
    G[Seed Storage] -.stateless replay.-> B

Where It’s Useful:

  • Fine-tuning quantized LLMs on edge/mobile devices with limited memory
  • Domain adaptation or RLHF-style reward optimization when backpropagation is infeasible (discrete parameters, non-differentiable objectives)
  • Low-resource scenarios requiring full-parameter updates without gradient checkpointing or optimizer state storage

Limitations / Caveats:

  • Requires many forward passes (high sample complexity) compared to gradient-based methods—compute cost shifts from backward passes to repeated forward evaluations
  • Convergence speed may lag behind full-precision Adam/SGD on tasks with smooth loss landscapes
  • Effectiveness depends on task structure; demonstrated primarily on arithmetic reasoning, may require tuning for open-ended generation or multimodal tasks

Internal Use Cases:

  • Deploy fine-tunable recommendation/ranking LLMs on memory-constrained inference servers (e.g., INT4/INT8 models that adapt to user feedback without full-precision retraining)
  • Continual learning for quantized chat models using online RL signals (thumbs up/down) where gradient computation is blocked by quantization
  • Federated learning with quantized edge models—QES enables local parameter updates without transmitting high-precision weights or gradients
