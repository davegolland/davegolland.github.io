Abstract

Post-training compression of Transformer models commonly relies on truncated singular value decomposition (SVD). However, enforcing a single shared subspace can degrade accuracy even at moderate compression. Sparse dictionary learning provides a more flexible union-of-subspaces representation, but existing approaches often suffer from iterative dictionary and coefficient updates. We propose COMPOT (Calibration-Optimized Matrix Procrustes Orthogonalization for Transformers), a training-free compression framework that uses a small calibration dataset to estimate a sparse weight factorization. COMPOT employs orthogonal dictionaries that enable closed-form Procrustes updates for the dictionary and analytical single-step sparse coding for the coefficients, eliminating iterative optimization. To handle heterogeneous layer sensitivity under a global compression budget, COMPOT further introduces a one-shot dynamic allocation strategy that adaptively redistributes layer-wise compression rates. Extensive experiments across diverse architectures and tasks show that COMPOT consistently delivers a superior quality-compression trade-off over strong low-rank and sparse baselines, while remaining fully compatible with post-training quantization for extreme compression. Code is available https://github.com/mts-ai/COMPOT{here}.

Key Techniques

  • Sparse dictionary learning with orthogonal dictionaries
  • Procrustes optimization for dictionary updates
  • analytical single-step sparse coding
  • dynamic layer-wise compression rate allocation

COMPOT (Calibration-Optimized Matrix Procrustes Orthogonalization for Transformers)

Core Idea: A training-free Transformer compression framework that factorizes weight matrices into sparse combinations of orthogonal dictionary atoms, using closed-form Procrustes updates and analytical sparse coding to eliminate iterative optimization.

Problem It Solves: Standard SVD-based low-rank compression forces a single shared subspace, which degrades accuracy at moderate compression rates. Sparse dictionary learning offers richer union-of-subspaces representations but traditionally requires slow alternating optimization between dictionary and coefficient updates.

How It Works:

  1. Calibration-guided estimation: Pass a small calibration dataset through the model to collect layer-wise activation statistics, which inform how each weight matrix should be factorized.
  2. Orthogonal dictionary construction: Decompose each weight matrix W ≈ D · A where D is an orthogonal dictionary (columns are orthonormal atoms); orthogonality enables a closed-form Procrustes solution for D — a single SVD step rather than iterative gradient updates.
  3. Analytical sparse coding: Given the fixed orthogonal D, compute sparse coefficients A in a single step via orthogonal matching pursuit or a closed-form thresholding operation (orthogonality decouples atom contributions, making coefficient estimation non-iterative).
  4. One-shot dynamic budget allocation: Estimate per-layer sensitivity (e.g., via reconstruction error or activation magnitude) and redistribute the global sparsity/rank budget across layers in one pass, assigning higher fidelity to sensitive layers without iterative search.

Architecture:

flowchart LR
    W[Weight Matrix W] --> CAL[Calibration\nActivations]
    CAL --> SENS[Layer Sensitivity\nEstimation]
    SENS --> ALLOC[Dynamic Budget\nAllocation]
    ALLOC --> PROC[Procrustes Update\nfor Dictionary D\nclosed-form SVD]
    PROC --> SC[Sparse Coding\nfor Coefficients A\nanalytical step]
    SC --> OUT[Compressed W ≈ D·A]
    OUT --> PTQ[Optional: Post-training\nQuantization]

Where It’s Useful:

  • Compressing large Transformer LLMs for inference on memory-constrained hardware
  • Scenarios requiring extreme compression via stacking with post-training quantization (PTQ)
  • Heterogeneous architectures where layer sensitivity varies significantly (vision + language models)

Limitations / Caveats:

  • Requires a calibration dataset; quality of compression depends on how representative it is of the target distribution
  • Orthogonal dictionary assumption may not be optimal for all weight geometries — non-orthogonal dictionaries can be more expressive
  • One-shot allocation heuristic may not match the globally optimal per-layer budget found by exhaustive search

Internal Use Cases:

  • Compressing retrieval or ranking LLMs (e.g., cross-encoders) for low-latency inference without fine-tuning on proprietary data
  • Reducing KV-cache footprint in attention layers via sparse factorization of projection matrices
  • Stacking with quantization for extreme model compression in embedding models used in two-tower RecSys architectures
