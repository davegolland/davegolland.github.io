RecSys Substack Digest: 10 Papers on RAG, Retrieval, and RecSys

This digest covers 10 recent papers. I’ve synthesized the most relevant ones below with brief notes on the others.

Highly Relevant (Full Notes Available)

1. Agentic-R: Learning to Retrieve for Agentic Search

→ See separate note: 2026-01-25-Agentic-R-Learning-to-Retrieve

TL;DR: Train retrievers for multi-turn agentic search by optimizing for final answer correctness, not just passage relevance. Bidirectional co-optimization of retriever + agent.

Relevant (Summarized)

2. Deep GraphRAG (Ant Group)

📚 https://arxiv.org/abs/2601.11144

Problem: Global graph search captures broad context but is slow; local search is fast but misses cross-community relationships.

Solution: Three-stage hierarchical retrieval:

  1. Inter-community filtering
  2. Community-level refinement
  3. Entity-level search

Uses beam search (k=3) over a weighted Louvain clustering hierarchy.

Key technique: DW-GRPO (Dynamic Weighting Reward GRPO) — RL approach that adaptively rebalances reward weights for relevance, faithfulness, and conciseness during training. Counteracts the “seesaw effect” where models over-optimize easy metrics.

Result: 1.5B model achieves 94% of 72B model’s performance on Natural Questions.

Relevance: Graph-based RAG for knowledge-intensive domains. Connects to knowledge graphs, RAG.

3. PruneRAG: Confidence-Guided Query Decomposition Trees

📚 https://arxiv.org/abs/2601.11024 👨‍💻 https://github.com/Fdioa/PruneRAG

Problem: Multi-hop RAG suffers from “evidence forgetting” (retrieved docs not used) and inefficiency from uncontrolled query expansion.

Solution: Structured query decomposition tree with:

  • Adaptive node expansion — Decides: answer directly, decompose into sub-queries, or abstract into entities
  • Confidence-guided pruning — Token-level log probability scoring, only accepts answers above threshold (τ=0.95)
  • Fine-grained retrieval — Extracts key entities as retrieval anchors

New metric: Evidence Forgetting Rate (EFR) — proportion of cases where all golden docs were retrieved but answer still wrong.

Result: Highest F1 on HotpotQA/2WikiMultihop/MusiQue, 4.9× faster than RAG-Star and Search-o1.

Relevance: Efficient multi-hop reasoning. Connects to RAG, hallucination (confidence thresholds).

4. Rank4Gen: Generator-Aware Document Ranking

📚 https://arxiv.org/abs/2601.11273 👨‍💻 https://github.com/JOHNNY-fans/Rank4Gen

Problem: Traditional rankers optimize for query-document relevance, but relevance ≠ generation quality. What helps Qwen3-8B may not help Llama-3.1-8B.

Solution: Train ranker conditioned on generator-specific information:

  • Generator unique IDs
  • Textual descriptions of each generator’s characteristics
  • Optimize for downstream response quality, not relevance scores

Key insight: Optimal document configurations vary by generator. A single ranker can learn generator-specific preferences.

Connects to ranking systems, RAG.

5. RT-RAG: Tree-Structured Reasoning for Multi-Hop QA

📚 https://arxiv.org/abs/2601.11255 👨‍💻 https://github.com/sakura20221/RT-RAG

Solution: Decompose questions into explicit tree structures via entity analysis. Traverse bottom-up (resolve leaf nodes before parents). Includes rejection sampling, query rewriting for failed retrievals, adaptive restructuring.

Relevance: Structured decomposition for complex questions. Connects to RAG, multi-hop reasoning.

RecSys-Focused (Brief Notes)

6. CoNRec: Negative Feedback Modeling (Alibaba)

📚 https://arxiv.org/abs/2601.15721

Models what users dislike rather than just what they like. Uses hierarchical semantic IDs via RQ-VAE and Progressive GRPO training. 24.9% improvement on long-tail items.

Relevance: Negative signal modeling for recommenders. Connects to recommender systems.

7. HyFormer: Long-Sequence CTR Prediction (ByteDance)

📚 https://arxiv.org/abs/2601.12681

Hybrid transformer for industrial CTR. Unifies long sequence modeling + feature interaction via Global Tokens. Better than Full Transformer + RankMixer with fewer FLOPs.

Relevance: Industrial-scale CTR.

8. Sparse Autoencoders for Collaborative Filtering

📚 https://arxiv.org/abs/2601.11182

Insert SAEs into CFAE to get interpretable, steerable recommendations. Individual neurons correspond to concepts (“Quentin Tarantino”, “love story”). Can boost/suppress neurons to steer recommendations.

Relevance: Interpretability + controllability in RecSys. Connects to recommender systems, interpretability.

9. PopSteer: Steering Popularity Bias (TU Delft)

📚 https://arxiv.org/abs/2601.15122

Uses SAEs to identify and suppress popularity-aligned neurons in recommender embeddings. Achieves fairness improvements while maintaining accuracy.

Relevance: Bias mitigation in RecSys. Connects to recommender systems, fairness.

10. Predicting RAG Utility and Answer Quality (Glasgow)

📚 https://arxiv.org/abs/2601.14546

Formalizes Retrieval Performance Prediction (RPP) and Generation Performance Prediction (GPP). Combines QPP methods + perplexity-based measures + document quality metrics.

Relevance: Meta-prediction for RAG systems. Connects to RAG, evaluation.

Cross-Cutting Themes

  1. Multi-hop reasoning is the frontier — single-turn RAG is increasingly solved
  2. Confidence/quality signals are critical for pruning and reliability
  3. Generator-aware retrieval — what the model can use matters more than relevance
  4. Graph structure helps for knowledge-intensive domains
  5. Sparse autoencoders emerging as interpretability tool for embeddings/recommendations
