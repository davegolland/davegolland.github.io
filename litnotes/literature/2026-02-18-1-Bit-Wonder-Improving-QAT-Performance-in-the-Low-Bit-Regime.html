1-Bit Wonder: Improving QAT Performance in the Low-Bit Regime through K-Means Quantization

1-Bit Wonder: Improving QAT Performance in the Low-Bit Regime through K-Means Quantization

Source: arXiv:cs.LG Published: 2026-02-17 Rank: #4 (score: 0.62, max_sim: 0.79) Reading time: ~0 min Tags: Uncategorized

Abstract

Quantization-aware training (QAT) is an effective method to drastically reduce the memory footprint of LLMs while keeping performance degradation at an acceptable level. However, the optimal choice of quantization format and bit-width presents a challenge in practice. The full design space of quantization is not fully explored in the context of QAT, and the precise trade-off between quantization and downstream performance is poorly understood, as comparisons often rely solely on perplexity-based evaluations. In this work, we address these shortcomings with an empirical study of QAT in the low-bit regime. We show that k-means based weight quantization outperforms integer formats and can be implemented efficiently on standard hardware. Furthermore, we find that, under a fixed inference memory budget, the best performance on generative downstream tasks is achieved with -bit quantized weights.

K-Means Quantization-Aware Training (KM-QAT)

Core Idea: Replace uniform integer quantization grids with k-means-learned codebooks during quantization-aware training, then show that 1-bit k-means quantization maximizes downstream generative task performance under a fixed inference memory budget.

Problem It Solves: Standard QAT uses integer (uniform) quantization formats whose fixed grid spacing is suboptimal for the non-uniform distributions of neural network weights; additionally, the QAT design space (format × bit-width) is underexplored, and evaluations rely on perplexity rather than generative task performance, obscuring the true quality/memory trade-off.

How It Works:

  1. Codebook initialization: Run k-means clustering on the weight tensor of each layer to find 2^b centroids (e.g., 2 centroids for 1-bit), producing a learned, non-uniform quantization grid that matches the actual weight distribution.
  2. Straight-through gradient estimation: During the forward pass, each weight is snapped to its nearest centroid; gradients are passed through unchanged (straight-through estimator) so the full-precision “latent” weights continue to be updated via backprop.
  3. Codebook update: Centroid values are periodically re-estimated (via k-means on the current latent weights) or jointly optimized, so the grid adapts as training progresses.
  4. Bit-width sweep under fixed memory budget: Models are trained at multiple bit-widths (1-bit, 2-bit, etc.); given a fixed inference VRAM ceiling, lower bit-widths allow larger models — the paper finds 1-bit quantized weights on a larger model outperform higher-bit quantized smaller models on generative benchmarks.

Architecture:

flowchart LR
    W[Full-precision weights] --> KM[K-Means clustering\n2^b centroids]
    KM --> CB[Codebook]
    CB --> Q[Quantized weights\nnearest centroid lookup]
    Q --> FWD[Forward pass / loss]
    FWD --> STE[Straight-through\ngradient]
    STE --> W
    CB -- periodic re-fit --> KM

Where It’s Useful:

  • Deploying large language models on memory-constrained hardware (edge devices, consumer GPUs) where integer quantization leaves accuracy on the table
  • Any scenario with a hard inference-memory budget where choosing the right model-size/bit-width Pareto point is non-obvious
  • Post-training quantization initialization: k-means codebooks can seed PTQ pipelines before fine-tuning

Limitations / Caveats:

  • Codebook lookup requires scatter/gather ops; although the paper claims efficient implementation on standard hardware, latency overhead versus integer formats needs careful kernel-level optimization
  • Centroid re-fitting adds training complexity and a hyperparameter (re-fit frequency) not present in integer QAT
  • 1-bit k-means is essentially a learned ternary/binary scheme — extreme compression may still hurt tasks requiring precise numerical reasoning even if generative benchmarks improve

Internal Use Cases:

  • LLM serving for ranking/retrieval: Compressing a reranker or query-understanding LLM to 1-bit k-means weights could fit a much larger model into the same GPU memory, improving ranking quality at the same inference cost
  • Embedding models at scale: Apply KM-QAT to embedding towers in two-tower RecSys architectures to reduce memory and bandwidth during candidate retrieval without reverting to perplexity-only validation — use downstream retrieval metrics (Recall@K, NDCG) as the evaluation signal, mirroring the paper’s shift to generative task benchmarks
  • Multi-task LLM fine-tuning: When fine-tuning a shared LLM backbone for multiple ranking signals, k-means QAT enables fitting larger backbones under GPU constraints that would otherwise force a smaller base model
