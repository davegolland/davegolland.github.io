ARGORA: Orchestrated Argumentation for Causally Grounded LLM Reasoning and Decision Making

Source: arXiv:cs.AI Published: 2026-01-29 Rank: #8 (score: 0.99, max_sim: 0.83) Reading time: ~15 min Tags: LLM, RecSys

Abstract

Existing multi-expert LLM systems gather diverse perspectives but combine them through simple aggregation, obscuring which arguments drove the final decision. We introduce ARGORA, a framework that organizes multi-expert discussions into explicit argumentation graphs showing which arguments support or attack each other. By casting these graphs as causal models, ARGORA can systematically remove individual arguments and recompute outcomes, identifying which reasoning chains were necessary and whether decisions would change under targeted modifications. We further introduce a correction mechanism that aligns internal reasoning with external judgments when they disagree. Across diverse benchmarks and an open-ended use case, ARGORA achieves competitive accuracy and demonstrates corrective behavior: when experts initially disagree, the framework resolves disputes toward correct answers more often than it introduces new errors, while providing causal diagnostics of decisive arguments.

Key Techniques

  • Argumentation graphs
  • Causal intervention analysis

Causal Intervention on Argument Graphs

Core Idea: Treat the argumentation graph as a causal model and systematically ablate individual arguments to identify which reasoning chains are necessary for the final decision and how outcomes change under targeted argument removal.

Problem It Solves: Simple aggregation hides argument importance. ARGORA enables counterfactual reasoning: “What would the system decide if we remove argument X?” This reveals which arguments are load-bearing vs. redundant.

How It Works:

  1. Start with complete argumentation graph G and initial decision D_original
  2. For each argument arg_i ∈ V:
    • Remove arg_i and all edges incident to it: G’ = G \ {arg_i}
    • Recompute final decision D_i using remaining arguments and their relationships in G’
    • Compute importance score: Δ_i = distance(D_original, D_i) or binary indicator I_i = [D_original ≠ D_i]
  3. Rank arguments by importance; arguments with high Δ_i are “load-bearing”
  4. Optionally perform higher-order ablations: remove pairs/triples of arguments to detect synergistic effects

Where It’s Useful:

  • Identifying brittle decision paths that rely on few arguments
  • Sensitivity analysis: which arguments would need to change for decision reversal?
  • Quality assurance: detecting if system relies on weak or redundant reasoning
  • Interactive debugging with humans: “If we challenge argument X, does the decision hold?”

Limitations / Caveats:

  • Computational cost is O(|V|) full recomputations—expensive for large graphs
  • Assumes argument removal is a valid counterfactual (may not reflect real dependencies)
  • Binary decisions make small impact hard to distinguish; continuous confidence scores more informative
  • Interaction effects between arguments may be missed with pairwise ablations

Internal Use Cases:

  • Ranking: remove features/signals and measure ranking quality drop to identify essential ranking factors
  • RecSys: determine if recommendation relies on single collaborative-filtering signal or has diverse justification

Expert Dispute Resolution with Correction Mechanism

Core Idea: When multiple experts disagree on an outcome, resolve disputes by identifying which expert’s reasoning aligns better with external ground truth, and use this alignment to correct the system toward correct answers.

Problem It Solves: Multi-expert systems often produce conflicting recommendations. ARGORA doesn’t just average disagreements—it learns to resolve them by comparing internal reasoning quality against external labels, correcting systematic biases.

How It Works:

  1. Identify cases where experts E₁, E₂, …, Eₖ produce different outputs for the same query
  2. For each expert’s argumentation graph G_i, extract quality metrics:
    • Argument coherence: how well do arguments logically connect?
    • Evidence strength: do cited sources/reasoning have high credibility?
    • Consistency: does expert contradict itself across similar queries?
  3. Obtain external signal (ground truth label, human judgment, test set) for the disputed query
  4. Compute expert alignment score: A_i = similarity(expert_i’s reasoning, ground truth’s reasoning)
  5. Apply correction: weight expert i’s output by A_i; select or interpolate output from highest-alignment expert(s)
  6. Update expert weights or argumentation graph structure for future similar queries

Where It’s Useful:

  • Multi-annotator disagreement resolution (NLP labeling, crowdsourced judgments)
  • Ensemble learning where some models are more reliable on specific query types
  • Iterative system improvement: use expert alignment to auto-correct biases
  • Medical/legal domains: resolve conflicting expert opinions toward proven best practices

Limitations / Caveats:

  • Requires ground truth to compute alignment—not always available or unambiguous
  • Alignment metric is task-dependent; no single metric works across all domains
  • Risk of overfitting to training-set ground truth; may not generalize to new dispute patterns
  • Doesn’t handle cases where all experts are wrong

Internal Use Cases:

  • Ranking: when different retrieval strategies (sparse vs. dense vs. hybrid) disagree, learn which is most reliable for each query class
  • RecSys: when collaborative and content-based approaches conflict, weight by historical performance on similar user segments

Causal Reasoning Graph Compilation

Core Idea: Translate the argumentation graph into an executable causal graph that models how argument truth-values propagate through support/attack relationships to produce a final decision outcome.

Problem It Solves: Argumentation graphs alone are structural; they don’t model how arguments combine. ARGORA formalizes argument propagation as a causal inference problem, enabling quantitative diagnostics of argument chains.

How It Works:

  1. Map argumentation graph edges to causal relationships:
    • Support edge (arg_i → arg_j) becomes: truth(arg_j) ∝ truth(arg_i) + context
    • Attack edge (arg_i ⇢ arg_j) becomes: truth(arg_j) ∝ -truth(arg_i) + context
  2. Assign truth values to leaf arguments (base claims) using LLM confidence scores
  3. Propagate truth values upward through the graph using rules:
    • AND-gate (multiple supports): truth(arg_j) = min(truth(arg_i) for supporting i)
    • OR-gate (alternative supports): truth(arg_j) = max(truth(arg_i) for supporting i)
    • Weighted aggregation: truth(arg_j) = Σ_i w_i × truth(arg_i) where w_i are learned or heuristic weights
  4. Compute final decision from root argument(s): Decision = apply_threshold(truth(root), τ)
  5. Decompose decision confidence into per-argument contributions via sensitivity analysis

Architecture:

flowchart TD
    A["Argumentation\nGraph"] --> B["Causal\nCompilation"]
    B --> C["Leaf Argument\nTruth Values"]
    C --> D["Upward\nPropagation"]
    D --> E["Root Truth\nValue"]
    E --> F["Final Decision\n+ Confidence"]
    D --> G["Per-Argument\nContributions"]

Where It’s Useful:

  • Structured prediction with transparent decision logic
  • Quality estimation: confidence in final decision propagates from argument certainties
  • Adversarial robustness: identify which low-confidence arguments could flip decisions
  • Counterfactual reasoning: trace impact of changing single argument’s truth value on final output

Limitations / Caveats:

  • Propagation rules (AND/OR/weighted aggregation) must be designed per-task; no universal rule
  • Assumes independence between argument truth values—real arguments often interact
  • Circular dependencies in argumentation graphs complicate propagation (requires acyclic compilation)
  • Manual weight assignment in aggregation reduces automation; learned weights need training data

Internal Use Cases:

  • Ranking system interpretability: compile ranking signals into causal graph, propagate to final score
  • RecSys reasoning: trace how user preferences, item features, and collaborative signals combine to justify a recommendation; identify which signals dominate
