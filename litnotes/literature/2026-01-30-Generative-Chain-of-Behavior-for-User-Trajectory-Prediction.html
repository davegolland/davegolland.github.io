Generative Chain of Behavior for User Trajectory Prediction

Generative Chain of Behavior for User Trajectory Prediction

Source: arXiv:cs.IR Published: 2026-01-26 Rank: #9 (score: 0.89, max_sim: 0.85) Reading time: ~0 min Tags: Uncategorized

Abstract

Modeling long-term user behavior trajectories is essential for understanding evolving preferences and enabling proactive recommendations. However, most sequential recommenders focus on next-item prediction, overlooking dependencies across multiple future actions. We propose Generative Chain of Behavior (GCB), a generative framework that models user interactions as an autoregressive chain of semantic behaviors over multiple future steps. GCB first encodes items into semantic IDs via RQ-VAE with k-means refinement, forming a discrete latent space that preserves semantic proximity. On top of this space, a transformer-based autoregressive generator predicts multi-step future behaviors conditioned on user history, capturing long-horizon intent transitions and generating coherent trajectories. Experiments on benchmark datasets show that GCB consistently outperforms state-of-the-art sequential recommenders in multi-step accuracy and trajectory consistency. Beyond these gains, GCB offers a unified generative formulation for capturing user preference evolution.

Key Techniques

  • Semantic ID encoding via RQ-VAE with k-means refinement
  • Autoregressive chain modeling for multi-step behavior prediction
  • Transformer-based trajectory generation
  • Discrete latent space with semantic proximity preservation

Autoregressive Chain of Behavior Generation

Core Idea: Use a transformer-based autoregressive decoder to predict multi-step future user behaviors as a sequence of semantic tokens conditioned on user interaction history, enabling coherent long-horizon trajectory generation.

Problem It Solves: Existing sequential recommenders predict only the next item, ignoring dependencies and intent transitions across multiple future steps. This limits the ability to understand evolving user preferences and generate coherent multi-step behavioral trajectories that respect long-term intent.

How It Works:

  1. Encode user history H = (s₁, s₂, …, sₜ) where each sᵢ is a semantic ID obtained from the RQ-VAE quantization
  2. Create positional embeddings and sum with semantic token embeddings: hᵢ = Embed(sᵢ) + PosEmbed(i)
  3. Pass through transformer encoder to build contextual history representation: C = Transformer_Encoder(**h₁, …, hₜ)
  4. Autoregressively generate future behaviors by predicting one semantic token at a time: P(sₜ₊₁, sₜ₊₂, …, sₜ₊ₙ | C) = ∏ᵢ P(sₜ₊ᵢ | C, sₜ₊₁:ₜ₊ᵢ₋₁)
  5. At each step, sample or argmax to obtain the next semantic ID and append to sequence
  6. Convert final semantic IDs back to actual items for the final trajectory prediction

Architecture:

    B --> C["Transformer<br/>Encoder"]
    C --> D["Context<br/>Representation"]

Where It’s Useful:

  • Multi-step recommendation systems requiring trajectory-level coherence
  • Long-horizon user intent modeling (e.g., session-based recommendations spanning multiple interactions)
  • Preference evolution analysis where capturing intent transitions matters
  • Conversational or sequential recommendation systems generating multi-turn interactions
  • User journey prediction for marketing and retention applications

Limitations / Caveats:

  • Autoregressive generation is slow at inference—must generate tokens sequentially rather than in parallel
  • Error accumulation during multi-step generation (early mistakes propagate to later predictions)
  • Exposure bias during training—the model sees ground truth at training time but its own predictions at test time
  • Requires learning long-range dependencies which can be challenging with standard transformers on very long horizons
  • Quality degrades significantly for very distant future steps (t+n where n is large)

Internal Use Cases:

  • Predicting user behavior chains in RecSys to enable proactive recommendations before users explicitly request them
  • Ranking multi-step user intents for display optimization (show items that fit the predicted chain, not just the next step)
  • Cold-start user modeling by generating realistic trajectory patterns from small interaction histories
  • A/B testing long-horizon recommendation strategies where multi-step coherence is the evaluation metric
