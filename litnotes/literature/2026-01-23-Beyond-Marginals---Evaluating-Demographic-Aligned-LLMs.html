Beyond Marginal Distributions: Evaluating Demographic-Aligned LLMs

Beyond Marginal Distributions: Evaluating Demographic-Aligned LLMs

Source: arXiv:2601.15755 (cs.CL) Date: 2026-01-22 Score: 2.46 (max_sim=0.754, fresh evaluation angle)

The Problem

LLMs are increasingly used to represent human opinions, values, and beliefs. Current evaluation of “aligned” models focuses on marginal response distributions — does the model’s answer distribution match the target population’s for each question independently?

But this misses deeper latent structures that characterize real populations:

  • Correlations between values
  • Cultural value theories
  • Multivariate patterns

Key Insight

Matching marginals ≠ matching the underlying structure. A model could perfectly match individual response distributions while completely failing to capture how those responses relate to each other.

The Evaluation Framework

Beyond Marginals

Propose evaluating aligned models through:

  1. Marginal response distributions (existing)
  2. Multivariate correlation patterns (new)

Comparison: Two Steering Techniques

TechniqueMarginalsCorrelations
Persona PromptingModeratePoor

Using World Values Survey as ground truth.

Key Finding

Demographically fine-tuned models better approximate marginal distributions, but both techniques fail to capture gold standard correlation patterns.

Implications

For Value Alignment Research

  • Representativeness is distinct from value alignment
  • Evaluation focused on marginals gives overly optimistic conclusions
  • Need multivariate metrics to detect structural failures

For LLM-as-Population-Proxy Use Cases

If you’re using LLMs to:

  • Simulate user responses
  • Predict population-level preferences
  • Generate synthetic survey data

…you may be getting individual answers right but missing systematic relationships between them.

Why This Matters for Recommendations

User Preference Modeling

If we use LLMs to model user preferences:

  • Getting individual preferences right isn’t enough
  • Need to capture preference correlations (people who like X tend to like Y)
  • Marginal accuracy could mask missing collaborative signal

Synthetic User Generation

For simulation/testing:

  • Individual response accuracy ≠ realistic user behavior
  • Correlation structure matters for evaluating diversity, coverage

Fairness/Representation

  • A model that matches marginal demographics but misses correlation structure could:
    • Flatten cultural differences
    • Miss intersectional effects
    • Produce unrealistic demographic representations

Key Technique to Extract

Multivariate correlation pattern matching as an evaluation metric for aligned/steered LLMs. Compare full correlation matrix, not just marginal distributions.

Questions

  1. What specific correlation metrics work best? (Pearson? Rank? Structural?)
  2. Could this apply to evaluating user simulators in RecSys?
  3. Is the correlation failure due to training data or architecture limitations?
