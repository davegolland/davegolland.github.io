ReThinker: Scientific Reasoning by Rethinking with Guided Reflection and Confidence Control

Source: arXiv:cs.AI Published: 2026-02-04 Rank: #5 (score: 0.99, max_sim: 0.82) Reading time: ~15 min Tags: RAG, Search, RecSys, LLM

Abstract

Expert-level scientific reasoning remains challenging for large language models, particularly on benchmarks such as Humanity’s Last Exam (HLE), where rigid tool pipelines, brittle multi-agent coordination, and inefficient test-time scaling often limit performance. We introduce ReThinker, a confidence-aware agentic framework that orchestrates retrieval, tool use, and multi-agent reasoning through a stage-wise Solver-Critic-Selector architecture. Rather than following a fixed pipeline, ReThinker dynamically allocates computation based on model confidence, enabling adaptive tool invocation, guided multi-dimensional reflection, and robust confidence-weighted selection. To support scalable training without human annotation, we further propose a reverse data synthesis pipeline and an adaptive trajectory recycling strategy that transform successful reasoning traces into high-quality supervision. Experiments on HLE, GAIA, and XBench demonstrate that ReThinker consistently outperforms state-of-the-art foundation models with tools and existing deep research systems, achieving state-of-the-art results on expert-level reasoning tasks.

ReThinker: Confidence-Aware Agentic Scientific Reasoning

Core Idea: A stage-wise Solver-Critic-Selector framework that dynamically allocates test-time compute based on model confidence, orchestrating retrieval, tool use, and multi-agent reasoning through adaptive tool invocation, guided multi-dimensional reflection, and confidence-weighted selection.

Problem It Solves: Existing scientific reasoning systems suffer from rigid tool pipelines, brittle multi-agent coordination, and inefficient test-time scaling—leading to poor performance on expert-level benchmarks like Humanity’s Last Exam (HLE) where flexible, confidence-aware reasoning is essential.

How It Works:

  1. Solver Stage: Generate initial candidate solutions using retrieval and tool-augmented reasoning; compute confidence scores for each candidate based on model logits and reasoning chain quality
  2. Critic Stage: For low-confidence solutions, invoke multi-dimensional reflection across correctness, logical consistency, evidence sufficiency, and tool usage validity; dynamically decide whether to retrieve additional information or invoke coding/visualization tools based on confidence thresholds
  3. Selector Stage: Aggregate refined candidates using confidence-weighted voting; select final answer based on both solution quality and model certainty
  4. Training Loop: Apply reverse data synthesis (starting from ground-truth answers to generate reasoning traces) and adaptive trajectory recycling (reusing successful paths as supervision) to create training data without human annotation

Architecture:

flowchart LR
    A[Question] --> B[Solver: Generate N candidates]
    B --> C{Confidence Check}
    C -->|Low| D[Critic: Multi-dimensional reflection]
    C -->|High| E[Selector]
    D -->|Need tools?| F[Adaptive Tool Invocation]
    F --> D
    D --> E[Selector: Confidence-weighted voting]
    E --> G[Final Answer]

Where It’s Useful:

  • Expert-level scientific reasoning benchmarks (HLE, GPQA, MATH-500)
  • Complex multi-step question answering requiring retrieval + computation (GAIA)
  • Open-domain reasoning tasks demanding adaptive tool selection (XBench)
  • Any scenario requiring test-time compute scaling with quality guarantees

Limitations / Caveats:

  • Increased inference cost from multi-stage reasoning and confidence-gated tool calls
  • Reverse data synthesis requires ground-truth answers, limiting applicability to fully open-ended exploration tasks
  • Confidence calibration quality directly impacts whether useful reflection/tools are triggered

Internal Use Cases:

  • Ranking systems that need adaptive re-ranking depth based on confidence (e.g., deeper LLM reasoning only for uncertain queries)
  • Recommendation explanation generation where multi-dimensional critique (relevance, diversity, novelty) mirrors the Critic stage
  • Search quality evaluation where confidence-weighted aggregation of multiple relevance signals improves robustness
  • Training data augmentation via trajectory recycling: successful recommendation sessions → synthetic training examples
