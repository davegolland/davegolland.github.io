Abstract

Recent studies increasingly explore Large Language Models (LLMs) as a new paradigm for recommendation systems due to their scalability and world knowledge. However, existing work has three key limitations: (1) most efforts focus on retrieval and ranking, while the reranking phase, critical for refining final recommendations, is largely overlooked; (2) LLMs are typically used in zero-shot or supervised fine-tuning settings, leaving their reasoning abilities, especially those enhanced through reinforcement learning (RL) and high-quality reasoning data, underexploited; (3) items are commonly represented by non-semantic IDs, creating major scalability challenges in industrial systems with billions of identifiers. To address these gaps, we propose the Generative Reasoning Reranker (GR2), an end-to-end framework with a three-stage training pipeline tailored for reranking. First, a pretrained LLM is mid-trained on semantic IDs encoded from non-semantic IDs via a tokenizer achieving 99% uniqueness. Next, a stronger larger-scale LLM generates high-quality reasoning traces through carefully designed prompting and rejection sampling, which are used for supervised fine-tuning to impart foundational reasoning skills. Finally, we apply Decoupled Clip and Dynamic sAmpling Policy Optimization (DAPO), enabling scalable RL supervision with verifiable rewards designed specifically for reranking. Experiments on two real-world datasets demonstrate GR2’s effectiveness: it surpasses the state-of-the-art OneRec-Think by 2.4% in Recall@5 and 1.3% in NDCG@5. Ablations confirm that advanced reasoning traces yield substantial gains across metrics. We further find that RL reward design is crucial in reranking: LLMs tend to exploit reward hacking by preserving item order, motivating conditional verifiable rewards to mitigate this behavior and optimize reranking performance.

Key Techniques

  • semantic ID tokenization
  • supervised fine-tuning with reasoning traces
  • Decoupled Clip and Dynamic sAmpling Policy Optimization (DAPO)

Generative Reasoning Reranker (GR2)

Core Idea: A three-stage LLM training pipeline for recommendation reranking that combines semantic ID encoding, reasoning trace generation via supervised fine-tuning, and reinforcement learning with verifiable rewards to optimize final ranking quality.

Problem It Solves: Existing LLM-based recommendation systems focus on retrieval/ranking while neglecting reranking, fail to leverage reasoning abilities enhanced through RL, suffer from scalability issues with non-semantic IDs in billion-scale industrial systems, and are vulnerable to reward hacking where models preserve input order instead of truly reranking.

How It Works:

  1. Mid-training on Semantic IDs: Pretrained LLM learns to encode non-semantic item IDs into semantic representations via a tokenizer achieving ≥99% uniqueness, solving scalability for billions of items
  2. Reasoning Trace SFT: A larger-scale LLM generates high-quality reasoning traces through carefully designed prompting and rejection sampling; these traces are used for supervised fine-tuning to teach foundational reasoning skills
  3. DAPO Reinforcement Learning: Apply Decoupled Clip and Dynamic sAmpling Policy Optimization with conditional verifiable rewards specifically designed to prevent order preservation exploitation and optimize actual reranking performance

Architecture:

flowchart LR
    A[Non-semantic IDs] --> B[Semantic ID Tokenizer]
    B --> C[Mid-trained LLM]
    C --> D[Reasoning Trace Generator]
    D --> E[SFT with Traces]
    E --> F[DAPO RL Training]
    F --> G[Reranked Item List]
    H[Verifiable Rewards] --> F

Where It’s Useful:

  • Industrial-scale recommendation systems requiring final-stage reranking with billions of items
  • E-commerce, content platforms, and search systems where ranking quality directly impacts business metrics
  • Any LLM-based recommender needing to exploit reasoning capabilities while avoiding reward hacking

Limitations / Caveats:

  • Requires multi-stage training pipeline with access to both smaller base LLM and larger reasoning LLM for trace generation
  • Reward design is critical and domain-specific; poorly designed rewards lead to order preservation instead of meaningful reranking
  • Semantic ID encoding may lose some information from original non-semantic IDs despite 99% uniqueness
  • Computational cost of RL training stage may be prohibitive for frequent retraining

Internal Use Cases:

  • RecSys final-stage reranking where candidate lists need refinement using world knowledge and reasoning
  • Multi-objective ranking optimization using RL with verifiable business metrics as rewards
  • Scaling LLM recommenders to handle massive item catalogs via semantic ID compression
  • Addressing reward hacking in ranking RL by designing conditional rewards that penalize order preservation
