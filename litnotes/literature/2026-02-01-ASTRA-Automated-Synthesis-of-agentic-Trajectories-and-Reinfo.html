ASTRA: Automated Synthesis of agentic Trajectories and Reinforcement Arenas

Source: arXiv:cs.CL Published: 2026-01-29 Rank: #2 (score: 1.00, max_sim: 0.82) Reading time: ~20 min Tags: RAG, RL, LLM Code: https://github.com/LianjiaTech/astra

Abstract

Large language models (LLMs) are increasingly used as tool-augmented agents for multi-step decision making, yet training robust tool-using agents remains challenging. Existing methods still require manual intervention, depend on non-verifiable simulated environments, rely exclusively on either supervised fine-tuning (SFT) or reinforcement learning (RL), and struggle with stable long-horizon, multi-turn learning. To address these challenges, we introduce ASTRA, a fully automated end-to-end framework for training tool-augmented language model agents via scalable data synthesis and verifiable reinforcement learning. ASTRA integrates two complementary components. First, a pipeline that leverages the static topology of tool-call graphs synthesizes diverse, structurally grounded trajectories, instilling broad and transferable tool-use competence. Second, an environment synthesis framework that captures the rich, compositional topology of human semantic reasoning converts decomposed question-answer traces into independent, code-executable, and rule-verifiable environments, enabling deterministic multi-turn RL. Based on this method, we develop a unified training methodology that integrates SFT with online RL using trajectory-level rewards to balance task completion and interaction efficiency. Experiments on multiple agentic tool-use benchmarks demonstrate that ASTRA-trained models achieve state-of-the-art performance at comparable scales, approaching closed-source systems while preserving core reasoning ability. We release the full pipelines, environments, and trained models at https://github.com/LianjiaTech/astra.

Key Techniques

  • Tool-call graph synthesis
  • verifiable environment synthesis

Environment Synthesis from Semantic Reasoning Traces

Core Idea: Convert human question-answer reasoning traces into independent, code-executable, and rule-verifiable environments that enable deterministic multi-turn reinforcement learning.

Problem It Solves: Existing RL methods for agents rely on non-verifiable simulated environments or manual environment design; this approach creates grounded, deterministic environments from human reasoning patterns while enabling reliable reward signals for RL training.

How It Works:

  1. Trace Decomposition: Break down complex human reasoning demonstrations into atomic question-answer pairs, extracting intermediate states and decision points
  2. Semantic Extraction: Identify the compositional reasoning rules and constraints implicit in each Q-A trace (e.g., logical conditions, causal relationships)
  3. Environment Codification: Convert extracted rules into executable code (deterministic state transitions and rule-based verification logic)
  4. Reward Definition: Define rule-verifiable rewards at the trajectory level based on task completion and interaction efficiency metrics

Architecture:

flowchart LR
    A[Human Q-A Traces] --> B[Decompose into Atomic Steps]
    B --> C[Extract Semantic Rules]
    C --> D[Codify as Executable Rules]
    D --> E[Define Verifiable Rewards]
    E --> F[Executable Environments]
    F --> G[Multi-turn RL Training]

Where It’s Useful:

  • Training agents with reliable, verifiable reward signals instead of simulators
  • Enabling stable long-horizon, multi-turn reinforcement learning
  • Creating compositional environments for complex, multi-step decision-making tasks
  • Grounding agent training in human semantic reasoning patterns

Limitations / Caveats:

  • Requires decomposable question-answer traces from human demonstrations; not all reasoning is easily decomposable
  • Codification effort depends on the complexity and diversity of reasoning patterns
  • Rule-based environments may miss edge cases not present in the human traces

Internal Use Cases:

  • Training recommendation agents with verifiable reward signals (e.g., user satisfaction rules derived from interaction traces)
  • Building ranking and search agents that can be evaluated deterministically without relying on learned reward models
  • Creating explainable agent training where rewards are tied to interpretable, rule-based semantics

Unified SFT + Online RL Training Methodology

Core Idea: Integrate supervised fine-tuning with online reinforcement learning using trajectory-level rewards to balance task completion effectiveness with interaction efficiency in agent training.

Problem It Solves: Prior approaches rely exclusively on either SFT or RL, lacking stable integration for long-horizon multi-turn learning; trajectory-level rewards enable balancing multiple objectives without agent instability.

How It Works:

  1. SFT Phase: Initialize the agent using synthetic trajectories from Tool-Call Graph Synthesis with supervised fine-tuning to establish broad tool-use competence
  2. Online RL Phase: Fine-tune the SFT model through online interaction with synthesized environments, collecting real agent trajectories
  3. Trajectory-Level Reward Assignment: Evaluate complete trajectories (not individual steps) using composite rewards: task completion score + interaction efficiency penalty
  4. Gradient Updates: Apply policy gradients based on trajectory rewards, enabling the agent to learn both what tools to use and how to use them efficiently

Architecture:

flowchart LR
    A[Synthetic Trajectories] --> B[Supervised Fine-Tuning]
    B --> C[Initialize Agent]
    C --> D[Online Environment Interaction]
    D --> E[Collect Agent Trajectories]
    E --> F[Compute Trajectory Rewards]
    F --> G[Policy Gradient Updates]
    G --> H{Converged?]
    H -->|No| D
    H -->|Yes| I[Trained Agent]

Where It’s Useful:

  • Training multi-step decision-making agents that require both competence and efficiency
  • Balancing exploration-exploitation tradeoffs in long-horizon agent tasks
  • Scenarios where both task success and minimizing interaction steps are important

Limitations / Caveats:

  • Requires well-designed trajectory-level reward functions; poorly balanced rewards can prioritize efficiency over correctness
  • Online RL phase depends on quality of synthesized environments; biases in environment creation propagate to agent behavior
  • Trajectory-level rewards may mask failure modes in individual steps

Internal Use Cases:

  • Training recommendation agents that both find relevant items and minimize user interaction overhead
  • Building search ranking agents that balance result quality with query interpretation steps
