LogicSkills: A Structured Benchmark for Formal Reasoning in Large Language Models

Source: arXiv:cs.CL Published: 2026-02-06 Rank: #1 (score: 1.00, max_sim: 0.82) Reading time: ~15 min Tags: RAG, LLM

Abstract

Large language models have demonstrated notable performance across various logical reasoning benchmarks. However, it remains unclear which core logical skills they truly master. To address this, we introduce LogicSkills, a unified benchmark designed to isolate three fundamental skills in formal reasoning: (i) translating premises into first-order logic; (ii) formulating a finite structure in which all premises are true while the conclusion is false; and (iii) deciding whether a conclusion follows from a given set of premises. Items are drawn from the two-variable fragment of first-order logic (without identity) and are presented in both natural English and a Carroll-style language with nonce words. All examples are verified for correctness and non-triviality using the SMT solver Z3. Across leading models, performance is high on validity but substantially lower on symbolization and countermodel construction, suggesting reliance on surface-level patterns rather than genuine symbolic or rule-based reasoning.

LogicSkills Benchmark

Core Idea: A structured benchmark that decomposes formal reasoning into three isolated, verifiable skills—symbolization, countermodel construction, and validity assessment—to diagnose which logical capabilities LLMs truly possess versus superficial pattern matching.

Problem It Solves: Existing logical reasoning benchmarks provide only end-to-end accuracy scores, making it impossible to determine whether models succeed through genuine symbolic reasoning or by exploiting surface-level linguistic patterns. This prevents targeted improvement of logical reasoning capabilities.

How It Works:

  1. Task Decomposition: Isolates three core skills from formal logic:
    • Formal symbolization: Translate natural language premises into first-order logic formulas
    • Countermodel construction: Generate a finite structure where all premises are true but the conclusion is false
    • Validity assessment: Determine whether a conclusion logically follows from premises
  2. Dual Language Testing: Presents each problem in both natural English and Carroll-style language with nonce words (e.g., “All blicks are flurbs”) to detect reliance on world knowledge versus pure logical structure
  3. Automated Verification: Uses Z3 SMT solver to verify correctness and ensure non-triviality of all test items, restricting problems to the two-variable fragment of first-order logic (without identity)

Where It’s Useful:

  • Diagnosing specific weaknesses in LLM reasoning pipelines (e.g., symbolization failure vs. inference failure)
  • Evaluating formal verification and proof assistant systems
  • Training curricula for logical reasoning in language models
  • Benchmarking neurosymbolic systems that combine neural and symbolic reasoning

Limitations / Caveats:

  • Restricted to two-variable first-order logic fragment, which may not capture complexity of real-world reasoning
  • High performance on validity assessment but low on symbolization/countermodels suggests models may use shortcuts rather than genuine logical processing
  • Carroll-style nonce words may introduce unnatural linguistic structures that don’t reflect real reasoning scenarios

Internal Use Cases:

  • RecSys/Ranking: Test whether ranking models can properly symbolize user preferences and constraints for explainable recommendations
  • LLM Guardrails: Evaluate whether safety models can correctly formalize policy rules into logical constraints before applying them
  • Retrieval Augmentation: Assess if query understanding modules can translate complex multi-constraint queries into formal search predicates
