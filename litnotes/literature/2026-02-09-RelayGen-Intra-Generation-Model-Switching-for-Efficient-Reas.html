RelayGen: Intra-Generation Model Switching for Efficient Reasoning

RelayGen: Intra-Generation Model Switching for Efficient Reasoning

Source: arXiv:cs.CL Published: 2026-02-06 Rank: #5 (score: 0.96, max_sim: 0.84) Reading time: ~0 min Tags: Uncategorized

Abstract

Large reasoning models (LRMs) achieve strong performance on complex reasoning tasks by generating long, multi-step reasoning trajectories, but inference-time scaling incurs substantial deployment cost. A key challenge is that generation difficulty varies within a single output, whereas existing efficiency-oriented approaches either ignore this intra-generation variation or rely on supervised token-level routing with high system complexity. We present \textbf{RelayGen}, a training-free, segment-level runtime model switching framework that exploits difficulty variation in long-form reasoning. Through offline analysis of generation uncertainty using token probability margins, we show that coarse-grained segment-level control is sufficient to capture difficulty transitions within a reasoning trajectory. RelayGen identifies model-specific switch cues that signal transitions to lower-difficulty segments and dynamically delegates their continuation to a smaller model, while preserving high-difficulty reasoning on the large model. Across multiple reasoning benchmarks, RelayGen substantially reduces inference latency while preserving most of the accuracy of large models. When combined with speculative decoding, RelayGen achieves up to 2.2 end-to-end speedup with less than 2% accuracy degradation, without requiring additional training or learned routing components.

Key Techniques

  • Training-free segment-level runtime model switching
  • token probability margin analysis for difficulty detection
  • dynamic delegation to smaller models
  • speculative decoding integration

RelayGen (Relay Generation)

Core Idea: Training-free segment-level model switching that dynamically delegates low-difficulty portions of long reasoning trajectories to smaller models while keeping high-difficulty reasoning on large models, guided by offline-analyzed token probability margins.

Problem It Solves: Large reasoning models (LRMs) generate expensive long-form reasoning chains where generation difficulty varies significantly within a single output, but existing efficiency methods either ignore this intra-generation variation or require complex supervised token-level routing.

How It Works:

  1. Offline Analysis: Analyze token probability margins across reasoning trajectories to identify model-specific difficulty patterns and switch cues that signal transitions to easier segments
  2. Runtime Monitoring: During generation with the large model, monitor for learned switch cues that indicate upcoming low-difficulty segments
  3. Dynamic Delegation: When a switch cue is detected, transfer generation control to a smaller model for the easier continuation, then return to the large model when difficulty increases or segment ends
  4. Coarse-Grained Control: Operate at segment level (multi-token chunks) rather than token-by-token, reducing switching overhead while capturing meaningful difficulty transitions

Architecture:

flowchart LR
    A[Input Query] --> B[Large Model<br/>High-Difficulty<br/>Reasoning]
    B --> C{Switch Cue<br/>Detected?}
    C -->|Yes| D[Small Model<br/>Low-Difficulty<br/>Continuation]
    C -->|No| B
    D --> E{Difficulty<br/>Increase?}
    E -->|Yes| B
    E -->|No| D
    B --> F[Final Output]
    D --> F

Where It’s Useful:

  • Long-form reasoning tasks (math, code generation, multi-step problem solving)
  • Production deployments where inference cost dominates
  • Scenarios requiring response quality with tight latency budgets
  • Compatible with speculative decoding for compounded speedups (2.2× observed)

Limitations / Caveats:

  • Requires offline analysis phase to identify model-specific switch cues and difficulty patterns
  • Segment-level switching may miss fine-grained difficulty variations within segments
  • Effectiveness depends on difficulty variation within reasoning trajectories (less benefit for uniformly hard problems)
  • Switch cue detection accuracy affects overall performance preservation
  • Up to 2% accuracy degradation observed even with best configurations

Internal Use Cases:

  • Hybrid reasoning systems where expensive LRM calls can be selectively replaced with cheaper models for straightforward explanation/justification portions
  • Multi-stage ranking pipelines that generate textual reasoning (e.g., why this item matches) - delegate simple match explanations to small models
  • LLM-based feature generation where some features require deep reasoning while others are formulaic - route accordingly
  • Real-time recommendation explanations where latency matters but quality degradation must stay minimal
