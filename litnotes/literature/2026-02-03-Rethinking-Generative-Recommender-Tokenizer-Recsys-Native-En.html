Rethinking Generative Recommender Tokenizer: Recsys-Native Encoding and Semantic Quantization Beyond LLMs

Source: arXiv:cs.IR Published: 2026-02-02 Rank: #5 (score: 0.00, max_sim: 0.90) Reading time: ~0 min Tags: Uncategorized Code: https://arxiv.org/pdf/2602.02338v1

Abstract

Semantic ID (SID)-based recommendation is a promising paradigm for scaling sequential recommender systems, but existing methods largely follow a semantic-centric pipeline: item embeddings are learned from foundation models and discretized using generic quantization schemes. This design is misaligned with generative recommendation objectives: semantic embeddings are weakly coupled with collaborative prediction, and generic quantization is inefficient at reducing sequential uncertainty for autoregressive modeling. To address these, we propose ReSID, a recommendation-native, principled SID framework that rethinks representation learning and quantization from the perspective of information preservation and sequential predictability, without relying on LLMs. ReSID consists of two components: (i) Field-Aware Masked Auto-Encoding (FAMAE), which learns predictive-sufficient item representations from structured features, and (ii) Globally Aligned Orthogonal Quantization (GAOQ), which produces compact and predictable SID sequences by jointly reducing semantic ambiguity and prefix-conditional uncertainty. Theoretical analysis and extensive experiments across ten datasets show the effectiveness of ReSID. ReSID consistently outperforms strong sequential and SID-based generative baselines by an average of over 10%, while reducing tokenization cost by up to 122x. Code is available at https://github.com/FuCongResearchSquad/ReSID.

Globally Aligned Orthogonal Quantization (GAOQ)

Core Idea: Quantize continuous item embeddings into discrete semantic IDs by jointly optimizing for semantic alignment and prefix-conditional uncertainty reduction, ensuring both item discrimination and predictable autoregressive sequences.

Problem It Solves: Generic quantization schemes (e.g., standard vector quantization, product quantization) optimize only for reconstruction accuracy, not for sequential predictability. This results in quantization errors that propagate and compound during autoregressive generation. GAOQ jointly minimizes semantic ambiguity (which items map to the same ID) and prefix-conditional entropy (uncertainty given the history), directly targeting the generative recommendation objective.

How It Works:

  1. Initialize K learned codebook vectors {c₁, c₂, …, c_K} where K is the vocabulary size
  2. For each item embedding ϕ(i), assign discrete ID z(i) = argmin_k ||ϕ(i) - c_k||₂ (nearest codebook vector)
  3. Apply orthogonal constraints to codebook to ensure vectors are well-separated and stable
  4. Optimize codebook jointly with two objectives:
    • Semantic alignment: minimize reconstruction error ||ϕ(i) - c_{z(i)}||² (reduce semantic ambiguity)
    • Prefix-conditional loss: for sequences, minimize cross-entropy loss L(z_t | z_{<t}) given history, forcing IDs to be predictable from context
  5. Use global alignment loss to ensure codebook vectors are balanced across the embedding space (avoid collapse to few IDs)
  6. Output discrete token sequences where each item i is represented by ID z(i) ∈ {1, …, K}

Where It’s Useful:

  • Generative sequential recommendation where tokens feed into autoregressive models
  • Scenarios requiring compact, discrete representations for efficient inference
  • Systems where both semantic quality and predictability matter (item similarity + sequence coherence)
  • Tokenization for language-model-based recommenders

Limitations / Caveats:

  • Requires balancing two potentially competing objectives (semantic vs. predictive); hyperparameter tuning needed
  • Orthogonal constraints scale poorly as codebook size K increases (expensive for very large vocabularies)
  • Assumes prefix history is informative for next-item prediction; may perform poorly in sparse/sparse-history regimes
  • Discrete quantization is non-differentiable; straight-through estimators or other tricks needed during backprop

Internal Use Cases:

  • Tokenizing recommendation sequences for fine-tuning LLMs or training transformer-based recommenders
  • Creating “semantic ID” vocabularies that compress item catalogs while maintaining predictive power
  • Reducing inference cost in production by using small discrete IDs instead of dense embeddings

Analysis: ReSID and Its Place in Semantic ID Research

ReSID fundamentally challenges the LLM-centric assumption that has dominated recent semantic ID work. While papers like the CoST (Contrastive Quantization based Semantic Tokenization) and the broader “Improving Recommendation Systems in the Age of LLM” literature assume foundation models as the starting point for embeddings, ReSID explicitly rejects this dependency. Instead, it proposes recommendation-native learning through Field-Aware Masked Auto-Encoding (FAMAE)—a paradigm shift that asks: why should we inherit semantic representations designed for language when we can learn representations optimized for sequential predictability? This directly extends the original RQ-VAE work (Better Generalization with Semantic IDs) by replacing generic quantization with domain-specific objectives, but goes further by questioning the embedding source itself.

The paper’s core innovation—Globally Aligned Orthogonal Quantization (GAOQ)—builds on insights from CoST’s success with neighborhood preservation during quantization (+43% recall@5 improvement), but reframes the problem. Where CoST focuses on maintaining collaborative relationships through contrastive learning, ReSID tackles prefix-conditional uncertainty and semantic ambiguity jointly. This is a crucial methodological bridge: both papers recognize that naive reconstruction-based quantization (minimizing reconstruction error) is misaligned with recommendation objectives, but ReSID goes deeper by treating quantization as an information-theoretic problem about reducing sequential predictability uncertainty rather than just preserving neighborhood structure. The 122x reduction in tokenization cost while achieving 10% average improvements suggests that FAMAE’s structured feature learning is more sample-efficient than relying on expensive foundation model embeddings.

What ReSID adds to the knowledge base is a post-LLM perspective on semantic tokenization. The matched notes show a clear trend toward integrating LLMs (Generative Explore-Exploit using LLM optimizers, semantic augmentation architectures), but ReSID’s 10% gains without LLMs suggest that this trend may be over-correcting. Its success indicates that for recommendation systems, task-specific inductive biases (structured fields, sequential prediction as the primary objective) may provide more value than general-purpose semantic knowledge. This doesn’t invalidate LLM augmentation but repositions it as complementary rather than foundational, opening a new research direction: hybrid approaches that selectively integrate LLM-derived signals only where collaborative filtering alone is weak (e.g., cold-start, cross-domain transfer).
