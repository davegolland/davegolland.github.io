When to Memorize and When to Stop: Gated Recurrent Memory for Long-Context Reasoning

Source: arXiv:cs.CL Published: 2026-02-11 Rank: #3 (score: 0.99, max_sim: 0.82) Reading time: ~20 min Tags: RAG, RL, LLM

Abstract

While reasoning over long context is crucial for various real-world applications, it remains challenging for large language models (LLMs) as they suffer from performance degradation as the context length grows. Recent work MemAgent has tried to tackle this by processing context chunk-by-chunk in an RNN-like loop and updating a textual memory for final answering. However, this naive recurrent memory update faces two crucial drawbacks: (i) memory can quickly explode because it can update indiscriminately, even on evidence-free chunks; and (ii) the loop lacks an exit mechanism, leading to unnecessary computation after even sufficient evidence is collected. To address these issues, we propose GRU-Mem, which incorporates two text-controlled gates for more stable and efficient long-context reasoning. Specifically, in GRU-Mem, the memory only updates when the update gate is open and the recurrent loop will exit immediately once the exit gate is open. To endow the model with such capabilities, we introduce two reward signals and within end-to-end RL, rewarding the correct updating and exiting behaviors respectively. Experiments on various long-context reasoning tasks demonstrate the effectiveness and efficiency of GRU-Mem, which generally outperforms the vanilla MemAgent with up to 400% times inference speed acceleration.

Key Techniques

  • Gated memory updates
  • Exit gate mechanism
  • End-to-end reinforcement learning with reward signals

GRU-Mem (Gated Recurrent Memory)

Core Idea: A gated recurrent memory system for long-context LLM reasoning that uses text-controlled update and exit gates to selectively update memory on informative chunks and terminate processing early when sufficient evidence is collected.

Problem It Solves: Naive recurrent memory approaches like MemAgent suffer from memory explosion (updating on irrelevant chunks) and computational waste (continuing to process after gathering sufficient evidence), leading to degraded performance and inefficiency on long-context reasoning tasks.

How It Works:

  1. Process long context chunk-by-chunk in an RNN-like loop, maintaining a textual memory state
  2. At each chunk, compute an update gate (text-controlled) that determines whether the memory should be updated based on chunk informativeness
  3. Apply gated update: memory changes only when update gate is open, preventing pollution from evidence-free chunks
  4. Compute an exit gate (text-controlled) that signals when sufficient evidence has been collected
  5. Terminate the recurrent loop immediately when exit gate opens, avoiding unnecessary computation
  6. Train both gates end-to-end via RL with two reward signals: (rewards correct updating behavior) and (rewards correct early termination)

Architecture:

flowchart LR
    A[Context Chunk t] --> B[Update Gate]
    A --> C[Memory Update]
    B -->|Gate Open?| C
    C --> D[Memory State t]
    D --> E[Exit Gate]
    E -->|Open?| F[Terminate & Answer]
    E -->|Closed?| G[Next Chunk t+1]
    G --> A

Where It’s Useful:

  • Long-document question answering where evidence is sparse
  • Multi-hop reasoning over extended contexts (books, reports, legal documents)
  • Retrieval-augmented generation with large candidate sets
  • Any task requiring selective information accumulation over long sequences

Limitations / Caveats:

  • Requires RL training with carefully designed reward signals for gate control
  • Text-controlled gates may struggle with implicit or ambiguous relevance signals
  • Early exit mechanism assumes evidence sufficiency can be reliably detected mid-stream
  • Performance depends on chunk segmentation strategy

Internal Use Cases:

  • Long-context RAG: Gate updates to only accumulate relevant retrieved chunks, exit early when answer is found
  • Session-based recommendation: Update user memory only on informative interactions, exit session modeling when intent is clear
  • Document ranking: Process candidate documents sequentially, exit when top-k confidence threshold is met
