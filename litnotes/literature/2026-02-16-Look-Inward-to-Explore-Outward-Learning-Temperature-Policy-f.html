Look Inward to Explore Outward: Learning Temperature Policy from LLM Internal States via Hierarchical RL

Source: arXiv:cs.CL Published: 2026-02-13 Rank: #3 (score: 1.00, max_sim: 0.82) Reading time: ~15 min Tags: LLM, RL

Abstract

Reinforcement Learning from Verifiable Rewards (RLVR) trains large language models (LLMs) from sampled trajectories, making decoding strategy a core component of learning rather than a purely inference-time choice. Sampling temperature directly controls the exploration—exploitation trade-off by modulating policy entropy, yet existing methods rely on static values or heuristic adaptations that are decoupled from task-level rewards. We propose Introspective LLM, a hierarchical reinforcement learning framework that learns to control sampling temperature during generation. At each decoding step, the model selects a temperature based on its hidden state and samples the next token from the resulting distribution. Temperature and token policies are jointly optimized from downstream rewards using a coordinate ascent scheme. Experiments on mathematical reasoning benchmarks show that learned temperature policies outperform fixed and heuristic baselines, while exhibiting interpretable exploration behaviors aligned with reasoning uncertainty.

Introspective LLM (Hierarchical Temperature Control via RL)

Core Idea: A hierarchical reinforcement learning framework that learns to dynamically control sampling temperature at each decoding step by conditioning on the LLM’s hidden states, jointly optimizing both temperature selection and token generation from downstream task rewards.

Problem It Solves: Existing RLVR methods use static temperature values or heuristic schedules that are decoupled from task rewards, leading to suboptimal exploration-exploitation trade-offs during trajectory sampling for policy learning.

How It Works:

  1. Hierarchical Policy: At each decoding step t, a high-level policy π_temp(τ_t | h_t) selects temperature τ_t conditioned on hidden state h_t; a low-level policy π_token(x_t | τ_t, context) samples the next token from the temperature-modulated distribution.
  2. Coordinate Ascent Optimization: Alternates between (a) fixing token policy and updating temperature policy via policy gradients on trajectory rewards, and (b) fixing temperature policy and updating token policy via standard RLVR (e.g., PPO, DPO).
  3. Joint Training: Both policies receive the same downstream task reward (e.g., correctness on mathematical reasoning), enabling the temperature controller to learn task-aligned exploration strategies based on internal uncertainty signals.

Architecture:

flowchart LR
    A[Hidden State h_t] --> B[Temperature Policy π_temp]
    B --> C[Sample τ_t]
    C --> D[Token Policy π_token]
    E[Context] --> D
    D --> F[Sample Token x_t]
    F --> G[Reward R]
    G --> H[Update π_temp via PG]
    G --> I[Update π_token via RLVR]

Where It’s Useful:

  • Mathematical reasoning tasks requiring adaptive exploration (proof search, multi-step problem solving)
  • RLVR training pipelines where sampling quality directly impacts policy optimization efficiency
  • Any LLM application with verifiable rewards where uncertainty-aware generation improves sample efficiency

Limitations / Caveats:

  • Requires verifiable downstream rewards (not applicable to open-ended generation without ground truth)
  • Coordinate ascent may be sensitive to alternation schedule and hyperparameters
  • Adds training complexity compared to fixed-temperature baselines
  • Interpretability of learned temperature policies depends on hidden state quality and task structure

Internal Use Cases:

  • LLM reasoning pipelines: Learn when to explore diverse reasoning paths vs. exploit high-confidence steps in multi-hop QA or code generation
  • RLHF data collection: Improve trajectory diversity during preference data generation by adapting temperature based on model uncertainty
  • Retrieval-augmented generation: Modulate exploration when model hidden states indicate low confidence or ambiguous context
