AfriNLLB: Efficient Translation Models for African Languages

AfriNLLB: Efficient Translation Models for African Languages

Source: arXiv:cs.CL Published: 2026-02-10 Rank: #7 (score: 0.10, max_sim: 0.75) Reading time: ~0 min Tags: Uncategorized

Abstract

In this work, we present AfriNLLB, a series of lightweight models for efficient translation from and into African languages. AfriNLLB supports 15 language pairs (30 translation directions), including Swahili, Hausa, Yoruba, Amharic, Somali, Zulu, Lingala, Afrikaans, Wolof, and Egyptian Arabic, as well as other African Union official languages such as Arabic (MSA), French, Portuguese, and Spanish. Our training data covers bidirectional translation between English and 13 languages, and between French and two languages (Lingala and Wolof). AfriNLLB models are based on NLLB-200 600M, which we compress using iterative layer pruning and quantization. We fine-tune the pruned models on parallel corpora we curated for African languages, employing knowledge distillation from a larger teacher model. Our work aims at enabling efficient deployment of translation models for African languages in resource-constrained settings. Our evaluation results demonstrate that AfriNLLB models achieve performance comparable to the baseline while being significantly faster. We release two versions of the AfriNLLB models, a Transformers version that allows further fine-tuning and a CTranslate2 version for efficient inference. Moreover, we release all the training data that we used for fine-tuning the baseline and pruned models to facilitate further research.

Key Techniques

  • Layer pruning
  • quantization
  • knowledge distillation
  • parallel corpus curation

Iterative Layer Pruning with Knowledge Distillation for Neural Machine Translation

Core Idea: Compress large multilingual translation models by iteratively removing transformer layers while fine-tuning with knowledge distillation from a larger teacher model to maintain translation quality.

Problem It Solves: Large multilingual translation models (like NLLB-200) are too computationally expensive and memory-intensive to deploy in resource-constrained settings, particularly for serving African languages where infrastructure may be limited.

How It Works:

  1. Layer Pruning: Start with NLLB-200 600M baseline model and iteratively remove transformer layers to reduce model size and inference latency
  2. Knowledge Distillation: Fine-tune the pruned model on curated parallel corpora for African languages, using predictions from the larger unpruned model as soft targets to preserve translation quality
  3. Quantization: Apply quantization techniques to further reduce model size and memory footprint
  4. Dual Deployment: Release both Transformers version (for further fine-tuning) and CTranslate2 version (for efficient inference)

Architecture:

    A[NLLB-200 600M<br/>Teacher Model] --> B[Layer Pruning]
    B --> C[Pruned Model<br/>Student]
    A -.Soft Targets.-> D[Knowledge Distillation]
    C --> D
    E[African Language<br/>Parallel Corpora] --> D
    D --> F[Fine-tuned Model]
    F --> G[Quantization]
    G --> H[AfriNLLB<br/>Compressed Model]

Where It’s Useful:

  • Deploying multilingual NMT systems in low-resource settings with limited compute/memory
  • Mobile or edge translation applications for African languages
  • Cost-efficient serving of translation models across 30 translation directions
  • Scenarios requiring fast inference with acceptable quality trade-offs

Limitations / Caveats:

  • Performance degradation from layer pruning may not be uniform across all 15 language pairs
  • Requires access to larger teacher model during training (though not at inference)
  • Quality-speed trade-off depends on how aggressively layers are pruned
  • May require language-specific tuning to determine optimal pruning ratios

Internal Use Cases:

  • Feed Localization: Compress multilingual content understanding models for real-time feed translation/summarization across diverse language markets
  • RecSys Efficiency: Apply iterative pruning + distillation to compress large ranking models or LLM-based recommenders for lower latency serving
  • Inference Optimization: Use CTranslate2-style quantization approach for deploying compressed LLMs in production ranking systems where millisecond latency matters
  • Multi-Market Models: Build single compressed models serving multiple geographic markets (similar to 15 language pairs here) instead of maintaining separate models per locale
