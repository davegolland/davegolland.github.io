The Vision Wormhole: Latent-Space Communication in Heterogeneous Multi-Agent Systems

The Vision Wormhole: Latent-Space Communication in Heterogeneous Multi-Agent Systems

Source: arXiv:cs.CL Published: 2026-02-17 Rank: #3 (score: 0.85, max_sim: 0.83) Reading time: ~0 min Tags: Uncategorized

Abstract

Multi-Agent Systems (MAS) powered by Large Language Models have unlocked advanced collaborative reasoning, yet they remain shackled by the inefficiency of discrete text communication, which imposes significant runtime overhead and information quantization loss. While latent state transfer offers a high-bandwidth alternative, existing approaches either assume homogeneous sender-receiver architectures or rely on pair-specific learned translators, limiting scalability and modularity across diverse model families with disjoint manifolds. In this work, we propose the Vision Wormhole, a novel framework that repurposes the visual interface of Vision-Language Models (VLMs) to enable model-agnostic, text-free communication. By introducing a Universal Visual Codec, we map heterogeneous reasoning traces into a shared continuous latent space and inject them directly into the receiver’s visual pathway, effectively treating the vision encoder as a universal port for inter-agent telepathy. Our framework adopts a hub-and-spoke topology to reduce pairwise alignment complexity from O(N^2) to O(N) and leverages a label-free, teacher-student distillation objective to align the high-speed visual channel with the robust reasoning patterns of the text pathway. Extensive experiments across heterogeneous model families (e.g., Qwen-VL, Gemma) demonstrate that the Vision Wormhole reduces end-to-end wall-clock time in controlled comparisons while maintaining reasoning fidelity comparable to standard text-based MAS. Code is available at https://github.com/xz-liu/heterogeneous-latent-mas

Vision Wormhole (VW)

Core Idea: Repurpose the visual input pathway of Vision-Language Models (VLMs) as a universal inter-agent communication channel by encoding heterogeneous reasoning traces into synthetic visual tokens, enabling text-free, model-agnostic latent state transfer between agents of different architectures.

Problem It Solves: Standard multi-agent LLM systems communicate via serialized text, incurring tokenization overhead, context-length bottlenecks, and lossy information quantization. Direct latent-space transfer is faster but breaks down across heterogeneous model families whose internal representation manifolds are disjoint — existing translators require O(N²) pairwise adapters that don’t generalize.

How It Works:

  1. Encode: The sender agent produces a reasoning trace (hidden states or intermediate activations); a Universal Visual Codec (a lightweight encoder) maps these heterogeneous latent vectors into a shared continuous visual embedding space — a “hub” representation agnostic to the source architecture.
  2. Transmit: The hub embedding is rendered as synthetic visual tokens and injected directly into the receiver VLM’s vision encoder input pathway, bypassing text generation entirely.
  3. Align: A teacher-student distillation objective trains the visual channel (student) to reproduce the reasoning behavior of the text pathway (teacher), label-free — keeping the visual shortcut faithful to the model’s native reasoning patterns without requiring task-specific supervision.

Architecture:

flowchart LR
    A[Sender Agent\nHidden States] --> B[Universal Visual Codec\nHub Encoder]
    B --> C[Shared Visual\nEmbedding Space]
    C --> D[Spoke Decoder\nper receiver family]
    D --> E[Receiver VLM\nVision Encoder Port]
    E --> F[Receiver Agent\nReasoning Output]

    G[Text Pathway\nTeacher] -.->|distillation loss| D

Where It’s Useful:

  • Heterogeneous multi-agent pipelines where different tasks are routed to specialized model families (e.g., Qwen-VL for vision-heavy subtasks, Gemma for text reasoning)
  • High-throughput agentic systems where text serialization latency is a bottleneck (real-time tool-use chains, streaming inference)
  • Cross-architecture knowledge distillation at inference time, not just training time
  • Any MAS topology requiring N agents to share reasoning state without a shared tokenizer or vocabulary

Limitations / Caveats:

  • Requires all receiver agents to be VLMs with an accessible vision encoder — pure text-only LLMs cannot serve as receivers without architectural modification
  • Hub-and-spoke alignment still requires training per model family (O(N) spokes), so cold-start cost exists for new architectures
  • Distillation fidelity depends on the visual channel’s capacity; very long or highly structured reasoning traces may exceed what visual token budgets can represent
  • Controlled wall-clock comparisons may not reflect real deployment variance (batching, KV-cache effects, hardware heterogeneity)

Internal Use Cases:

  • Multi-stage ranking pipelines: Pass dense retrieval signals from an embedding model directly into a VLM reranker’s visual pathway rather than serializing candidate features as text, reducing reranking latency
  • LLM cascade architectures: In a cheap-model-first / expensive-model-fallback setup, encode the small model’s partial reasoning state visually and inject it into the large model, avoiding redundant prefill computation
  • Ensemble reasoning: Aggregate reasoning traces from N specialized agents (e.g., one per content domain) into a hub embedding and inject into a single arbitrator VLM, replacing expensive concatenated text prompts
