Masked Diffusion Generative Recommendation

Masked Diffusion Generative Recommendation

Source: arXiv:cs.IR Published: 2026-01-27 Rank: #6 (score: 0.92, max_sim: 0.87) Reading time: ~0 min Tags: Uncategorized

Abstract

Generative recommendation (GR) typically first quantizes continuous item embeddings into multi-level semantic IDs (SIDs), and then generates the next item via autoregressive decoding. Although existing methods are already competitive in terms of recommendation performance, directly inheriting the autoregressive decoding paradigm from language models still suffers from three key limitations: (1) autoregressive decoding struggles to jointly capture global dependencies among the multi-dimensional features associated with different positions of SID; (2) using a unified, fixed decoding path for the same item implicitly assumes that all users attend to item attributes in the same order; (3) autoregressive decoding is inefficient at inference time and struggles to meet real-time requirements. To tackle these challenges, we propose MDGR, a Masked Diffusion Generative Recommendation framework that reshapes the GR pipeline from three perspectives: codebook, training, and inference. (1) We adopt a parallel codebook to provide a structural foundation for diffusion-based GR. (2) During training, we adaptively construct masking supervision signals along both the temporal and sample dimensions. (3) During inference, we develop a warm-up-based two-stage parallel decoding strategy for efficient generation of SIDs. Extensive experiments on multiple public and industrial-scale datasets show that MDGR outperforms ten state-of-the-art baselines by up to 10.78%. Furthermore, by deploying MDGR on a large-scale online advertising platform, we achieve a 1.20% increase in revenue, demonstrating its practical value.

Key Techniques

  • Parallel codebook for semantic IDs
  • Adaptive masking along temporal and sample dimensions
  • Warm-up-based two-stage parallel decoding
  • Masked diffusion for generative recommendation

Adaptive Masking Supervision (Training Strategy)

Core Idea: Construct masking supervision signals adaptively along both temporal and sample dimensions during training to guide diffusion model learning without enforcing fixed decoding orders.

Problem It Solves: Standard autoregressive training fixes a single decoding path per item, implicitly assuming uniform user attention patterns. This prevents the model from learning flexible, user-specific generation orders.

How It Works:

  1. Temporal masking dimension: Vary which diffusion timesteps mask which SID positions; create multiple masking schedules for the same item
  2. Sample masking dimension: Across the batch, apply different masking patterns to the same item at the same timestep, allowing the model to learn multiple valid generation paths
  3. Generate masked supervision signals M ∈ [0,1]^{k×T} where M[i,t] indicates whether SID position i is masked at diffusion step t
  4. Train the diffusion model to predict unmasked SIDs given masked inputs and noise level, without enforcing a specific unmasking order
  5. Learn that different positions can be revealed in different orders depending on user context

Where It’s Useful:

  • Training diffusion models for discrete sequence generation
  • Recommendation systems where user preferences vary in attribute importance
  • Multi-dimensional code generation with flexible orderings

Limitations / Caveats:

  • Requires careful design of masking pattern sampling to ensure coverage of meaningful orderings
  • Increased training complexity compared to fixed autoregressive supervision
  • May require more training data to learn all valid decoding paths

Warm-up-Based Two-Stage Parallel Decoding

Core Idea: Use a two-stage inference strategy with warm-up initialization to efficiently generate all SID positions in parallel, eliminating sequential generation latency.

Problem It Solves: Autoregressive decoding requires T sequential forward passes (where T = number of SID positions), creating unacceptable inference latency for real-time recommendation systems. This prevents deployment in latency-sensitive online scenarios.

How It Works:

  1. Warm-up Stage: Initialize all SID positions with a rough estimate (e.g., mask tokens or position embeddings), creating initial noisy codes s⁽ᵀ⁾ = {[MASK], [MASK], ..., [MASK]}
  2. Parallel Decoding Stage: Iteratively refine all k SID positions simultaneously using the diffusion model’s reverse process:
    • At each diffusion step t: predict all unmasked positions in parallel given current noisy SIDs and timestep
    • Update all k positions jointly: s⁽ᵗ⁻¹⁾ = denoise(s⁽ᵗ⁾, t, context)
    • Continue for T_decode steps until convergence
  3. Output final SID vector s = [s₁*, s₂*, ..., sₖ*] ready for item retrieval
  4. Total inference passes: T_decode (typically T_decode ≪ k), vs. k passes in autoregressive approach

Architecture:

flowchart LR
    A["User Context"] --> B["Warm-up: Init Masked SIDs"]
    B --> C["t=T: Add Noise"]
    C --> D["Stage 1: Denoising Step"]
    D --> E{"t > 0?"}
    E -->|Yes| F["Predict All Positions in Parallel"]
    F --> D
    E -->|No| G["Final SID Vector"]
    G --> H["Item Retrieval"]

Where It’s Useful:

  • Real-time recommendation systems with latency constraints
  • Large-scale online advertising platforms (as demonstrated in the paper: 1.20% revenue increase)
  • Any sequential generation task requiring sub-100ms inference latency

Limitations / Caveats:

  • Requires pre-trained diffusion model; cannot use in cold-start without warm-up strategy
  • Parallel generation may produce slightly lower quality than sequential refinement in some domains
  • Warm-up initialization quality impacts final result; needs careful tuning
  • T_decode hyperparameter balances quality vs. latency; requires empirical tuning per deployment

Integration: These three techniques work together—the parallel codebook enables multi-position generation, adaptive masking trains the model to handle flexible orderings, and parallel decoding leverages this flexibility for efficient inference.
