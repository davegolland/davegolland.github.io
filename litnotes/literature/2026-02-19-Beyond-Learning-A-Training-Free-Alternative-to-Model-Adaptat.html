Beyond Learning: A Training-Free Alternative to Model Adaptation

Source: arXiv:cs.CL Published: 2026-02-18 Rank: #2 (score: 0.99, max_sim: 0.83) Reading time: ~20 min Tags: Search, LLM

Abstract

Despite the continuous research and evolution of language models, they sometimes underperform previous versions. Existing approaches to overcome these challenges are resource-intensive, highlighting the need for alternatives that enable immediate action. We assume that each language model has a local module inside that is suitable for a specific function. First, this work identifies a set of modules showing consistent and local activation changes under an inference workload through activation-based analysis. Subsequently, we transplant an internal module that is properly activated for a specific task into the target model, leading to immediate and measurable functional changes without additional training or fine-tuning. To experimentally demonstrate the effectiveness of the transplant technique, we quantify the relationship between transplant strength and performance improvement under different conditions for two language models. In the cross-generation setting, we find that transplanting activation-selected modules can substantially improve the underperforming model, reaching up to twice the target baseline and achieving gap-based recovery above 100%. Moreover, in transplant experiments between a base model and its instruction-tuned counterpart, transplantation improves the underperforming model toward the stronger baseline, yielding up to about 2.33 times the target baseline with gap-based recovery reaching up to 100% in the best case. These results show that meaningful capacity transfer can be realized through the implantation of highly localized modules implied by language models. Overall, this work provides empirical evidence for task-localized modularity in language models and presents a new research area: model transplantation.

Key Techniques

  • Activation-based module identification
  • Internal module transplantation
  • Cross-generation transplantation
  • Transplant strength quantification

Model Transplantation

Core Idea: Selectively identify task-localized modules in a donor language model via activation analysis, then directly implant those modules into a target (underperforming) model to transfer functional capacity without any training.

Problem It Solves: Model adaptation methods (fine-tuning, LoRA, RLHF) are resource-intensive and require time before deployment; newer model versions sometimes regress on specific tasks compared to predecessors, with no lightweight remedy.

How It Works:

  1. Activation Profiling: Run inference workloads on both donor and target models; record per-module activation patterns to identify modules exhibiting consistent, localized activation changes correlated with the target task.
  2. Module Selection: Rank modules by activation-based criteria to select the minimal set that is reliably “lit up” for the specific function in the donor model.
  3. Transplantation: Directly copy selected donor modules (weight tensors) into the corresponding positions in the target model at a tunable transplant strength (interpolation coefficient between donor and target weights), then evaluate immediately—no gradient updates required.

Architecture:

flowchart LR
    A[Donor Model] --> B[Activation Profiler]
    B --> C[Module Selector\ntask-localized]
    C --> D[Transplant Op\nstrength α]
    E[Target Model] --> D
    D --> F[Augmented Target\nno retraining]

Where It’s Useful:

  • Rapid recovery when a newer model version regresses on a specific benchmark vs. a prior version (cross-generation setting)
  • Transferring instruction-following capability from an instruction-tuned model into its base counterpart without fine-tuning
  • Low-resource or latency-constrained deployments where retraining is infeasible

Limitations / Caveats:

  • Assumes structural compatibility (matching architecture) between donor and target; cross-architecture transplants are not demonstrated
  • Transplant strength is a scalar hyperparameter requiring empirical tuning per task—wrong values can degrade performance
  • Activation-based module selection may not generalize cleanly to tasks with diffuse, non-localized activation patterns

Internal Use Cases:

  • Ranking/RecSys LLMs: If a production ranking model regresses after a version update, transplant task-relevant modules from the prior version to recover performance instantly without redeployment cycles
  • MultiLoRA integration: Rather than training new LoRA adapters for each task, use transplanted modules as warm initialization targets to reduce adapter training cost
  • Instruction-following in base models: Transplant instruction-tuning modules into a base model serving as a retrieval or reranking backbone to improve prompt adherence without full RLHF
