Source: arXiv:cs.LG Published: 2026-01-30 Rank: #7 (score: 0.98, max_sim: 0.83) Reading time: ~20 min Tags: LLM, RAG Code: https://github.com/amahankali10/DC_CoT_RL_for_Low_Latency_CoT_with_Parallel_Reasoning

Abstract

Long chain-of-thought reasoning (Long CoT) is now fundamental to state-of-the-art LLMs, especially in mathematical reasoning. However, LLM generation is highly sequential, and long CoTs lead to a high latency. We propose to train Divide-and-Conquer CoT (DC-CoT) to reduce the latency. With DC-CoT, the model can act as a director that identifies distinct subtasks that can be performed in parallel in its reasoning process, and then spawns workers to execute the subtasks. Our goal is to achieve high accuracy, with a low longest path length, which is a theoretical measure of the latency needed for the response. We start with a long CoT base model (DeepScaleR-1.5B-Preview), and first use SFT with a small curated demonstration set to initialize its ability to spawn workers in a certain format. Because SFT degrades the accuracy significantly, we design a multi-stage RL algorithm, with various data filtering strategies, to recover the accuracy while decreasing the longest path length. Across several benchmarks including AIME 2024 and HMMT 2025, DC-CoT achieves similar accuracy as DeepScaleR-1.5B-Preview while decreasing longest path length by 35-40%. Our code, SFT dataset and models are publicly available at https://github.com/amahankali10/DC_CoT_RL_for_Low_Latency_CoT_with_Parallel_Reasoning.

Multi-Stage RL Algorithm with Data Filtering

Core Idea: Use reinforcement learning with progressive data filtering strategies across multiple stages to recover the accuracy degraded by initial supervised fine-tuning while simultaneously reducing the longest path length of DC-CoT reasoning.

Problem It Solves: Supervised fine-tuning (SFT) on the DC-CoT spawning format causes significant accuracy degradation compared to the base long-CoT model. A straightforward RL approach needs strategic data curation to maintain high accuracy while learning the parallelization objective.

How It Works:

  1. Initialize with SFT on a small curated demonstration set to establish the worker-spawning format (even though accuracy drops)
  2. Design stage-based RL training where each stage has specific data filtering criteria
  3. Apply filtering strategies (e.g., only use trajectories meeting accuracy thresholds, filtering by path length improvements) to construct the RL training distribution
  4. Progressively relax or adjust filtering criteria across stages to recover accuracy while maintaining path length reduction
  5. Track dual objectives: maintain/improve accuracy from base model while minimizing longest path length

Where It’s Useful:

  • Fine-tuning pre-trained models when format changes cause performance drops
  • Multi-objective optimization in RL where competing goals (accuracy vs. efficiency) require staged learning
  • Domain-specific reasoning tasks requiring controlled recovery of base model capabilities

Limitations / Caveats:

  • Requires careful design of filtering strategies; incorrect filtering can lead to convergence to poor solutions
  • Multiple stages add engineering complexity compared to single-phase RL
  • No guarantee that accuracy will be fully recovered; may represent accuracy-latency tradeoff
  • Heavily dependent on quality of the initial curated SFT demonstration set

Internal Use Cases:

  • Training ranking models to optimize multiple metrics (relevance + efficiency) sequentially
  • Fine-tuning retrieval systems where format changes (e.g., adding structured outputs) hurt base performance
  • LLM-based recommendation systems balancing response quality with inference speed constraints
