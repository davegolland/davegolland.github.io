Abstract

While LLaDA2.0 showcased the scaling potential of 100B-level block-diffusion models and their inherent parallelization, the delicate equilibrium between decoding speed and generation quality has remained an elusive frontier. Today, we unveil LLaDA2.1, a paradigm shift designed to transcend this trade-off. By seamlessly weaving Token-to-Token (T2T) editing into the conventional Mask-to-Token (M2T) scheme, we introduce a joint, configurable threshold-decoding scheme. This structural innovation gives rise to two distinct personas: the Speedy Mode (S Mode), which audaciously lowers the M2T threshold to bypass traditional constraints while relying on T2T to refine the output; and the Quality Mode (Q Mode), which leans into conservative thresholds to secure superior benchmark performances with manageable efficiency degrade. Furthering this evolution, underpinned by an expansive context window, we implement the first large-scale Reinforcement Learning (RL) framework specifically tailored for dLLMs, anchored by specialized techniques for stable gradient estimation. This alignment not only sharpens reasoning precision but also elevates instruction-following fidelity, bridging the chasm between diffusion dynamics and complex human intent. We culminate this work by releasing LLaDA2.1-Mini (16B) and LLaDA2.1-Flash (100B). Across 33 rigorous benchmarks, LLaDA2.1 delivers strong task performance and lightning-fast decoding speed. Despite its 100B volume, on coding tasks it attains an astounding 892 TPS on HumanEval+, 801 TPS on BigCodeBench, and 663 TPS on LiveCodeBench.

LLaDA2.1: Joint Mask-to-Token and Token-to-Token Diffusion Editing

Core Idea: Combine Mask-to-Token (M2T) prediction with Token-to-Token (T2T) editing in a configurable threshold-based decoding scheme to enable speed-quality tradeoffs in diffusion language models.

Problem It Solves: Traditional diffusion language models face a rigid tradeoff between decoding speed and generation quality. Lowering confidence thresholds accelerates decoding but degrades output; conservative thresholds maintain quality but sacrifice speed.

How It Works:

  1. M2T Phase: Predict tokens for masked positions using the diffusion model; accept predictions that exceed a configurable confidence threshold
  2. T2T Editing Phase: For tokens that fall below threshold (or in aggressive “Speedy Mode”), apply token-to-token editing to refine predictions without full regeneration
  3. Threshold Configuration: Adjust M2T threshold to create personas—“Speedy Mode” (low threshold, heavy T2T reliance) or “Quality Mode” (high threshold, minimal T2T refinement)
  4. RL Alignment: Apply reinforcement learning with specialized gradient estimation techniques to align diffusion dynamics with instruction-following and reasoning objectives

Architecture:

flowchart LR
    A[Masked Input] --> B[M2T Prediction]
    B --> C{Confidence > Threshold?}
    C -->|Yes| D[Accept Token]
    C -->|No| E[T2T Editing]
    E --> F[Refined Token]
    D --> G[Output Sequence]
    F --> G

Where It’s Useful:

  • Code generation tasks requiring high throughput (achieves 663-892 TPS on coding benchmarks)
  • Interactive applications where speed-quality tradeoffs must be dynamically adjusted
  • Large-scale inference where 100B parameter models need practical deployment speeds

Limitations / Caveats:

  • T2T editing effectiveness depends on quality of initial M2T predictions; catastrophic failures in M2T may not be recoverable
  • RL alignment for diffusion models requires specialized gradient estimation—training complexity increases
  • Threshold tuning introduces hyperparameter sensitivity; optimal values may vary across task domains

Internal Use Cases:

  • Code completion systems: Deploy in “Speedy Mode” for autocomplete, “Quality Mode” for full function generation
  • LLM serving infrastructure: Dynamically adjust thresholds based on load—aggressive editing during peak traffic, conservative during low-demand periods
  • Retrieval-augmented generation: Use T2T editing to refine retrieved candidates rather than full regeneration, reducing latency in document-grounded QA
