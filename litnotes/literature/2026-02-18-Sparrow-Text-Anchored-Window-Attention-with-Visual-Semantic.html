Source: arXiv:cs.AI Published: 2026-02-17 Rank: #1 (score: 0.99, max_sim: 0.82) Reading time: ~20 min Tags: LLM, RAG, Block-Sparse

Abstract

Although speculative decoding is widely used to accelerate Vision-Language Models (VLMs) inference, it faces severe performance collapse when applied to Video Large Language Models (Vid-LLMs). The draft model typically falls into the trap of attention dilution and negative visual gain due to key-value cache explosion and context window mismatches. We observe a visual semantic internalization phenomenon in Vid-LLMs, indicating that critical visual semantics are implicitly encoded into text hidden states during deep-layer interactions, which renders raw visual inputs structurally redundant during deep inference. To address this, we propose the Sparrow framework, which first utilizes visually-aware text-anchored window attention via hidden state reuse to fully offload visual computation to the target model, and leverages intermediate-layer visual state bridging to train the draft model with semantic-rich intermediate states, thereby filtering out low-level visual noise. Additionally, a multi-token prediction strategy is introduced to bridge the training-inference distribution shift. Experiments show that Sparrow achieves an average speedup of 2.82x even with 25k visual tokens, effectively resolving the performance degradation in long sequences and offering a practical solution for real-time long video tasks.

Sparrow: Text-Anchored Window Attention with Visual-Semantic Glimpsing

Core Idea: A speculative decoding framework for Video LLMs that offloads visual computation entirely to the target model by reusing cached text hidden states in a sliding window attention mechanism, while training the draft model on semantically-enriched intermediate layer states instead of raw visual tokens.

Problem It Solves: Standard speculative decoding collapses on Video LLMs because the draft model suffers from (1) attention dilution — visual KV-cache grows proportionally with video length, drowning out relevant signal — and (2) negative visual gain — processing raw visual tokens in the draft model adds noise rather than useful signal, since by deep inference layers the visual semantics have already been absorbed into text hidden states.

How It Works:

  1. Identify visual semantic internalization: Empirically confirm that deep-layer text hidden states already encode critical visual semantics, making raw visual token inputs to the draft model structurally redundant.
  2. Text-anchored window attention (TAWA): The draft model attends only over a sliding window of recent text tokens using cached hidden states (bypassing raw visual tokens entirely), effectively routing all visual computation to the target model’s full attention pass.
  3. Intermediate-layer visual state bridging (glimpsing): During draft model training, inject semantic-rich hidden states from intermediate layers of the target model as supervision signal, filtering low-level visual noise and aligning the draft model’s internal representations.
  4. Multi-token prediction (MTP): Train the draft model to predict multiple future tokens per step, closing the distribution gap between training (teacher-forced) and inference (autoregressive speculation) modes.

Architecture:

flowchart LR
    V[Video Frames] --> T[Target Model\nFull Attention]
    T -->|Intermediate hidden states| B[Visual State Bridge\nGlimpsing]
    B -->|Semantic supervision| D[Draft Model\nTAWA Sliding Window]
    D -->|Candidate token sequences| V2[Verification\nby Target Model]
    V2 -->|Accept / reject| O[Final Output Tokens]
    T --> V2

Where It’s Useful:

  • Real-time long-video understanding (25k+ visual tokens) where standard speculative decoding degrades
  • Any Vid-LLM inference pipeline with KV-cache pressure from dense visual token sequences
  • Streaming video captioning, video QA, and temporal reasoning tasks requiring low-latency generation

Limitations / Caveats:

  • Requires access to intermediate target model activations at training time, coupling draft model training tightly to a specific target architecture
  • Window size in TAWA is a fixed hyperparameter — may need tuning per task or video length regime
  • Speedup gains are measured against a specific draft/target pairing; generalization to arbitrary model pairs is unverified

Internal Use Cases:

  • Long-context ranking/retrieval: The hidden-state-reuse principle applies when a lightweight scorer needs to re-rank items conditioned on a long multimodal context — offload heavy modality encoding to a single full-pass model, reuse its states for fast candidate scoring
  • LLM serving with multimodal inputs: Any system processing high-token-count visual inputs (product images, ad creatives, video ads) can use TAWA-style draft models to reduce per-request latency without sacrificing accuracy on the full model verification pass
  • Speculative decoding for dense retrieval re-rankers: The intermediate-layer bridging idea generalizes to training lightweight re-rankers supervised on deep representations from a large cross-encoder, preserving semantic richness while cutting inference cost
