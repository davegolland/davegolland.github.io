FOCUS: DLLMs Know How to Tame Their Compute Bound

Source: arXiv:cs.CL Published: 2026-01-30 Rank: #2 (score: 1.00, max_sim: 0.82) Reading time: ~15 min Tags: LLM, RAG, Diffusion Code: https://github.com/sands-lab/FOCUS

Abstract

Diffusion Large Language Models (DLLMs) offer a compelling alternative to Auto-Regressive models, but their deployment is constrained by high decoding cost. In this work, we identify a key inefficiency in DLLM decoding: while computation is parallelized over token blocks, only a small subset of tokens is decodable at each diffusion step, causing most compute to be wasted on non-decodable tokens. We further observe a strong correlation between attention-derived token importance and token-wise decoding probability. Based on this insight, we propose FOCUS — an inference system designed for DLLMs. By dynamically focusing computation on decodable tokens and evicting non-decodable ones on-the-fly, FOCUS increases the effective batch size, alleviating compute limitations and enabling scalable throughput. Empirical evaluations demonstrate that FOCUS achieves up to 3.52 throughput improvement over the production-grade engine LMDeploy, while preserving or improving generation quality across multiple benchmarks. The FOCUS system is publicly available on GitHub: https://github.com/sands-lab/FOCUS.

Attention-Derived Token Importance Scoring

Core Idea: Leverage attention weights across DLLM layers to estimate token-wise decoding probability, enabling data-driven selection of which tokens are worth computing in each diffusion step.

Problem It Solves: DLLM decoders lack explicit signals for which tokens will decode at each step. Without principled scoring, systems must either compute all tokens (wasteful) or use heuristics (unreliable). Attention provides a learned, model-inherent signal of token importance.

  4. Use importance scores as proxy for token decoding probability in downstream filtering decisions

Where It’s Useful:

  • Token selection in any attention-based LLM architecture
  • Adaptive computation systems that prioritize high-impact tokens
  • Uncertainty quantification for token-level predictions
  • Interpretability work to identify which tokens matter most

Limitations / Caveats:

  • Correlation between attention and true decodability is empirical; may not generalize across all DLLM architectures or datasets
  • Attention weights can be noisy, especially in middle layers; aggregation strategy impacts quality
  • Doesn’t account for dependencies between tokens—focusing on high-importance token may inadvertently hurt neighbors

Internal Use Cases:

  • Ranking systems could use attention-based relevance scoring to prioritize high-confidence items for deeper processing
  • LLM serving could apply per-token importance weighting for early-exit strategies or token-level beam search
