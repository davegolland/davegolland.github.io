Fat-Cat: Document-Driven Metacognitive Multi-Agent System for Complex Reasoning

Source: arXiv:cs.LG Published: 2026-02-02 Rank: #8 (score: 0.99, max_sim: 0.83) Reading time: ~20 min Tags: LLM, RAG Code: https://github.com/answeryt/Fat-Cat

Abstract

The effectiveness of LLM-based agents is often limited not by model capacity alone, but by how efficiently contextual information is utilized at runtime. Existing agent frameworks rely on rigid, syntax-heavy state representations such as nested JSON, which require models to devote a substantial portion of their limited attention to syntactic processing rather than semantic reasoning. In this paper, we propose Fat-Cat, a document-driven agent architecture that improves the signal-to-noise ratio of state management. By integrating three key components: (1) a Semantic File System that represents agent state as Markdown documents aligned with common pre-training corpora, (2) a Textual Strategy Evolution module that accumulates task-solving knowledge without parameter updates, and (3) a Closed-Loop Watcher that monitors reasoning trajectories to reduce hallucinations. Extensive reasoning, retrieval, and coding benchmarks, Fat-Cat consistently improves agent performance. It enables the Kimi-k2 model to outperform the proprietary GPT-4o baseline on HotPotQA. Replacing the document-based state with JSON leads to performance drop, while empirically validating the critical necessity of document-driven state modeling over rigid syntax. The code is available at https://github.com/answeryt/Fat-Cat.

Textual Strategy Evolution (TSE)

Core Idea: Accumulate task-solving knowledge and heuristics as evolving Markdown documents without modifying model parameters, enabling continuous improvement of agent behavior across task instances.

Problem It Solves: Traditional RL/fine-tuning approaches require gradient updates and training loops. TSE addresses how agents can learn and improve strategies through experience while remaining frozen—critical for deployed models where parameter updates are infeasible.

How It Works:

  1. As the agent solves tasks, successful reasoning patterns and heuristics are documented in a Strategy Document (e.g., “For retrieval tasks, search breadth-first before depth-first”)
  2. When encountering new tasks of similar type, the agent retrieves relevant strategies from the accumulated knowledge
  3. Strategies are refined iteratively: new effective approaches are added, failed strategies are marked as context-dependent or deprecated
  4. No gradient-based learning occurs; knowledge is purely linguistic/symbolic, integrated via prompting

Where It’s Useful:

  • Zero-shot to few-shot improvement on task families
  • Multi-turn problem solving where patterns repeat
  • Domain-specific agent customization without retraining
  • Live system adaptation based on observed failure modes

Limitations / Caveats:

  • Strategy documentation quality depends on the agent’s ability to introspect and articulate reasoning
  • Scaling to many strategies may increase context length; retrieval/filtering mechanisms needed
  • No formal mechanism to prevent strategy interference or conflicts
  • Strategies are heuristic; no guarantees of correctness or convergence

Internal Use Cases:

  • Recommendation rankers learning what ranking features correlate with user engagement
  • Query expansion strategies in retrieval systems
  • A/B testing summary: codifying which ranking signals outperform in different user segments

Closed-Loop Watcher (CLW)

Core Idea: Monitor agent reasoning trajectories in real-time to detect and mitigate hallucinations by identifying when outputs diverge from grounded reasoning patterns.

Problem It Solves: LLM agents are prone to hallucination—generating confident but false intermediate results that propagate through downstream reasoning. Existing systems lack runtime monitoring to catch these failures before they compromise final outputs.

How It Works:

  1. As the agent generates reasoning steps (e.g., intermediate retrieved facts, computation results), the Watcher compares outputs against a grounding signal (retrieved documents, previous validated facts, external tools)
  2. Discrepancies trigger a flag (e.g., “claimed fact X not found in retrieved context”)
  3. The agent is prompted to either: (a) recover the grounding information, (b) revise the claim, or (c) explicitly mark it as speculative
  4. Reasoning trajectory is annotated with confidence/grounding levels, preventing low-confidence outputs from propagating

Where It’s Useful:

  • Retrieval-augmented generation (RAG) systems where factuality is critical
  • Reasoning tasks (HotPotQA) requiring multi-hop validation
  • Code generation and execution where syntax errors/runtime errors must be caught
  • Any domain where false intermediate steps accumulate and compound errors

Limitations / Caveats:

  • Requires a ground-truth signal for comparison (retrieved docs, tool outputs)—not applicable to pure generation tasks
  • Adds computational cost via continuous validation loops
  • False positive corrections may misdirect the agent; threshold tuning needed
  • Effectiveness depends on how well grounding signals align with task correctness

Internal Use Cases:

  • Recommendation ranking: flagging predictions where score diverges from item embedding similarity
  • LLM-based retrieval: verify retrieved passages actually contain answers before reranking
  • Conversational agents: cross-check user fact claims against knowledge base before responding

Fat-Cat’s Extension of Agent Architecture Paradigms

Fat-Cat fundamentally reframes the state management problem in agent systems, challenging a core assumption embedded in both the Agentic Reasoning Pipeline and Multi-Agent Agentic RAG frameworks. While these existing approaches focus on how agents delegate tasks and coordinate retrieval across specialists, Fat-Cat identifies a deeper bottleneck: the syntactic overhead of rigid state representations. By replacing JSON-based state with Markdown documents aligned to pre-training corpora, Fat-Cat extends the Mind-Map Agent concept’s intuition that agents benefit from naturally-structured, query-friendly representations. However, where the Mind-Map Agent constructs dynamically-updated knowledge graphs during reasoning, Fat-Cat takes a more pragmatic approach—leveraging the fact that models are already trained to reason fluently with natural language documents rather than formal specifications. This shift from syntax-heavy to semantic-first state modeling represents a critical perspective: agent performance ceilings may be determined less by reasoning capacity than by how much model attention is wasted on parsing rather than thinking.

Bridging Textual Strategy Evolution and Persistent Memory

Fat-Cat’s Textual Strategy Evolution module directly parallels the persistent memory concept in the Mind-Map Agent, but operates without graph construction overhead. Rather than maintaining an explicit knowledge graph with community detection and clustering, Fat-Cat accumulates task-solving knowledge as natural-language strategy documents—essentially creating a self-updating textbook of solutions without requiring parameter fine-tuning. This is particularly novel in contrast to the Multi-Agent Agentic RAG framework, which emphasizes horizontal scaling through specialization: Fat-Cat proposes vertical scaling through accumulated semantic memory. The empirical validation that replacing Markdown with JSON causes performance degradation provides quantitative evidence that the presentation layer of agent memory is not merely ergonomic but computationally consequential—a finding that should reshape how Agentic Reasoning Pipeline designers structure their state serialization.

New Contribution: Closed-Loop Hallucination Reduction

Fat-Cat introduces a novel Closed-Loop Watcher mechanism that directly addresses a shared vulnerability across all three matched frameworks: the tendency for reasoning trajectories to diverge into hallucination. While the Agentic Reasoning Pipeline mitigates this through external grounding (web search, code execution) and the Mind-Map Agent through explicit relational consistency checking, Fat-Cat proposes real-time trajectory monitoring. This extends beyond error detection to enable active course-correction—a metacognitive layer that treats the agent’s own reasoning chain as a queryable artifact. Combined with document-driven state, this creates a feedback loop where semantic clarity naturally constrains hallucination, bypassing the need for explicit formal verification. The practical result—enabling Kimi-k2 to exceed GPT-4o on HotPotQA—suggests that structured environmental representations may be more leverageable than raw model scale, a lesson particularly salient for resource-constrained deployments building on the Multi-Agent Agentic RAG paradigm.
