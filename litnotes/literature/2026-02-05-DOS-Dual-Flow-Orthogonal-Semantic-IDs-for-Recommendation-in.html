Abstract

Semantic IDs serve as a key component in generative recommendation systems. They not only incorporate open-world knowledge from large language models (LLMs) but also compress the semantic space to reduce generation difficulty. However, existing methods suffer from two major limitations: (1) the lack of contextual awareness in generation tasks leads to a gap between the Semantic ID codebook space and the generation space, resulting in suboptimal recommendations; and (2) suboptimal quantization methods exacerbate semantic loss in LLMs. To address these issues, we propose Dual-Flow Orthogonal Semantic IDs (DOS) method. Specifically, DOS employs a user-item dual flow-framework that leverages collaborative signals to align the Semantic ID codebook space with the generation space. Furthermore, we introduce an orthogonal residual quantization scheme that rotates the semantic space to an appropriate orientation, thereby maximizing semantic preservation. Extensive offline experiments and online A/B testing demonstrate the effectiveness of DOS. The proposed method has been successfully deployed in Meituan’s mobile application, serving hundreds of millions of users.

Key Techniques

  • Dual-flow framework for Semantic IDs
  • Orthogonal residual quantization scheme
  • Collaborative signals alignment

Dual-Flow Orthogonal Semantic IDs (DOS)

Core Idea: A dual-flow framework that uses collaborative signals from both user and item representations to align semantic ID codebook space with generation space, combined with orthogonal residual quantization to minimize semantic loss during ID compression.

Problem It Solves: Existing semantic ID methods suffer from (1) lack of contextual awareness causing misalignment between the codebook space and generation space, and (2) suboptimal quantization methods that lose semantic information when compressing LLM embeddings into discrete IDs.

How It Works:

  1. Dual-Flow Architecture: Simultaneously encode user historical interactions and candidate items through separate flows, generating user representations and item semantic IDs that incorporate collaborative filtering signals rather than pure content embeddings
  2. Orthogonal Residual Quantization: Apply rotation transformations to the semantic embedding space before residual vector quantization (RQ-VAE), orienting vectors to maximize orthogonality between codebook entries and minimize reconstruction error
  3. Context-Aware Codebook Learning: Train the semantic ID codebook jointly with user-item interaction data, ensuring generated IDs are optimized for the downstream recommendation generation task rather than pure semantic reconstruction

Architecture:

flowchart LR
    U[User History] --> UE[User Encoder]
    I[Item Features] --> IE[Item Encoder]
    UE --> CS[Collaborative Signals]
    IE --> CS
    CS --> OQ[Orthogonal Rotation]
    OQ --> RQ[Residual Quantization]
    RQ --> SID[Semantic IDs]
    UE --> GEN[LLM Generator]
    SID --> GEN
    GEN --> REC[Recommendations]

Where It’s Useful:

  • Large-scale industrial recommendation systems requiring semantic item representations (e.g., food delivery, e-commerce)
  • Generative recommendation models that need to compress items into discrete tokens for LLM-based generation
  • Systems requiring interpretable item IDs that preserve both semantic and collaborative information
  • Cold-start scenarios where semantic similarity helps with new item recommendations

Limitations / Caveats:

  • Requires joint training of user/item encoders with the quantization codebook, increasing training complexity
  • Orthogonal rotation adds computational overhead during encoding
  • Effectiveness may depend on sufficient collaborative signal density (user-item interactions)
  • Codebook size trades off between semantic expressiveness and generation difficulty

Internal Use Cases:

  • Feed Ranking: Generate semantic IDs for content items that encode both content features and user engagement patterns, enabling LLM-based ranking models to better capture user preferences
  • Sequential Recommendation: Use dual-flow architecture to create context-aware item IDs that understand position in user session sequences
  • Cross-Domain Transfer: Orthogonal quantization could improve semantic ID stability when transferring learned representations across different recommendation domains (e.g., reels → stories)
  • Vocabulary Efficiency: Apply to constrained decoding scenarios where minimizing codebook size while preserving semantics is critical for generation latency
