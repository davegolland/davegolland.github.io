Abstract

Elastic precision quantization enables multi-bit deployment via a single optimization pass, fitting diverse quantization scenarios.Yet, the high storage and optimization costs associated with the Transformer architecture, research on elastic quantization remains limited, particularly for large language models.This paper proposes QuEPT, an efficient post-training scheme that reconstructs block-wise multi-bit errors with one-shot calibration on a small data slice. It can dynamically adapt to various predefined bit-widths by cascading different low-rank adapters, and supports real-time switching between uniform quantization and mixed precision quantization without repeated optimization. To enhance accuracy and robustness, we introduce Multi-Bit Token Merging (MB-ToMe) to dynamically fuse token features across different bit-widths, improving robustness during bit-width switching. Additionally, we propose Multi-Bit Cascaded Low-Rank adapters (MB-CLoRA) to strengthen correlations between bit-width groups, further improve the overall performance of QuEPT. Extensive experiments demonstrate that QuEPT achieves comparable or better performance to existing state-of-the-art post-training quantization methods.Our code is available at https://github.com/xuke225/QuEPT

QuEPT (Quantized Elastic Precision Transformers)

Core Idea: Enable dynamic multi-bit quantization switching in Transformers through one-shot post-training calibration using cascaded low-rank adapters, eliminating the need for separate optimization per bit-width.

Problem It Solves: Traditional quantization methods require retraining or re-optimization for each target bit-width (2-bit, 3-bit, 4-bit, etc.), making it expensive to deploy models across diverse hardware constraints. Elastic precision quantization has remained underexplored for large language models due to high storage and optimization costs.

How It Works:

  1. Block-wise Multi-Bit Error Reconstruction: Perform one-shot calibration on a small data slice to reconstruct quantization errors across multiple bit-widths simultaneously at the block level
  2. Multi-Bit Cascaded Low-Rank Adapters (MB-CLoRA): Deploy cascaded LoRA modules where each adapter corresponds to a specific bit-width group, strengthening correlations between different quantization levels
  3. Multi-Bit Token Merging (MB-ToMe): Dynamically fuse token features across different bit-width representations to improve robustness during runtime bit-width switching
  4. Runtime Switching: Select appropriate cascaded adapters to switch between uniform quantization (single bit-width) and mixed precision quantization (layer-specific bit-widths) without re-optimization

Architecture:

flowchart LR
    A[Input Tokens] --> B[Quantized Transformer Block]
    B --> C[MB-ToMe<br/>Token Fusion]
    C --> D[MB-CLoRA<br/>Bit-Width Adapters]
    D --> E{Bit-Width<br/>Selection}
    E -->|2-bit| F[2-bit Output]
    E -->|3-bit| G[3-bit Output]
    E -->|4-bit| H[4-bit Output]

Where It’s Useful:

  • Deploying LLMs across heterogeneous edge devices with varying memory/compute constraints
  • Dynamic resource allocation in multi-tenant serving systems (allocate higher precision to priority requests)
  • A/B testing different quantization levels without maintaining separate model checkpoints

Limitations / Caveats:

  • Requires predefined set of target bit-widths during calibration (cannot switch to arbitrary bit-widths post-deployment)
  • MB-CLoRA introduces additional storage overhead compared to single bit-width quantization (though still lower than maintaining separate quantized models)
  • Performance at extreme low bit-widths (2-bit) may still degrade compared to precision-specific optimization

Internal Use Cases:

  • Multi-tenant LLM serving: Dynamically adjust model precision based on request priority/SLA without model swapping
  • Ranking model deployment: Switch between high-precision (online ranking) and low-precision (candidate generation) modes in the same model
  • Progressive model loading: Start serving with low-bit weights for fast cold-start, progressively load higher-precision adapters as resources permit
