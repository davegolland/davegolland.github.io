DeltaKV: Residual-Based KV Cache Compression via Long-Range Similarity

Source: arXiv:cs.CL Published: 2026-02-08 Rank: #1 (score: 0.97, max_sim: 0.84) Reading time: ~0 min Tags: Uncategorized

Abstract

The deployment of efficient long-context LLMs in applications like autonomous agents, long-chain reasoning, and creative writing is fundamentally bottlenecked by the linear growth of KV cache memory. Existing compression and eviction methods often struggle to balance accuracy, compression ratio, and hardware efficiency. We propose DeltaKV, a residual-based KV cache compression framework motivated by two empirical findings: long-range inter-token similarity and highly shared latent components in KV representations. Instead of discarding tokens, DeltaKV encodes semantic residuals relative to retrieved historical references, preserving fidelity while substantially reducing storage. To translate compression gains into real system speedups, we further introduce Sparse-vLLM, a high-performance inference engine with decoupled memory management and kernels optimized for sparse and irregular KV layouts. Experiments show that DeltaKV reduces KV cache memory to 29% of the original while maintaining near-lossless accuracy on LongBench, SCBench, and AIME. When integrated with Sparse-vLLM, it achieves up to 2 throughput improvement over vLLM in long-context scenarios, demonstrating a practical path toward scalable long-context LLM deployment. Code, model checkpoints, and datasets are available at https://github.com/CURRENTF/Sparse-vLLM.

DeltaKV: Residual-Based KV Cache Compression

Core Idea: Compress KV cache by storing only semantic residuals relative to similar historical tokens rather than full key-value vectors, exploiting long-range inter-token similarity to reduce memory while preserving accuracy.

Problem It Solves: KV cache memory grows linearly with context length, creating a fundamental bottleneck for long-context LLM inference (100K+ tokens) in applications like autonomous agents and long-chain reasoning. Existing eviction methods sacrifice accuracy, while naive compression lacks hardware efficiency.

How It Works:

  1. Similarity Detection: For each new token’s KV pair, retrieve the most similar historical token from cache using learned similarity metrics
  2. Residual Encoding: Compute and store only the difference (residual) between the new KV vector and the retrieved reference: ΔKV = KV_new - KV_reference
  3. Reference Indexing: Maintain lightweight pointers to reference tokens, exploiting the observation that KV representations share highly similar latent components across long distances
  4. Reconstruction: During attention, reconstruct full KV vectors on-demand: KV_reconstructed = ΔKV + KV_reference

Architecture:

flowchart LR
    A[New Token KV] --> B[Similarity Search]
    C[Historical KV Cache] --> B
    B --> D[Find Best Reference]
    D --> E[Compute Residual]
    E --> F[Store ΔKV + Pointer]
    F --> G[Compressed Cache]
    G --> H[On-demand Reconstruct]
    H --> I[Attention Computation]

Where It’s Useful:

  • Long-context document understanding (legal, medical, technical documents)
  • Autonomous agents with persistent memory across extended conversations
  • Creative writing and story generation requiring coherence over 50K+ tokens
  • Long-chain reasoning tasks (math proofs, multi-hop QA)
  • Code analysis over entire repositories

Limitations / Caveats:

  • Requires training/fine-tuning for optimal similarity metric learning
  • Reconstruction overhead adds latency per token if not properly optimized (hence need for Sparse-vLLM engine)
  • Effectiveness depends on actual inter-token similarity distribution in the workload
  • May underperform on completely random or adversarial sequences where similarity assumptions break down

Internal Use Cases:

  • RecSys context compression: Compress user interaction history embeddings in session-based recommendation by storing residuals relative to archetypal user behavior patterns
  • Ranking model serving: Reduce memory for document encoders in multi-stage ranking by caching only residuals relative to cluster centroids
  • LLM inference optimization: Deploy for internal long-context RAG systems, reducing serving costs for documentation QA and code assistance tools
  • Embedding cache efficiency: Apply residual encoding to frequently-accessed embedding tables in real-time feature stores
