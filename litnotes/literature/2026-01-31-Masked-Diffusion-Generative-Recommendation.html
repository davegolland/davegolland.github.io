Masked Diffusion Generative Recommendation

Masked Diffusion Generative Recommendation

Source: arXiv:cs.IR Published: 2026-01-27 Rank: #6 (score: 0.92, max_sim: 0.87) Reading time: ~0 min Tags: Uncategorized

Abstract

Generative recommendation (GR) typically first quantizes continuous item embeddings into multi-level semantic IDs (SIDs), and then generates the next item via autoregressive decoding. Although existing methods are already competitive in terms of recommendation performance, directly inheriting the autoregressive decoding paradigm from language models still suffers from three key limitations: (1) autoregressive decoding struggles to jointly capture global dependencies among the multi-dimensional features associated with different positions of SID; (2) using a unified, fixed decoding path for the same item implicitly assumes that all users attend to item attributes in the same order; (3) autoregressive decoding is inefficient at inference time and struggles to meet real-time requirements. To tackle these challenges, we propose MDGR, a Masked Diffusion Generative Recommendation framework that reshapes the GR pipeline from three perspectives: codebook, training, and inference. (1) We adopt a parallel codebook to provide a structural foundation for diffusion-based GR. (2) During training, we adaptively construct masking supervision signals along both the temporal and sample dimensions. (3) During inference, we develop a warm-up-based two-stage parallel decoding strategy for efficient generation of SIDs. Extensive experiments on multiple public and industrial-scale datasets show that MDGR outperforms ten state-of-the-art baselines by up to 10.78%. Furthermore, by deploying MDGR on a large-scale online advertising platform, we achieve a 1.20% increase in revenue, demonstrating its practical value.

Key Techniques

  • Parallel codebook
  • Adaptive masking supervision
  • Warm-up-based two-stage parallel decoding
  • Masked diffusion generative framework

Parallel Codebook Design

Core Idea: Decompose the traditional single sequential SID into independent parallel SID sequences (one per feature dimension), enabling simultaneous diffusion across multiple item attribute dimensions.

Problem It Solves:

  • Single sequential SID forces a fixed ordering of feature generation, but different users may attend to different item attributes in different orders
  • Sequential dependencies in traditional SID prevent efficient parallel generation
  • One-dimensional SID representation loses structural information about multi-faceted items

How It Works:

  1. Feature Decomposition: Identify and separate key item dimensions (e.g., category ID, brand ID, price-range ID, rating-tier ID) as independent attribute spaces
  2. Per-Dimension Quantization: Apply separate quantization codebooks to each dimension:
    • Dimension 1: Item category → codebook C₁ of size K₁
    • Dimension 2: Brand → codebook C₂ of size K₂
    • … Dimension d: Attribute d → codebook Cₐ of size Kₐ
  3. Parallel SID Representation: For each item, generate SID tuples: (sid₁, sid₂, …, sidₐ) where sidᵢ ∈ Cᵢ, rather than a single concatenated sequence
  4. Embedding Fusion: Combine embeddings from all dimensions before passing to the diffusion model: combined_embedding = FUSION(emb₁, emb₂, …, embₐ)

Where It’s Useful:

  • Multi-attribute recommendation where items have naturally distinct feature types
  • Product recommendation (color, size, brand, category are independent)
  • Real estate recommendation (location, property type, price, amenities)
  • Any domain with heterogeneous item attributes

Limitations / Caveats:

  • Requires domain knowledge to identify which attributes should be separate dimensions vs. features within a single dimension
  • Naive parallelization loses any inherent dependencies between attributes (e.g., certain brands only available in certain price ranges)
  • Codebook sizes must be chosen carefully; too many dimensions with large vocabularies increases parameter count
  • Fusion function design is non-trivial and domain-specific

Internal Use Cases:

  • Recommendation systems: Replace single SID with multi-dimensional representation for richer item descriptions
  • Language models: Not directly applicable (tokens have intrinsic sequence order), but concept could inform multi-modal token representations

Adaptive Masking Supervision (Temporal + Sample Dimensions)

Core Idea: During diffusion training, dynamically construct different masking patterns across both timesteps (temporal) and training examples (sample), forcing the model to learn multiple valid generation paths instead of a single fixed path.

Problem It Solves:

  • Standard masked language model training uses fixed masking ratios/patterns, leading to one canonical generation order
  • Users don’t all attend to item attributes in the same sequence; fixed masking assumes homogeneous attention
  • Insufficient training diversity limits the model’s ability to handle diverse inference-time generation strategies

How It Works:

  1. Temporal Masking Variation: For each diffusion timestep t ∈ [0, T], randomly select different subsets of SID dimensions to mask:

    • Timestep t₁: mask dimensions {1, 3, 5} → denoise dimensions {2, 4, 6}
    • Timestep t₂: mask dimensions {2, 4} → denoise dimensions {1, 3, 5, 6}
    • Timestep t₃: mask all dimensions randomly
    • This ensures the model learns to denoise from various partial observation states

  2. Sample Masking Variation: Across different training examples in a batch, apply different masking schedules:

    • Item A: gradually unmask left-to-right
    • Item B: gradually unmask right-to-left
    • Item C: randomly unmask dimensions each step
    • Training loss: L = Σ ||denoise_prediction(masked_sids) - target_sids||²

  3. Trajectory Diversity: Each training sample follows a unique corruption-to-clean trajectory, preventing overfitting to a single generation path

Where It’s Useful:

  • Masked language model training for flexible generation
  • Diffusion model training where multiple valid decoding orders exist
  • Recommendation systems with heterogeneous user preferences over attribute importance
  • Any conditional generation task with multiple valid output orderings

Limitations / Caveats:

  • Significantly increases training complexity and computational cost (must compute denoising losses for many different masking patterns)
  • Requires careful hyperparameter tuning: masking ratios, temporal schedules, sampling strategies
  • No guarantee that all learned paths are equally high-quality; some may converge to suboptimal solutions
  • Difficult to analyze which generation orders the model actually learns (interpretability challenge)

Internal Use Cases:

  • RecSys: Could train multi-path recommendation models where different user types follow different attribute exploration sequences
  • LLM training: Could improve generation diversity for tasks where multiple valid orderings exist (e.g., generating sets of related concepts)

Extending the Generative Recommendation Paradigm

This MDGR paper directly extends the generative recommendation framework described in the matched vault notes, particularly addressing limitations of the autoregressive approaches highlighted in “Actions Speak Louder than Words.” While that work established treating user actions as a language and applying Transformer architectures with sequential ID generation, MDGR fundamentally challenges the assumption that autoregressive decoding is optimal for this domain. The paper identifies three critical weaknesses in the language model inheritance pattern: inability to jointly capture global dependencies across multi-dimensional semantic ID positions, the implicit assumption of uniform user attention ordering, and inference latency issues. These observations align with the “scaling limitations” and real-time requirements mentioned in the vault notes, but MDGR proposes a radically different solution—masked diffusion rather than simply scaling autoregressive models.

Bridging Autoregressive and Parallel Generation

The parallel codebook and masked diffusion training strategy in MDGR represents a conceptual bridge between the sequential transducer approach (from the Shaped.ai work) and the broader LLM-enhanced recommendation trends documented in “Generative Large Recommendation Model.” Rather than treating item generation purely as next-token prediction, MDGR adopts a denoising diffusion paradigm that constructs masking supervision adaptively across temporal and sample dimensions. This connects to the “feature interaction” and “user behavior modeling” improvements mentioned in the survey work—MDGR’s diffusion process can be seen as a sophisticated form of feature interaction that doesn’t assume a fixed temporal ordering. The two-stage warm-up decoding strategy also addresses the practical deployment concerns that the vault notes emphasize, achieving the real-time personalization goals while maintaining the generative modeling benefits.

Novel Contributions to the Knowledge Base

MDGR adds three distinctive perspectives absent from the existing notes: (1) the application of diffusion models specifically to recommendation’s semantic ID generation problem, moving beyond both traditional DLRMs and pure autoregressive generative approaches; (2) adaptive masking supervision that respects that different users may attend to item attributes in different orders—a nuanced view of personalization not explicitly captured in the sequential transducer framework; and (3) concrete evidence (10.78% improvement over SOTA) that parallel decoding can outperform autoregressive methods in this domain. This challenges the implicit assumption in the vault notes that “treating actions as language” necessarily means adopting language modeling’s left-to-right generation paradigm, opening a new research direction where diffusion-based generation may be more natural for multi-attribute item representation.
