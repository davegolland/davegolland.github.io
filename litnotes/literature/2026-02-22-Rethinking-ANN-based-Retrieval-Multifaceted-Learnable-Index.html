Rethinking ANN-based Retrieval: Multifaceted Learnable Index for Large-scale Recommendation System

Rethinking ANN-based Retrieval: Multifaceted Learnable Index for Large-scale Recommendation System

Source: arXiv:cs.IR Published: 2026-02-18 Rank: #12 (score: 0.91, max_sim: 0.87) Reading time: ~0 min Tags: Uncategorized

Abstract

Approximate nearest neighbor (ANN) search is widely used in the retrieval stage of large-scale recommendation systems. In this stage, candidate items are indexed using their learned embedding vectors, and ANN search is executed for each user (or item) query to retrieve a set of relevant items. However, ANN-based retrieval has two key limitations. First, item embeddings and their indices are typically learned in separate stages: indexing is often performed offline after embeddings are trained, which can yield suboptimal retrieval quality-especially for newly created items. Second, although ANN offers sublinear query time, it must still be run for every request, incurring substantial computation cost at industry scale. In this paper, we propose MultiFaceted Learnable Index (MFLI), a scalable, real-time retrieval paradigm that learns multifaceted item embeddings and indices within a unified framework and eliminates ANN search at serving time. Specifically, we construct a multifaceted hierarchical codebook via residual quantization of item embeddings and co-train the codebook with the embeddings. We further introduce an efficient multifaceted indexing structure and mechanisms that support real-time updates. At serving time, the learned hierarchical indices are used directly to identify relevant items, avoiding ANN search altogether. Extensive experiments on real-world data with billions of users show that MFLI improves recall on engagement tasks by up to 11.8%, cold-content delivery by up to 57.29%, and semantic relevance by 13.5% compared with prior state-of-the-art methods. We also deploy MFLI in the system and report online experimental results demonstrating improved engagement, less popularity bias, and higher serving efficiency.

MultiFaceted Learnable Index (MFLI)

Core Idea: Co-train multifaceted item embeddings with a hierarchical residual quantization codebook to produce discrete indices that directly identify relevant items at serving time, eliminating ANN search entirely.

Problem It Solves: ANN-based retrieval suffers from a training-indexing disconnect (embeddings and indices optimized separately), poor cold-start performance for new items, and unavoidable per-request ANN computation cost at billion-user scale.

How It Works:

  1. Codebook construction: Apply residual quantization (RQ) to item embeddings to build a multifaceted hierarchical codebook — each item is assigned a sequence of discrete codes (one per RQ level), capturing different semantic facets.
  2. Co-training: The codebook and item embeddings are jointly optimized end-to-end with the recommendation objective, so discrete indices reflect task-relevant structure rather than post-hoc clustering.
  3. Index structure: A multifaceted indexing structure maps each code sequence to the items it covers, with mechanisms for real-time updates (new/cold items can be inserted by quantizing their embedding on-the-fly).
  4. Serving: Given a user query, the model predicts a distribution over codebook entries at each RQ level; the hierarchical index is traversed directly to retrieve candidate items — no ANN search executed.

Architecture:

flowchart LR
    A[Item Embeddings] --> B[Residual Quantization\nHierarchical Codebook]
    B --> C[Discrete Code Sequences\nper Item]
    C --> D[Multifaceted Index\nStructure]
    Q[User Query] --> E[Code Predictor\nper RQ Level]
    E --> D
    D --> F[Retrieved Candidates]
    B <-->|Co-training| A

Where It’s Useful:

  • Large-scale recommendation retrieval stages (e.g., short-video, e-commerce) operating at billions of users and millions of QPS
  • Cold-start content delivery where newly created items must surface immediately without reindexing
  • Latency-constrained serving where eliminating ANN lookup materially reduces compute cost

Limitations / Caveats:

  • Residual quantization introduces information loss; codebook capacity must be carefully tuned to avoid embedding collapse
  • Co-training adds complexity to the training pipeline and may require careful learning rate scheduling between the continuous embedding and discrete codebook components
  • Real-time update quality depends on how well the frozen codebook generalizes to new item embeddings not seen during training

Internal Use Cases:

  • Hybrid retrieval system: Replace or complement the ANN leg of the two-tower retrieval with a learned index leg — especially valuable for reducing re-ranking pressure by improving candidate quality upstream
  • Cold-start mitigation: The RQ quantization + real-time insert path directly addresses the cold-content delivery gap noted in the vault’s hybrid retrieval architecture note
  • Popularity bias reduction: Discrete codebook structure can enforce more uniform coverage across code buckets, counteracting the embedding-space crowding that biases ANN toward popular items
