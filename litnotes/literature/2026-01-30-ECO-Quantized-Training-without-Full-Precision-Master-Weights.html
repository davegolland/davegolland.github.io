Abstract

Quantization has significantly improved the compute and memory efficiency of Large Language Model (LLM) training. However, existing approaches still rely on accumulating their updates in high-precision: concretely, gradient updates must be applied to a high-precision weight buffer, known as . This buffer introduces substantial memory overhead, particularly for Sparse Mixture of Experts (SMoE) models, where model parameters and optimizer states dominate memory usage. To address this, we introduce the Error-Compensating Optimizer (ECO), which eliminates master weights by applying updates directly to quantized parameters. ECO quantizes weights after each step and carefully injects the resulting quantization error into the optimizer momentum, forming an error-feedback loop with no additional memory. We prove that, under standard assumptions and a decaying learning rate, ECO converges to a constant-radius neighborhood of the optimum, while naive master-weight removal can incur an error that is inversely proportional to the learning rate. We show empirical results for pretraining small Transformers (30-800M), a Gemma-3 1B model, and a 2.1B parameter Sparse MoE model with FP8 quantization, and fine-tuning DeepSeek-MoE-16B in INT4 precision. Throughout, ECO matches baselines with master weights up to near-lossless accuracy, significantly shifting the static memory vs validation loss Pareto frontier.

Quantization Error Feedback Mechanism

Core Idea: Capture the rounding error introduced by quantizing parameter updates and systematically feed this error back into the optimizer’s momentum buffer to correct for optimization drift.

Problem It Solves: Quantizing weights introduces systematic rounding errors that accumulate over training steps, causing the optimizer to descend toward a perturbed optimum rather than the true loss surface minimum. Ignoring this error breaks convergence properties; simply reducing learning rate is inefficient.

How It Works:

  1. Ideal vs. Quantized Update: Compute what the parameter should be after update () versus what it actually becomes after quantization ().

  3. Momentum Injection: Add error directly to next step’s effective gradient in momentum update: .
  4. Error Propagation: The feedback loop ensures uncompensated error from round influences the update direction in round , preventing error accumulation.

Where It’s Useful:

  • Any quantized optimizer where per-step weight changes are accumulated (AdamW, SGD with momentum)
  • Post-training quantization scenarios where weights drift from training distribution
  • Low-bit quantization (FP8, INT4, INT8) where rounding errors are larger relative to update magnitudes
  • Distributed training where gradient accumulation across steps must remain stable

Limitations / Caveats:

  • Error computation itself can introduce numerical instability if quantization scheme is poorly designed
  • Requires storing error feedback buffer (negligible cost compared to master weights, but non-zero)
  • Effectiveness depends on quantization granularity—per-channel quantization may require per-channel error tracking
  • Error feedback must be scaled appropriately; wrong scaling factor can amplify numerical noise

Internal Use Cases:

  • Compensating for FP8 quantization errors in gradient accumulation steps for large batch training
  • Maintaining optimizer stability across distributed training steps where quantized parameter updates are synchronized
  • Improving convergence of sparse model training where weight changes are localized to specific experts
