FlashSchNet: Fast and Accurate Coarse-Grained Neural Network Molecular Dynamics

FlashSchNet: Fast and Accurate Coarse-Grained Neural Network Molecular Dynamics

Source: arXiv:cs.LG Published: 2026-02-13 Rank: #12 (score: 0.26, max_sim: 0.76) Reading time: ~0 min Tags: Uncategorized

Abstract

Graph neural network (GNN) potentials such as SchNet improve the accuracy and transferability of molecular dynamics (MD) simulation by learning many-body interactions, but remain slower than classical force fields due to fragmented kernels and memory-bound pipelines that underutilize GPUs. We show that a missing principle is making GNN-MD IO-aware, carefully accounting for reads and writes between GPU high-bandwidth memory (HBM) and on-chip SRAM. We present FlashSchNet, an efficient and accurate IO-aware SchNet-style GNN-MD framework built on four techniques: (1) flash radial basis, which fuses pairwise distance computation, Gaussian basis expansion, and cosine envelope into a single tiled pass, computing each distance once and reusing it across all basis functions; (2) flash message passing, which fuses cutoff, neighbor gather, filter multiplication, and reduction to avoid materializing edge tensors in HBM; (3) flash aggregation, which reformulates scatter-add via CSR segment reduce, reducing atomic writes by a factor of feature dimension and enabling contention-free accumulation in both forward and backward passes; (4) channel-wise 16-bit quantization that exploits the low per-channel dynamic range in SchNet MLP weights to further improve throughput with negligible accuracy loss. On a single NVIDIA RTX PRO 6000, FlashSchNet achieves 1000 ns/day aggregate simulation throughput over 64 parallel replicas on coarse-grained (CG) protein containing 269 beads (6.5x faster than CGSchNet baseline with 80% reduction of peak memory), surpassing classical force fields (e.g. MARTINI) while retaining SchNet-level accuracy and transferability.

FlashSchNet: IO-Aware GNN Molecular Dynamics

Core Idea: Accelerate graph neural network molecular dynamics by redesigning SchNet operations to minimize GPU memory transfers through kernel fusion and CSR-based aggregation.

Problem It Solves: GNN-based molecular dynamics potentials (like SchNet) are orders of magnitude slower than classical force fields because they fragment computation into many small kernels that repeatedly read/write intermediate tensors to GPU high-bandwidth memory (HBM), creating a memory-bound bottleneck that underutilizes GPU compute.

How It Works:

  1. Flash Radial Basis: Fuse pairwise distance calculation, Gaussian basis expansion, and cosine cutoff envelope into a single tiled kernel that computes each distance once and reuses it across all basis functions in SRAM
  2. Flash Message Passing: Fuse neighbor gathering, cutoff filtering, edge feature multiplication, and aggregation into one kernel to avoid materializing large edge tensors (N×K×F) in HBM
  3. Flash Aggregation: Reformulate scatter-add as CSR segment reduction, reducing atomic writes by a factor of feature dimension F and enabling contention-free parallel accumulation in both forward/backward passes
  4. Channel-wise Quantization: Apply 16-bit quantization to SchNet MLP weights exploiting low per-channel dynamic range to further boost throughput with <1% accuracy loss

Architecture:

flowchart LR
    A[Atom Positions] --> B[Flash Radial Basis<br/>fused: distance + basis + cutoff]
    B --> C[Flash Message Passing<br/>fused: gather + filter + reduce]
    C --> D[Flash Aggregation<br/>CSR segment reduce]
    D --> E[Quantized MLP<br/>16-bit channel-wise]
    E --> F[Forces/Energy]

Where It’s Useful:

  • Coarse-grained protein molecular dynamics (269-bead systems achieving 1000 ns/day on single GPU)
  • Any GNN-based physical simulation requiring high throughput (materials, drug discovery)
  • Applications needing classical force field speed with ML accuracy/transferability
  • Multi-replica MD workflows (parallel tempering, ensemble sampling)

Limitations / Caveats:

  • Optimization focused on SchNet architecture; other GNN variants (e.g., DimeNet++, GemNet) may need different fusion strategies
  • CSR aggregation requires neighbor list pre-sorting by destination atom
  • Quantization benefits depend on weight distribution; some GNN layers may have higher dynamic range
  • Achieves parity with classical force fields only on coarse-grained systems (269 beads); all-atom scaling not demonstrated

Internal Use Cases:

  • GPU kernel fusion patterns: Same principle applies to fusing embedding lookup + MLP in recommendation models to avoid materializing user/item features
  • Scatter-gather optimization: CSR-based aggregation technique directly applicable to graph-based ranking (e.g., knowledge graph embeddings, GraphSAGE for user-item graphs)
  • Quantization strategy: Per-channel 16-bit quantization can accelerate LLM inference in ranking rerankers while maintaining accuracy
  • Memory-compute tradeoff: Flash attention pattern (compute on-the-fly vs materialize) applies to large-batch scoring in retrieval pipelines
