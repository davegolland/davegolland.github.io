Abstract

Autoregressive policies offer a compelling foundation for scalable robot learning by enabling discrete abstraction, token-level reasoning, and flexible inference. However, applying autoregressive modeling to continuous robot actions requires an effective action tokenization scheme. Existing approaches either rely on analytical discretization methods that produce prohibitively long token sequences, or learned latent tokenizers that lack structure, limiting their compatibility with next-token prediction. In this work, we identify three desiderata for action tokenization - high compression, total decodability, and a left-to-right causally ordered token space - and introduce Ordered Action Tokenization (OAT), a learned action tokenizer that satisfies all three. OAT discretizes action chunks into an ordered sequence of tokens using transformer with registers, finite scalar quantization, and ordering-inducing training mechanisms. The resulting token space aligns naturally with autoregressive generation and enables prefix-based detokenization, yielding an anytime trade-off between inference cost and action fidelity. Across more than 20 tasks spanning four simulation benchmarks and real-world settings, autoregressive policies equipped with OAT consistently outperform prior tokenization schemes and diffusion-based baselines, while offering significantly greater flexibility at inference time.

Ordered Action Tokenization (OAT)

Core Idea: A learned action tokenizer that discretizes continuous robot action chunks into an ordered, causally-structured token sequence using transformer registers, finite scalar quantization, and ordering-inducing training, enabling efficient autoregressive policy learning with anytime decoding.

Problem It Solves: Existing action tokenization methods either produce prohibitively long sequences (analytical discretization) or lack causal structure (learned latent tokenizers like VQ-VAE), making them incompatible with efficient autoregressive next-token prediction for robot control policies.

How It Works:

  1. Input: Continuous action chunks (sequences of robot actions across multiple timesteps)
  2. Encoding: Transformer encoder with register tokens processes the action chunk and produces a fixed number of latent representations
  3. Quantization: Finite scalar quantization (FSQ) discretizes each latent into a discrete token, creating an ordered sequence
  4. Ordering Training: Apply causal masking and ordering-inducing losses during training to ensure tokens have left-to-right dependency structure
  5. Prefix Decoding: At inference, decode any prefix of the token sequence to get progressively refined action predictions (anytime property)
  6. Output: Discrete token sequence that can be generated autoregressively, then decoded back to continuous actions

Architecture:

flowchart LR
    A[Action Chunk] --> B[Transformer Encoder + Registers]
    B --> C[Latent Representations]
    C --> D[Finite Scalar Quantization]
    D --> E[Ordered Token Sequence]
    E --> F[Autoregressive Policy]
    F --> G[Generated Tokens]
    G --> H[Prefix Decoder]
    H --> I[Continuous Actions]

Where It’s Useful:

  • Robot learning and manipulation tasks requiring continuous action control
  • Autoregressive policy architectures (transformer-based behavioral cloning)
  • Settings where inference-time flexibility is needed (trading accuracy for speed)
  • Multi-task robot learning across diverse manipulation benchmarks

Limitations / Caveats:

  • Requires training an additional tokenizer model before policy learning
  • Finite scalar quantization introduces discretization error in action space
  • Ordering constraint may not be optimal for all action modalities or task structures
  • Effectiveness depends on quality of causal ordering learned during training

Internal Use Cases:

  • Could adapt for sequential recommendation: tokenize user action sequences with causal ordering for autoregressive user modeling
  • Ranking systems: compress feature vectors or ranking decisions into ordered tokens for transformer-based rankers
  • LLM reasoning: apply ordered tokenization to continuous embeddings in multimodal models where actions/predictions need discrete abstraction
