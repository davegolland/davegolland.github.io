Hierarchical Adaptive Eviction for KV Cache Management in Multimodal Language Models

Hierarchical Adaptive Eviction for KV Cache Management in Multimodal Language Models

Source: arXiv:cs.AI Published: 2026-02-02 Rank: #7 (score: 0.98, max_sim: 0.83) Reading time: ~0 min Tags: Uncategorized Code: https://arxiv.org/pdf/2602.02197v1

Abstract

The integration of visual information into Large Language Models (LLMs) has enabled Multimodal LLMs (MLLMs), but the quadratic memory and computational costs of Transformer architectures remain a bottleneck. Existing KV cache eviction strategies fail to address the heterogeneous attention distributions between visual and text tokens, leading to suboptimal efficiency or degraded performance. In this paper, we propose Hierarchical Adaptive Eviction (HAE), a KV cache eviction framework that optimizes text-visual token interaction in MLLMs by implementing Dual-Attention Pruning during pre-filling (leveraging visual token sparsity and attention variance) and a Dynamic Decoding Eviction Strategy (inspired by OS Recycle Bins) during decoding. HAE minimizes KV cache usage across layers, reduces computational overhead via index broadcasting, and theoretically ensures superior information integrity and lower error bounds compared to greedy strategies, enhancing efficiency in both comprehension and generation tasks. Empirically, HAE reduces KV-Cache memory by 41% with minimal accuracy loss (0.3% drop) in image understanding tasks and accelerates story generation inference by 1.5x while maintaining output quality on Phi3.5-Vision-Instruct model.

Dynamic Decoding Eviction Strategy (DDES)

Core Idea: Implement a hierarchical, OS-inspired recycle bin mechanism for KV cache eviction during generation, where tokens are first demoted to a lower-priority tier before hard removal, enabling re-access of previously important tokens if they regain relevance.

Problem It Solves: Standard greedy KV cache eviction (removing oldest or least-attended tokens immediately) is irreversible—once a token is evicted, any future attention to it causes recomputation waste. This is especially problematic for visual tokens that may have non-monotonic importance as the model generates diverse continuation sequences.

How It Works:

  1. Maintain a multi-tiered cache structure during decoding: Tier 1 (hot/recent tokens), Tier 2 (warm/demoted tokens), Tier 3 (cold/candidate for eviction)
  2. When generating new tokens, compute attention weights to Tier 1 tokens and estimate future attention likelihood
  3. When cache capacity is exceeded, move low-attention tokens to Tier 2 (demote) instead of hard-deleting, storing only compressed KV representations
  4. If a token in Tier 2 receives attention during subsequent decoding steps, promote it back to Tier 1 (reclaim from recycle bin)
  5. Only permanently evict tokens from Tier 3 when memory pressure requires absolute space; prioritize evicting tokens with lowest aggregate attention across previous steps

Architecture:

    B --> C{"Cache Capacity<br/>Exceeded?"}
    C -->|No| D["Add to Tier 1<br/>Hot Cache"]
    C -->|Yes| E["Identify Low-Attention<br/>Tokens"]

    H --> I
    B -.->|Tier 2 receives attention| J["Promote Back to Tier 1<br/>Reclaim from Bin"]
    J --> I

Where It’s Useful:

  • Open-ended text generation where narrative direction is unpredictable and tokens regain relevance unexpectedly
  • Multi-turn conversations where earlier context tokens may become relevant again
  • Story generation and creative writing tasks where long-range dependencies re-emerge non-monotonically

Limitations / Caveats:

  • Tier 2 (demoted) tokens still consume memory, so this trades hard eviction cost for space overhead of multiple tiers
  • Requires predicting future attention likelihood, which is non-trivial and may require separate learned predictors
  • Promotion/demotion operations add bookkeeping overhead; needs efficient indexing to avoid O(n) promotion costs

Internal Use Cases:

  • Analogous to cache replacement policies in CPU memory systems (LRU, LFU variants); could inspire token-level caching strategies in retrieval systems
  • Applicable to ranking/recommendation pipelines where candidate items are cached and might regain relevance based on user session evolution

Relationship to Heavy Hitter Oracle (H2O)

The Hierarchical Adaptive Eviction (HAE) framework significantly extends the conceptual foundation established by Heavy Hitter Oracle (H2O), but addresses a critical gap that H2O leaves unresolved: heterogeneous attention patterns across modalities. While H2O identifies and retains “heavy hitter” tokens based on cumulative attention magnitude—a globally uniform strategy—HAE recognizes that visual and text tokens exhibit fundamentally different attention dynamics in multimodal contexts. The paper’s Dual-Attention Pruning mechanism during pre-filling leverages this insight by explicitly modeling visual token sparsity and attention variance separately, rather than applying a one-size-fits-all eviction policy. This represents both an evolution of H2O’s pruning logic and a practical acknowledgment of its limitations: H2O’s attention-based ranking, while effective for homogeneous text sequences, risks over-pruning visually-grounded information or under-pruning redundant textual tokens. HAE’s theoretical framework also strengthens the foundation by providing error bounds that guarantee “superior information integrity” compared to greedy strategies—a quantitative refinement absent in H2O’s original formulation.

Bridge to Sparse Retrieval-Augmented Approaches

HAE implicitly validates and operationalizes ideas from Sparse Attention Retrieval-Augmented KV Cache Integration by treating the KV cache as a dynamically managed, tiered memory system rather than a monolithic buffer. The Dynamic Decoding Eviction Strategy (inspired by OS Recycle Bins) introduces a retrieval-friendly architecture where recently evicted tokens remain accessible in a secondary tier, mirroring the hybrid dense-sparse memory pattern described in the retrieval-augmented notes. However, HAE’s innovation is practical rather than architectural: instead of externalizing sparsely-accessed values to separate vector indices, it manages eviction policies within the existing cache hierarchy using index broadcasting to reduce computational overhead. This positions HAE as a complementary optimization—one that can coexist with or precede full retrieval-augmented pipelines, especially in resource-constrained settings where external index lookups add latency.

Novel Contribution: Multimodal-Aware Temporal Dynamics

Beyond extending H2O and retrieval-augmented concepts, HAE introduces a genuinely new perspective: the temporal and structural coupling between visual comprehension and text generation in MLLMs. The framework’s distinction between pre-filling (where visual sparsity dominates) and decoding (where dynamic token utility shifts) reflects an understanding that KV cache management in multimodal settings cannot be static or purely attention-based. The OS Recycle Bin metaphor is particularly telling—it suggests that utility in MLLMs is not monotonic across time, and that tokens valuable during comprehension may become less relevant during generation, but warrant recovery rather than permanent deletion. This temporal-adaptive view fills a gap in the prior literature, which treats H2O’s heavy-hitter ranking and sparse retrieval indexing as largely stateless operations. By coupling cache eviction to generation phase dynamics and achieving 41% memory reduction with only 0.3% accuracy loss, HAE demonstrates that multimodal-aware eviction strategies can outperform text-optimized baselines not just in theory, but across diverse downstream tasks (comprehension and story generation).
