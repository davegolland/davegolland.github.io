Context Compression via Explicit Information Transmission

Context Compression via Explicit Information Transmission

Source: arXiv:cs.CL Published: 2026-02-03 Rank: #11 (score: 0.90, max_sim: 0.88) Reading time: ~0 min Tags: Uncategorized Code: https://arxiv.org/pdf/2602.03784v1

Abstract

Long-context inference with Large Language Models (LLMs) is costly due to quadratic attention and growing key-value caches, motivating context compression. In this work, we study soft context compression, where a long context is condensed into a small set of continuous representations. Existing methods typically re-purpose the LLM itself as a trainable compressor, relying on layer-by-layer self-attention to iteratively aggregate information. We argue that this paradigm suffers from two structural limitations: (i) progressive representation overwriting across layers (ii) uncoordinated allocation of compression capacity across tokens. We propose ComprExIT (Context Compression via Explicit Information Transmission), a lightweight framework that formulates soft compression into a new paradigm: explicit information transmission over frozen LLM hidden states. This decouples compression from the model’s internal self-attention dynamics. ComprExIT performs (i) depth-wise transmission to selectively transmit multi-layer information into token anchors, mitigating progressive overwriting, and (ii) width-wise transmission to aggregate anchors into a small number of slots via a globally optimized transmission plan, ensuring coordinated allocation of information. Across six question-answering benchmarks, ComprExIT consistently outperforms state-of-the-art context compression methods while introducing only ~1% additional parameters, demonstrating that explicit and coordinated information transmission enables more effective and robust long-context compression.

Width-Wise Transmission with Globally Optimized Allocation

Core Idea: Aggregate token anchors into a small set of slots via a globally optimized transmission plan that coordinates compression capacity allocation across all tokens, rather than independently treating each token.

Problem It Solves: Naive soft compression allocates compression capacity independently per token, leading to imbalanced information distribution where some tokens consume excessive capacity while others are underutilized. This uncoordinated allocation wastes the limited compression budget.

How It Works:

  1. Initialize a set of learnable compression slots (e.g., 4-8 slots) that will hold the final compressed context
  2. Compute transmission costs or affinities between each token anchor and each slot using learned parameters
  3. Formulate a bipartite matching or assignment problem: assign each token anchor to slots to maximize total information transmission while respecting capacity constraints
  4. Solve the assignment problem globally (e.g., via optimal transport, Hungarian algorithm, or learned routing) to produce a transmission plan
  5. Execute the plan: aggregate anchors into slots according to the optimized assignments, combining information from multiple anchors per slot if beneficial
  6. Return the small set of populated slots as the compressed context representation

Architecture:

    Aff --> Opt["Global Optimization<br/>Assignment Problem"]
    Opt --> Plan["Transmission Plan<br/>Anchor → Slot Mapping"]
    Plan --> Agg["Aggregate Anchors<br/>into Slots"]
    Agg --> Out["Final Compressed<br/>Representation"]

Where It’s Useful:

  • Compressing retrieval contexts where different documents need unequal compression (some redundant, some critical)
  • Budget-constrained inference where the number of final tokens is strictly limited
  • Multi-document question-answering where slot capacity must be optimally distributed across sources

Limitations / Caveats:

  • Computing global optimal assignment can be expensive (though approximations exist); must balance optimality with computational cost
  • Requires differentiable solvers or relaxations to integrate into end-to-end training pipelines
  • Fixed number of slots may not adapt well to varying input lengths or complexity levels without additional mechanisms

Internal Use Cases:

  • RecSys ranking where compressed context summaries must fit within a token budget, using globally coordinated allocation to prioritize high-value item features
  • Long-context RAG systems where compressed documents are allocated slots based on relevance, not uniformly across all retrieved passages
