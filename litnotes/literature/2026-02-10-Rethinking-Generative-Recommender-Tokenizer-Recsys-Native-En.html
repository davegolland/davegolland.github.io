Abstract

Semantic ID (SID)-based recommendation is a promising paradigm for scaling sequential recommender systems, but existing methods largely follow a semantic-centric pipeline: item embeddings are learned from foundation models and discretized using generic quantization schemes. This design is misaligned with generative recommendation objectives: semantic embeddings are weakly coupled with collaborative prediction, and generic quantization is inefficient at reducing sequential uncertainty for autoregressive modeling. To address these, we propose ReSID, a recommendation-native, principled SID framework that rethinks representation learning and quantization from the perspective of information preservation and sequential predictability, without relying on LLMs. ReSID consists of two components: (i) Field-Aware Masked Auto-Encoding (FAMAE), which learns predictive-sufficient item representations from structured features, and (ii) Globally Aligned Orthogonal Quantization (GAOQ), which produces compact and predictable SID sequences by jointly reducing semantic ambiguity and prefix-conditional uncertainty. Theoretical analysis and extensive experiments across ten datasets show the effectiveness of ReSID. ReSID consistently outperforms strong sequential and SID-based generative baselines by an average of over 10%, while reducing tokenization cost by up to 122x. Code is available at https://github.com/FuCongResearchSquad/ReSID.

ReSID (Recommendation-Native Semantic ID)

Core Idea: A generative recommendation framework that learns collaborative-predictive item representations and quantizes them into semantic IDs optimized for sequential predictability, without relying on pre-trained language models.

Problem It Solves: Existing semantic ID methods use foundation model embeddings weakly coupled with collaborative signals and generic quantization schemes (like RQ-VAE) that don’t minimize sequential uncertainty for autoregressive generation, leading to suboptimal recommendation quality and inefficient tokenization.

How It Works:

  1. Field-Aware Masked Auto-Encoding (FAMAE): Learns item representations by masking and reconstructing structured item features (title, category, brand, etc.), preserving information predictive for collaborative filtering rather than semantic similarity alone
  2. Globally Aligned Orthogonal Quantization (GAOQ): Quantizes embeddings into discrete token sequences by jointly optimizing:
    • Semantic ambiguity reduction (items close in embedding space get distinct codes)
    • Prefix-conditional uncertainty reduction (given previous tokens in sequence, next token is more predictable)
    • Orthogonality constraints on codebook vectors for compact representation
  3. Autoregressive Generation: Uses the resulting SID sequences to train transformer-based sequential models that predict next item via token-by-token generation

Architecture:

flowchart LR
    A[Item Features] --> B[FAMAE Encoder]
    B --> C[Item Embeddings]
    C --> D[GAOQ Quantizer]
    D --> E[Semantic ID Sequence]
    E --> F[Autoregressive Transformer]
    F --> G[Next Item Prediction]

Where It’s Useful:

  • Large-scale sequential recommendation systems (e-commerce, video, music streaming)
  • Cold-start scenarios where generative models can extrapolate from limited interaction history
  • Multi-domain recommendation requiring unified item representations
  • Extreme vocabulary settings where codebook efficiency matters

Limitations / Caveats:

  • Requires structured item metadata for FAMAE; purely collaborative filtering signals may not suffice
  • GAOQ joint optimization adds training complexity compared to standard vector quantization
  • Autoregressive generation has higher inference latency than retrieval-based methods
  • May require careful tuning of semantic vs. predictive loss weights for domain transfer

Internal Use Cases:

  • Ranking systems: Use GAOQ-style quantization to create compact item embeddings that preserve ranking signal while reducing embedding table size
  • Search-to-recommendation bridge: FAMAE could unify semantic (search) and collaborative (recsys) representations in hybrid retrieval
  • LLM-free generative recsys: Demonstrates competitive performance without foundation model dependency, reducing compute/licensing costs
  • Embedding stability: Orthogonal quantization constraints may improve embedding drift issues in dynamic catalogs
