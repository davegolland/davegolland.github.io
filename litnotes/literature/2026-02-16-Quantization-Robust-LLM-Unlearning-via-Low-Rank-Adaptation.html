Abstract

Large Language Model (LLM) unlearning aims to remove targeted knowledge from a trained model, but practical deployments often require post-training quantization (PTQ) for efficient inference. However, aggressive low-bit PTQ can mask or erase unlearning updates, causing quantized models to revert to pre-unlearning behavior. We show that standard full-parameter fine-tuning often induce parameter changes that are too small to survive 4-bit quantization. We propose quantization-robust unlearning via low-rank adaptation (LoRA): we freeze the base model and concentrate unlearning into trainable adapters so that the effective update is preserved after quantization. On Llama-2-7B evaluated with MUSE dataset (BOOKS and NEWS), LoRA improves 4-bit utility by up to 7.93 points (NPO+GDR on BOOKS: 50.17 to 58.10) and yields higher 4-bit utility on NEWS for GA+GDR (40.06 to 44.82, increase of 4.76). LoRA also substantially reduces privacy leakage under 4-bit PTQ, e.g., for GA+KLR on BOOKS, PrivLeak moves from -25.68 to -5.86 (closer to ideal 0), while maintaining strong forgetting (VerMem and KnowMem near 0). Thus, using LoRA for Machine Unlearning is beneficial for scenarios where quantization is necessary for model deployment.

Quantization-Robust Unlearning via Low-Rank Adaptation (LoRA-based Unlearning)

Core Idea: Freeze the base LLM and concentrate machine unlearning updates into trainable LoRA adapters, ensuring that unlearning edits remain effective after aggressive post-training quantization (PTQ) to 4-bit precision.

Problem It Solves: Standard full-parameter fine-tuning for LLM unlearning produces parameter changes too small to survive 4-bit quantization, causing quantized models to revert to pre-unlearning behavior and leak previously “forgotten” knowledge. This undermines privacy guarantees and model utility when deploying compressed models.

How It Works:

  1. Freeze base model weights: Lock the original LLM parameters to prevent diffuse, small-magnitude updates across millions of parameters
  2. Train LoRA adapters for unlearning: Apply unlearning algorithms (e.g., NPO, GA, GDR, KLR) exclusively to low-rank adapter matrices (ΔW = BA where B, A are low-rank)
  3. Quantize the frozen base + adapters: Apply 4-bit PTQ to the base model; the concentrated LoRA updates maintain sufficient magnitude to survive quantization
  4. Deploy merged or separate adapters: Either merge quantized adapters into the base or serve them separately, preserving unlearning behavior post-compression

Where It’s Useful:

  • Privacy-compliant LLM deployment requiring GDPR “right to be forgotten” compliance on edge devices or mobile hardware
  • Resource-constrained inference scenarios where both unlearning (e.g., removing copyrighted content, PII) and quantization are mandatory
  • Multi-tenant LLM serving where different unlearning profiles must be maintained efficiently across quantized model variants

Limitations / Caveats:

  • LoRA rank and alpha hyperparameters require tuning; suboptimal settings may still lose unlearning signal under extreme quantization
  • Evaluation relies on specific benchmarks (MUSE BOOKS/NEWS); generalization to other unlearning tasks or quantization schemes (e.g., 2-bit, GPTQ) not fully characterized
  • Adapter overhead adds minor latency/memory cost compared to fully merged models; trade-off depends on adapter rank

Internal Use Cases:

  • RecSys: Remove user interaction history from personalized LLM rankers while maintaining 4-bit deployment for mobile recommendations
  • Content moderation: Unlearn toxic or outdated training data from ranking LLMs without re-quantizing entire model stacks
  • Federated learning: Enable clients to request selective unlearning in quantized on-device models without degrading utility metrics
