Toward Culturally Aligned LLMs through Ontology-Guided Multi-Agent Reasoning

Source: arXiv:cs.IR Published: 2026-01-29 Rank: #9 (score: 0.98, max_sim: 0.83) Reading time: ~15 min Tags: LLM

Abstract

Large Language Models (LLMs) increasingly support culturally sensitive decision making, yet often exhibit misalignment due to skewed pretraining data and the absence of structured value representations. Existing methods can steer outputs, but often lack demographic grounding and treat values as independent, unstructured signals, reducing consistency and interpretability. We propose OG-MAR, an Ontology-Guided Multi-Agent Reasoning framework. OG-MAR summarizes respondent-specific values from the World Values Survey (WVS) and constructs a global cultural ontology by eliciting relations over a fixed taxonomy via competency questions. At inference time, it retrieves ontology-consistent relations and demographically similar profiles to instantiate multiple value-persona agents, whose outputs are synthesized by a judgment agent that enforces ontology consistency and demographic proximity. Experiments on regional social-survey benchmarks across four LLM backbones show that OG-MAR improves cultural alignment and robustness over competitive baselines, while producing more transparent reasoning traces.

Key Techniques

  • Ontology-Guided Multi-Agent Reasoning
  • World Values Survey value summarization
  • competency question-based relation elicitation
  • multi-agent synthesis with judgment

Dual-Constraint Retrieval for Demographic-Ontology Alignment

Core Idea: At inference time, retrieve culturally-aligned persona profiles and ontology constraints by filtering for consistency with the constructed cultural ontology while ranking by demographic proximity to target profiles.

Problem It Solves: Standard retrieval-augmented generation doesn’t enforce hard logical constraints from cultural ontologies, and demographic-aware systems often ignore value consistency. This method bridges both: ensuring retrieved profiles respect cultural relations while remaining demographically grounded.

How It Works:

  1. Given target demographic d and input query q, query the cultural ontology G
  2. Ontology Consistency Filter: Identify all persona candidates whose cultural value profiles satisfy constraints in G (remove candidates violating relational edges)
  3. Demographic Ranking: Among consistent candidates, compute similarity sim(d, d_candidate) using demographic features (age, region, education, etc.)
  4. Retrieval: Return top-K consistent profiles sorted by demographic proximity, along with their instantiated ontology constraint edges
  5. Pass retrieved profiles and constraints to agent instantiation stage

Architecture:

flowchart LR
    A["Target Demographic d<br/>+ Query q"] --> B["Query Cultural<br/>Ontology G"]
    B --> C["Ontology Consistency<br/>Filter"]
    C --> D["Demographic Similarity<br/>Ranking"]
    D --> E["Return Top-K<br/>Profiles + Constraints"]
    E --> F["To Agent<br/>Instantiation"]

Where It’s Useful:

  • Retrieval-augmented generation for culturally-aware systems
  • Persona selection in multi-agent social simulation
  • Bias detection by ensuring demographic fairness during retrieval
  • Cross-cultural information retrieval systems

Limitations / Caveats:

  • Ontology consistency filtering is hard constraint—invalid candidates are dropped entirely (brittle if ontology is incomplete)
  • Demographic similarity metric choice significantly impacts results (handcrafted features vs. learned embeddings)
  • Assumes retrieved profiles adequately represent target demographic
  • Can fail if no profiles satisfy both ontology and demographic constraints
  • Computational cost scales with candidate pool size and ontology density

Internal Use Cases:

  • Filtering recommendation candidates by both cultural values and user demographics
  • Ensuring diversity signals align with explicit cultural constraints
  • Building interpretable per-demographic rankers in ranking systems

Ontology-Constrained Multi-Agent Synthesis (OCMAS)

Core Idea: Instantiate multiple value-persona agents using retrieved demographic profiles and ontology constraints, then synthesize outputs through a judgment agent that enforces ontology consistency and demographic proximity across all responses.

Problem It Solves: Multi-agent reasoning can produce inconsistent outputs when agents lack shared cultural grounding. Existing multi-agent frameworks don’t enforce logical constraints from structured ontologies, risking outputs that contradict each other or violate cultural value relations.

How It Works:

  1. Agent Instantiation: For each of top-K retrieved profiles, create agent prompt P_i containing:
    • Demographic profile attributes (age, region, values, etc.)
    • Instantiated ontology constraints C_i (specific relations this persona must respect)
    • Input query q
  2. Parallel Generation: Independently generate responses R_1, R_2, …, R_K from each persona agent
  3. Judgment Agent Validation: Judgment agent receives all responses R_i and checks:
    • Do all responses satisfy their respective ontology constraints C_i?
    • Are responses demographically coherent (no contradictions within persona)?
    • Do responses respect demographic proximity to retrieved profiles?
  4. Consistency Enforcement: If violations detected, either:
    • Accept and flag inconsistencies with confidence scores, or
    • Refine responses using strengthened constraint prompting to judgment agent
  5. Output Synthesis: Return synthesized response respecting all enforced constraints

Architecture:

flowchart LR
    A["Retrieved Profiles +<br/>Ontology Constraints"] --> B["Instantiate<br/>Agent Prompts"]
    B --> C["Parallel Generation<br/>from Persona Agents"]
    C --> D["Judgment Agent<br/>Validation"]
    D --> E["Consistency<br/>Check"]
    E -->|Pass| F["Output Synthesis"]
    E -->|Fail| G["Refine with<br/>Constraints"]
    G --> F

Where It’s Useful:

  • Multi-perspective summarization with cultural alignment
  • Cross-cultural debate or deliberation systems
  • Culturally-aware content generation requiring consensus
  • Interpretable multi-stakeholder decision support

Limitations / Caveats:

  • Judgment agent validation is itself an LLM call—can suffer from the same misalignment issues it’s trying to prevent
  • Parallel agent generation costs scale linearly with number of personas
  • Constraint refinement may loop indefinitely if no valid synthesis exists
  • Synthesized output quality depends heavily on quality of agent instantiation prompts
  • Difficult to interpret which constraint(s) caused rejection of a response

Internal Use Cases:

  • Multi-candidate ranking where candidates must represent different but non-contradictory cultural perspectives
  • Bias auditing in generative ranking systems (check if different personas generate contradictory ranking orders)
  • Building explainable multi-stakeholder recommendation systems
