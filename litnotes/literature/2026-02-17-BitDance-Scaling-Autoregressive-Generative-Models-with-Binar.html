BitDance: Scaling Autoregressive Generative Models with Binary Tokens

BitDance: Scaling Autoregressive Generative Models with Binary Tokens

Source: HuggingFace Published: 2026-02-15 Rank: #2 (score: 0.47, max_sim: 0.78) Reading time: ~0 min Tags: Uncategorized

Abstract

We present BitDance, a scalable autoregressive (AR) image generator that predicts binary visual tokens instead of codebook indices. With high-entropy binary latents, BitDance lets each token represent up to 2^{256} states, yielding a compact yet highly expressive discrete representation. Sampling from such a huge token space is difficult with standard classification. To resolve this, BitDance uses a binary diffusion head: instead of predicting an index with softmax, it employs continuous-space diffusion to generate the binary tokens. Furthermore, we propose next-patch diffusion, a new decoding method that predicts multiple tokens in parallel with high accuracy, greatly speeding up inference. On ImageNet 256x256, BitDance achieves an FID of 1.24, the best among AR models. With next-patch diffusion, BitDance beats state-of-the-art parallel AR models that use 1.4B parameters, while using 5.4x fewer parameters (260M) and achieving 8.7x speedup. For text-to-image generation, BitDance trains on large-scale multimodal tokens and generates high-resolution, photorealistic images efficiently, showing strong performance and favorable scaling. When generating 1024x1024 images, BitDance achieves a speedup of over 30x compared to prior AR models. We release code and models to facilitate further research on AR foundation models. Code and models are available at: https://github.com/shallowdream204/BitDance.

Next-Patch Diffusion (NPD)

Core Idea: A parallel autoregressive decoding method that uses continuous-space diffusion to jointly generate multiple binary visual tokens per step, breaking the sequential token-by-token bottleneck of standard autoregressive generation.

Problem It Solves: Standard AR models decode one token at a time, making inference slow—especially at high resolutions (e.g., 1024×1024). Parallel AR methods exist but typically require large models or sacrifice quality. NPD achieves fast parallel decoding with a small model by operating in binary token space rather than categorical codebook space.

How It Works:

  1. Binary tokenization: Images are encoded into high-entropy binary latent vectors (e.g., 256-bit), where each patch is a binary vector rather than a single codebook index—yielding up to 2^256 representable states per token.
  2. Transformer backbone (next-token prediction): A standard causal transformer processes the sequence of binary patch tokens autoregressively, producing a conditioning context for each upcoming patch or group of patches.
  3. Binary diffusion head: Instead of a softmax classifier over a codebook, a lightweight diffusion model runs in the continuous relaxation of binary space, conditioned on the transformer’s hidden state, and iteratively denoises a noise vector into the target binary token(s).
  4. Patch-level parallelism: At each AR step, the diffusion head jointly denoises an entire patch group (multiple tokens) in parallel, conditioned on all previously generated patches, rather than decoding one token at a time.
  5. Sampling: The denoised continuous vector is binarized (e.g., via sign function / straight-through estimator boundary at inference) to produce discrete binary tokens, which are appended to the sequence.

Architecture:

flowchart LR
    A[Image Patches] --> B[Binary Tokenizer\ne.g. 256-bit/patch]
    B --> C[Causal Transformer\nAR backbone]
    C --> D[Hidden State\ncontext vector]
    D --> E[Binary Diffusion Head\ncontinuous denoising]
    E --> F[Binarize\nsign / STE]
    F --> G[Next N Binary Tokens]
    G -->|append| C

Where It’s Useful:

  • High-resolution image generation (1024×1024) where sequential decoding is a latency bottleneck
  • Scenarios requiring small model footprint with competitive quality (260M params beating 1.4B baselines)
  • Any discrete-token AR generation task where the vocabulary is too large for softmax (genomics, molecular design, audio codecs with high-bitrate codes)

Limitations / Caveats:

  • Binary diffusion head adds inference complexity vs. a simple linear classifier; the diffusion steps add latency per patch group, partially offsetting parallelism gains
  • Binarization at inference requires careful calibration—threshold/STE behavior must be consistent with training to avoid distribution shift
  • The 2^256 state space expressiveness is theoretical; actual coverage depends on the tokenizer training and may cluster in practice
  • Requires a custom binary tokenizer training pipeline, which adds engineering overhead compared to off-the-shelf VQ-VAE codebooks

Internal Use Cases:

  • RecSys user embeddings: Binary token representations (cf. DV365’s 4-bit quantized embeddings) could be generated autoregressively from user history sequences using a diffusion head instead of a retrieval head—enabling richer per-user generative modeling
  • Ranking feature generation: Parallel diffusion decoding over item feature spaces could generate diverse candidate representations for exploration in retrieval stages
  • Large vocabulary softmax replacement: In LLM output heads over huge item catalogs (millions of SKUs), replacing softmax with a binary diffusion head sidesteps the O(V) bottleneck—predict a binary item code and decode to the item
