MATA: A Trainable Hierarchical Automaton System for Multi-Agent Visual Reasoning

Source: arXiv:cs.AI Published: 2026-01-27 Rank: #8 (score: 0.97, max_sim: 0.84) Reading time: ~20 min Tags: Search, LLM, RecSys Code: https://github.com/ControlNet/MATA

Abstract

Recent vision-language models have strong perceptual ability but their implicit reasoning is hard to explain and easily generates hallucinations on complex queries. Compositional methods improve interpretability, but most rely on a single agent or hand-crafted pipeline and cannot decide when to collaborate across complementary agents or compete among overlapping ones. We introduce MATA (Multi-Agent hierarchical Trainable Automaton), a multi-agent system presented as a hierarchical finite-state automaton for visual reasoning whose top-level transitions are chosen by a trainable hyper agent. Each agent corresponds to a state in the hyper automaton, and runs a small rule-based sub-automaton for reliable micro-control. All agents read and write a shared memory, yielding transparent execution history. To supervise the hyper agent’s transition policy, we build transition-trajectory trees and transform to memory-to-next-state pairs, forming the MATA-SFT-90K dataset for supervised finetuning (SFT). The finetuned LLM as the transition policy understands the query and the capacity of agents, and it can efficiently choose the optimal agent to solve the task. Across multiple visual reasoning benchmarks, MATA achieves the state-of-the-art results compared with monolithic and compositional baselines. The code and dataset are available at https://github.com/ControlNet/MATA.

Key Techniques

  • Multi-agent hierarchical finite-state automaton
  • trainable hyper agent for transition policy
  • shared memory with transparent execution history
  • transition-trajectory trees with supervised finetuning

Analysis: MATA in Context of Prior Hierarchical and Agentic Reasoning Work

MATA fundamentally advances the Hierarchical Reasoning Model framework by introducing trainable governance over agent composition. While HRM proposes hierarchical, multi-timescale processing inspired by brain architecture to reduce computational brittleness and data requirements, MATA operationalizes this hierarchy through a concrete automaton structure with a key innovation: a learnable hyper-agent that dynamically routes between specialized sub-agents rather than following static decomposition. Both systems reject the limitations of monolithic chain-of-thought approaches, but MATA adds a critical capability—the ability to learn when to delegate to which agent through supervised finetuning on transition trajectories. This transforms hierarchical reasoning from a fixed architectural pattern into an adaptive system that can balance collaboration across complementary agents and competition among overlapping ones, directly addressing the task decomposition brittleness that HRM identifies as a core failure mode.

The paper’s approach to multi-agent orchestration deeply resonates with Agentic Reasoning Design Patterns, particularly in leveraging reflection and tool use as foundational principles. However, MATA extends these patterns by embedding them within a formal automaton structure with shared memory, creating explicit, interpretable execution traces. Where traditional agentic systems rely on implicit LLM reasoning for agent selection, MATA’s trainable transition policy makes this selection process supervised and verifiable—each agent’s sub-automaton ensures reliable micro-control through deterministic rules rather than emergent behavior. The shared memory architecture also echoes the Mind-Map Agent concept’s persistence mechanism, but rather than dynamically building knowledge graphs during reasoning, MATA pre-defines agent roles with memory writes/reads that create transparent execution history. This suggests a deliberate trade-off: sacrificing dynamic graph construction flexibility for guaranteed interpretability and formal verification of agent behavior.

What MATA uniquely contributes is the synthesis of formal systems (finite-state automata) with learned policies, and the creation of a labeled dataset (MATA-SFT-90K) specifically for training the transition policy. While HRM addresses latency and brittleness at the architectural level, and Agentic Patterns suggest reflection loops and planning strategies, MATA demonstrates that structured supervision of multi-agent transitions—derived from transition-trajectory trees—can outperform compositional baselines on visual reasoning benchmarks. This positions MATA not merely as a refinement of existing frameworks but as a proof that interpretable multi-agent systems can achieve SOTA results when the system architecture enforces transparency through automata structure and the learning objective directly supervises agent selection.
