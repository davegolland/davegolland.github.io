Variable-Length Semantic IDs for Recommender Systems

Variable-Length Semantic IDs for Recommender Systems

Source: arXiv:cs.IR Published: 2026-02-18 Rank: #9 (score: 0.00, max_sim: 0.88) Reading time: ~0 min Tags: Uncategorized

Abstract

Generative models are increasingly used in recommender systems, both for modeling user behavior as event sequences and for integrating large language models into recommendation pipelines. A key challenge in this setting is the extremely large cardinality of item spaces, which makes training generative models difficult and introduces a vocabulary gap between natural language and item identifiers. Semantic identifiers (semantic IDs), which represent items as sequences of low-cardinality tokens, have recently emerged as an effective solution to this problem. However, existing approaches generate semantic identifiers of fixed length, assigning the same description length to all items. This is inefficient, misaligned with natural language, and ignores the highly skewed frequency structure of real-world catalogs, where popular items and rare long-tail items exhibit fundamentally different information requirements. In parallel, the emergent communication literature studies how agents develop discrete communication protocols, often producing variable-length messages in which frequent concepts receive shorter descriptions. Despite the conceptual similarity, these ideas have not been systematically adopted in recommender systems. In this work, we bridge recommender systems and emergent communication by introducing variable-length semantic identifiers for recommendation. We propose a discrete variational autoencoder with Gumbel-Softmax reparameterization that learns item representations of adaptive length under a principled probabilistic framework, avoiding the instability of REINFORCE-based training and the fixed-length constraints of prior semantic ID methods.

Variable-Length Semantic IDs (VL-SID)

Core Idea: A discrete variational autoencoder (dVAE) with Gumbel-Softmax reparameterization that learns item representations as variable-length token sequences, assigning shorter codes to frequent items and longer codes to rare ones, guided by information-theoretic principles.

Problem It Solves: Fixed-length semantic IDs allocate equal description length to all items regardless of frequency, wasting capacity on popular items and under-representing rare long-tail items — misaligned with both natural language structure and the skewed frequency distributions of real-world catalogs.

How It Works:

  1. Encode: An item’s features (text, metadata, interaction signals) are passed through an encoder to produce a continuous latent representation.
  2. Discretize with Gumbel-Softmax: The latent is quantized into a sequence of discrete tokens from a shared low-cardinality vocabulary using Gumbel-Softmax reparameterization, enabling stable gradient flow through the discrete bottleneck without REINFORCE variance.
  3. Adaptive length via KL pressure: A probabilistic length prior (derived from the variational ELBO) penalizes unnecessary token usage — frequent items, having lower information content, collapse to shorter sequences; rare items expand to use more tokens to capture their specificity.
  4. Decode: The discrete token sequence is decoded back to reconstruct item representations, with the learned IDs then used as item identifiers in a downstream generative recommender.

Architecture:

flowchart LR
    A[Item Features] --> B[Encoder]
    B --> C[Gumbel-Softmax\nQuantizer]
    C --> D{Length\nController\nKL Prior}
    D -->|Short seq| E[Frequent Item ID\ne.g. tok1·tok2]
    D -->|Long seq| F[Rare Item ID\ne.g. tok1·tok2·tok3·tok4]
    E --> G[Decoder / Recon Loss]
    F --> G
    G --> H[Semantic ID\nfor GenRec]

Where It’s Useful:

  • Generative recommender systems using autoregressive LLMs to predict item IDs as token sequences
  • Large catalogs with heavy-tailed popularity distributions (e-commerce, streaming, search)
  • LLM-integrated recommendation pipelines where vocabulary efficiency and alignment with natural language token structure matters
  • Emergent communication systems requiring efficient discrete coding of concepts with skewed frequency

Limitations / Caveats:

  • Variable-length IDs complicate beam search and constrained decoding at inference time, requiring dynamic trie/prefix-tree management rather than fixed-depth traversal
  • Gumbel-Softmax temperature annealing requires careful scheduling; too fast collapses diversity, too slow delays discretization
  • Length adaptation is learned end-to-end and may not perfectly correlate with item frequency if other signals dominate the encoder
  • Evaluation metrics (e.g., Hit@K) may need adjustment to handle variable-length generation stops

Internal Use Cases:

  • Semantic ID constrained decoding (per vault notes): the variable-length structure maps naturally onto prefix-tree constrained decoding in SGLang — a dynamic trie that terminates branches at different depths per item replaces a fixed-depth grammar
  • Vocabulary gap reduction: shorter IDs for popular items mean the generative model sees those tokens more often during training, improving calibration and reducing hallucination for head items
  • Ranking integration: length itself becomes a signal — if a model generates a long ID, it is implicitly predicting a tail/niche item, which can inform diversity-exploitation trade-offs in ranking
  • IDGenRec compatibility: VL-SID could replace fixed-length codebooks in IDGenRec-style pipelines, inheriting semantic richness while adding frequency-aware compression
