Don't be so Stief! Learning KV Cache low-rank approximation over the Stiefel manifold

Don’t be so Stief! Learning KV Cache low-rank approximation over the Stiefel manifold

Source: arXiv:cs.LG Published: 2026-01-29 Rank: #2 (score: 1.00, max_sim: 0.82) Reading time: ~0 min Tags: Uncategorized

Abstract

Key—value (KV) caching enables fast autoregressive decoding but at long contexts becomes a dominant bottleneck in High Bandwidth Memory (HBM) capacity and bandwidth. A common mitigation is to compress cached keys and values by projecting per-head matrixes to a lower rank, storing only the projections in the HBM. However, existing post-training approaches typically fit these projections using SVD-style proxy objectives, which may poorly reflect end-to-end reconstruction after softmax, value mixing, and subsequent decoder-layer transformations. For these reasons, we introduce StiefAttention, a post-training KV-cache compression method that learns \emph{orthonormal} projection bases by directly minimizing \emph{decoder-layer output reconstruction error}. StiefAttention additionally precomputes, for each layer, an error-rank profile over candidate ranks, enabling flexible layer-wise rank allocation under a user-specified error budget. Noteworthy, on Llama3-8B under the same conditions, StiefAttention outperforms EigenAttention by points on C4 perplexity and on 0-shot MMLU accuracy at iso-compression, yielding lower relative error and higher cosine similarity with respect to the original decoder-layer outputs.

Key Techniques

Error-Rank Profile Computation

Core Idea: Precompute a lookup table of reconstruction errors for each layer and rank pair, enabling post-hoc flexible rank allocation without retraining.

Problem It Solves: Standard compression methods require choosing a single rank for all layers, or retraining for each rank configuration. This is inflexible when different layers have different sensitivity to compression. An error-rank profile decouples layer-wise rank selection from end-to-end training, allowing dynamic allocation under a global error budget.

How It Works:

  1. Train projections for multiple ranks: Using StiefAttention, optimize projection bases U_r for several candidate ranks r = {r_1, r_2, …, r_max} (e.g., r ∈ {8, 16, 32, 64}) at each layer L.

  2. Measure layer-wise reconstruction error: For each (layer L, rank r) pair, compute the reconstruction error ε_{L,r} = ||output_L(original) - output_L(compressed with rank r)||.

  3. Store error profile: Build a lookup table mapping (layer_id, rank) → reconstruction error.

  4. Greedy or DP rank allocation: Given a global error budget E_total, select layer-wise ranks to maximize compression (minimize total rank sum) such that Σ_{L} ε_{L,r_L} ≤ E_total. Use dynamic programming or greedy assignment (e.g., compress high-error-per-rank layers more aggressively).

Where It’s Useful:

  • Adaptive inference systems with varying latency/memory constraints
  • Serving scenarios where error budgets change per request
  • Understanding which layers benefit most from compression
  • Layer-specific pruning strategies in broader model compression pipelines

Limitations / Caveats:

  • Error additivity assumption (Σ ε_L ≈ total error) may not hold perfectly due to error compounding through layers
  • Requires storing multiple trained projections (one per rank), increasing post-training storage
  • Greedy rank selection is suboptimal; true optimal allocation is NP-hard, but profile enables quick approximation

Internal Use Cases:

  • Ranker budgeting: In listwise ranking, allocate lower ranks to early transformer layers (lower information density) and higher ranks to final layers (decision-critical) under a fixed compression budget
  • Cascade models: Use profiles to compress early/cheap ranker layers more aggressively than final expensive rankers
  • A/B testing: Experiment with different error budgets quickly by reusing precomputed profiles without retraining
