PaperSearchQA: Learning to Search and Reason over Scientific Papers with RLVR

Source: arXiv:cs.IR Published: 2026-01-26 Rank: #4 (score: 0.99, max_sim: 0.82) Reading time: ~20 min Tags: LLM, RL, RAG, Search

Abstract

Search agents are language models (LMs) that reason and search knowledge bases (or the web) to answer questions; recent methods supervise only the final answer accuracy using reinforcement learning with verifiable rewards (RLVR). Most RLVR search agents tackle general-domain QA, which limits their relevance to technical AI systems in science, engineering, and medicine. In this work we propose training agents to search and reason over scientific papers — this tests technical question-answering, it is directly relevant to real scientists, and the capabilities will be crucial to future AI Scientist systems. Concretely, we release a search corpus of 16 million biomedical paper abstracts and construct a challenging factoid QA dataset called PaperSearchQA with 60k samples answerable from the corpus, along with benchmarks. We train search agents in this environment to outperform non-RL retrieval baselines; we also perform further quantitative analysis and observe interesting agent behaviors like planning, reasoning, and self-verification. Our corpus, datasets, and benchmarks are usable with the popular Search-R1 codebase for RLVR training and released on https://huggingface.co/collections/jmhb/papersearchqa. Finally, our data creation methods are scalable and easily extendable to other scientific domains.

Key Techniques

  • Reinforcement learning with verifiable rewards (RLVR)
  • search agents with language models
  • PaperSearchQA dataset construction
  • retrieval-based question answering over scientific corpora

RLVR Search Agent Training over Scientific Papers

Core Idea: Training language models as search agents using reinforcement learning with verifiable rewards (RLVR), where agents learn to iteratively search and reason over scientific paper collections to answer complex questions.

Problem It Solves: Search agents trained via RLVR typically optimize only final answer accuracy on general-domain questions. This work extends RLVR to technical, multi-step reasoning tasks where intermediate steps (retrieval decisions, reasoning chains) matter as much as the final answer—crucial for scientific AI systems.

How It Works:

  1. Agent Action Space: Define discrete actions: SEARCH(query) → returns ranked papers, READ(paper_id) → extracts content, REASON(evidence) → generates answer or plan.
  2. Reward Signal: Verifiable reward r(trajectory) checks if final answer matches ground truth AND (optionally) if intermediate retrieval steps cite correct papers.
  3. Policy Optimization: Use policy gradient methods (PPO or A3C) to maximize expected cumulative reward:
    • where is a trajectory of search/read/reason actions
  4. Exploration Strategy: Explore different search query formulations and reasoning paths early in training; converge to high-reward trajectories.
  5. Agent Behaviors Emergence: Through RL, agents learn emergent behaviors like query planning (reformulating questions), multi-step reasoning, and self-verification (checking answers against retrieved evidence).

Where It’s Useful:

  • Training agents for multi-source scientific literature synthesis
  • Building AI systems that can explain answers by citing retrieved papers
  • Hybrid retrieval-generation systems that need to learn when to search vs. reason from memory
  • Fact-checking and evidence grounding in technical domains

Limitations / Caveats:

  • Reward function must be verifiable (binary correctness); harder for open-ended questions (opinion, methodology comparison)
  • Agent behavior is learned from final answers only; intermediate reasoning chains not explicitly supervised unless added to reward
  • Computational cost: full RL requires thousands of rollouts per question to estimate policy gradients
  • Generalization unclear: agents trained on PaperSearchQA biomedical papers may not transfer to other domains
  • Search action space is large (millions of possible queries); exploration is inefficient without good initialization

Internal Use Cases:

  • Training LLM-based ranking models that learn to prioritize papers based on query intent (vs. BM25’s lexical matching)
  • Implementing self-improving search pipelines where user feedback on answer quality updates the reward function
  • Multi-agent collaboration: teams of search agents that divide QA tasks and verify each other’s findings

Integration with Search-R1 Framework

Core Idea: Leveraging the popular Search-R1 codebase as a standardized training infrastructure for RLVR agents, enabling reproducible experiments and community contributions.

Problem It Solves: Custom implementations of RLVR training are brittle and hard to reproduce. Using Search-R1 as a common foundation reduces implementation variance and lowers the barrier for researchers to benchmark new retrieval corpora and reasoning strategies.

Where It’s Useful:

  • Rapid prototyping of domain-specific search agents
  • Community benchmarking (comparing agents trained on different datasets)
  • Transfer learning experiments (pre-train on general QA, fine-tune on scientific papers)

Limitations / Caveats:

  • Dependent on Search-R1 API stability; framework updates may break existing experiments
  • May constrain agent architecture choices (only designs compatible with Search-R1 can be easily evaluated)
