Source: arXiv:cs.IR Published: 2026-02-06 Rank: #2 (score: 0.99, max_sim: 0.82) Reading time: ~20 min Tags: Embeddings, RAG, LLM Code: https://github.com/abananbao/R2LED

Abstract

Lifelong user modeling, which leverages users’ long-term behavior sequences for CTR prediction, has been widely applied in personalized services. Existing methods generally adopted a two-stage “retrieval-refinement” strategy to balance effectiveness and efficiency. However, they still suffer from (i) noisy retrieval due to skewed data distribution and (ii) lack of semantic understanding in refinement. While semantic enhancement, e.g., LLMs modeling or semantic embeddings, offers potential solutions to these two challenges, these approaches face impractical inference costs or insufficient representation granularity. Obsorbing multi-granularity and lightness merits of semantic identity (SID), we propose a novel paradigm that equips retrieval and refinement in Lifelong User Modeling with SEmantic IDs (R2LED) to address these issues. First, we introduce a Multi-route Mixed Retrieval for the retrieval stage. On the one hand, it captures users’ interests from various granularities by several parallel recall routes. On the other hand, a mixed retrieval mechanism is proposed to efficiently retrieve candidates from both collaborative and semantic views, reducing noise. Then, for refinement, we design a Bi-level Fusion Refinement, including a target-aware cross-attention for route-level fusion and a gate mechanism for SID-level fusion. It can bridge the gap between semantic and collaborative spaces, exerting the merits of SID. The comprehensive experimental results on two public datasets demonstrate the superiority of our method in both performance and efficiency. To facilitate the reproduction, we have released the code online https://github.com/abananbao/R2LED.

R2LED (Retrieval and Refinement in Lifelong User Modeling with Semantic IDs)

Core Idea: Use multi-granularity semantic IDs to enhance both the retrieval and refinement stages of lifelong user modeling for CTR prediction, enabling mixed collaborative-semantic retrieval and bi-level fusion.

Problem It Solves:

  • Noisy retrieval in lifelong user modeling caused by skewed data distribution in long behavior sequences
  • Lack of semantic understanding in refinement stage where collaborative filtering alone misses content meaning
  • Impractical inference costs of LLM-based semantic enhancement at scale
  • Insufficient granularity of simple semantic embeddings for capturing multi-level user interests

How It Works:

  1. Multi-route Mixed Retrieval: Generate semantic IDs (SIDs) for items using RQ-VAE to create hierarchical codes. Build parallel recall routes at different granularities (coarse to fine-grained SIDs). For each route, retrieve candidates using both collaborative signals (item ID embeddings) and semantic signals (SID-based retrieval), then merge results.
  2. Bi-level Fusion Refinement: Apply target-aware cross-attention to fuse information across different retrieval routes (route-level fusion). Use gate mechanism to blend semantic ID representations with collaborative ID embeddings (SID-level fusion), bridging semantic and collaborative spaces.
  3. CTR Prediction: Feed fused user representation and target item features into final ranking model for click-through rate prediction.

Architecture:

flowchart LR
    A[Long User History] --> B[Multi-route Retrieval]
    B --> C1[Route 1: Coarse SIDs]
    B --> C2[Route 2: Medium SIDs]
    B --> C3[Route 3: Fine SIDs]
    C1 --> D[Mixed Retrieval<br/>Collab + Semantic]
    C2 --> D
    C3 --> D
    D --> E[Bi-level Fusion]
    E --> F1[Route-level<br/>Cross-Attention]
    E --> F2[SID-level<br/>Gate Fusion]
    F1 --> G[CTR Model]
    F2 --> G
    G --> H[Prediction]

Where It’s Useful:

  • Large-scale recommendation systems with long user behavior histories (e-commerce, video platforms)
  • CTR prediction tasks requiring both collaborative filtering and semantic understanding
  • Cold-start scenarios where semantic similarity helps bridge missing collaborative signals
  • Systems needing efficient inference despite long sequence modeling

Limitations / Caveats:

  • Requires pre-training RQ-VAE to generate semantic IDs, adding infrastructure complexity
  • Multi-route retrieval increases memory footprint compared to single-route methods
  • Effectiveness depends on quality of semantic ID codebook learned from data distribution
  • Additional hyperparameters (number of routes, granularity levels, fusion weights) require tuning

Internal Use Cases:

  • Prompt compression for history-based retrieval: Replace long concatenated text histories with compact semantic ID sequences, dramatically reducing token count while preserving semantic information for two-tower models
  • Multi-granularity interest modeling: Capture user interests at different abstraction levels (topic-level to item-level) using hierarchical SIDs from RQ-VAE codebooks
  • Hybrid retrieval systems: Combine collaborative filtering (user-item interactions) with semantic matching (content similarity) by indexing both ID embeddings and SID codes
  • Cold-start mitigation: Leverage semantic IDs to retrieve relevant items for new users based on content similarity when behavioral signals are sparse
