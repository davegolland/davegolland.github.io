Unleash the Potential of Long Semantic IDs for Generative Recommendation

Unleash the Potential of Long Semantic IDs for Generative Recommendation

Source: arXiv:cs.IR Published: 2026-02-14 Rank: #1 (score: 0.00, max_sim: 0.89) Reading time: ~0 min Tags: Uncategorized

Abstract

Semantic ID-based generative recommendation represents items as sequences of discrete tokens, but it inherently faces a trade-off between representational expressiveness and computational efficiency. Residual Quantization (RQ)-based approaches restrict semantic IDs to be short to enable tractable sequential modeling, while Optimized Product Quantization (OPQ)-based methods compress long semantic IDs through naive rigid aggregation, inevitably discarding fine-grained semantic information. To resolve this dilemma, we propose ACERec, a novel framework that decouples the granularity gap between fine-grained tokenization and efficient sequential modeling. It employs an Attentive Token Merger to distill long expressive semantic tokens into compact latents and introduces a dedicated Intent Token serving as a dynamic prediction anchor. To capture cohesive user intents, we guide the learning process via a dual-granularity objective, harmonizing fine-grained token prediction with global item-level semantic alignment. Extensive experiments on six real-world benchmarks demonstrate that ACERec consistently outperforms state-of-the-art baselines, achieving an average improvement of 14.40% in NDCG@10, effectively reconciling semantic expressiveness and computational efficiency.

ACERec (Attentive Compression for Expressive Recommendation)

Core Idea: Decouple item tokenization granularity from sequential modeling granularity by compressing long Optimized Product Quantization (OPQ) semantic ID token sequences into compact latent representations via attention-based merging, then using a dedicated intent token as a dynamic prediction anchor.

Problem It Solves: Existing generative recommendation approaches face a hard trade-off: RQ-VAE-based short semantic IDs are tractable for sequential modeling but sacrifice representational expressiveness, while OPQ-based long semantic IDs are expressive but require naive rigid aggregation that discards fine-grained semantic information. ACERec resolves this by separating the tokenization and modeling stages.

How It Works:

  1. Tokenization: Items are encoded into long OPQ-based semantic ID sequences (multiple discrete tokens per item), capturing fine-grained semantic content from item embeddings.
  2. Attentive Token Merger: A cross-attention mechanism compresses the long per-item token sequence into a small set of compact latent vectors, preserving semantic signal without forcing a fixed aggregation (e.g., mean pooling).
  3. Intent Token Injection: A learnable Intent Token is prepended to the user’s interaction sequence; it aggregates cohesive user intent across the compressed item latents and serves as the dynamic anchor from which the next-item prediction is generated.
  4. Dual-Granularity Training Objective: The model is trained jointly on (a) fine-grained token-level prediction (autoregressively predicting the OPQ token sequence of the next item) and (b) global item-level semantic alignment (contrastive or similarity loss against the item’s full embedding), harmonizing local token fidelity with global semantic consistency.

Architecture:

flowchart LR
    A[Item Embedding] --> B[OPQ Tokenizer\nlong token seq]
    B --> C[Attentive Token Merger\ncross-attention]
    C --> D[Compact Latents\nper item]
    D --> E[Sequential Encoder\n+ Intent Token]
    E --> F[Intent Token State]
    F --> G1[Fine-grained\nToken Prediction Head]
    F --> G2[Global Semantic\nAlignment Loss]
    G1 --> H[Generated Semantic ID\n→ Item Retrieval]

Where It’s Useful:

  • Generative retrieval recommender systems where items must be represented as discrete token sequences for autoregressive decoding
  • Large-scale ranking pipelines needing richer item semantics than short RQ-VAE codes provide without prohibitive sequence length costs
  • Cold-start and long-tail settings where content-derived OPQ embeddings carry more signal than interaction-derived codes

Limitations / Caveats:

  • The Attentive Token Merger adds a learned compression step that must be jointly trained; suboptimal merger training could bottleneck the whole pipeline
  • OPQ codebook construction is offline and static; dynamic item catalogs may require periodic retraining of the tokenizer
  • Dual-granularity loss introduces an additional hyperparameter (loss weighting) that may require tuning per dataset

Internal Use Cases:

  • YouTube-style semantic IDs: The OPQ tokenizer + Attentive Token Merger could replace naive mean-pooling used to collapse RQ-VAE codes into a single ID, preserving the multi-subspace structure of product quantization
  • Ranking model cold-start: Intent Token paradigm is directly applicable to session-based ranking where a single query-side vector must summarize heterogeneous interaction history with varying item granularities
  • Generative retrieval beam search: Fine-grained token prediction head is compatible with constrained beam search over a semantic ID trie, enabling exact candidate retrieval without ANN index
