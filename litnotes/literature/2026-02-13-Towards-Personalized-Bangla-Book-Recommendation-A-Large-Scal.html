Towards Personalized Bangla Book Recommendation: A Large-Scale Multi-Entity Book Graph Dataset

Source: arXiv:cs.IR Published: 2026-02-12 Rank: #11 (score: 0.70, max_sim: 0.81) Reading time: ~0 min Tags: Uncategorized

Abstract

Personalized book recommendation in Bangla literature has been constrained by the lack of structured, large-scale, and publicly available datasets. This work introduces RokomariBG, a large-scale, multi-entity heterogeneous book graph dataset designed to support research on personalized recommendation in a low-resource language setting. The dataset comprises 127,302 books, 63,723 users, 16,601 authors, 1,515 categories, 2,757 publishers, and 209,602 reviews, connected through eight relation types and organized as a comprehensive knowledge graph. To demonstrate the utility of the dataset, we provide a systematic benchmarking study on the Top-N recommendation task, evaluating a diverse set of representative recommendation models, including classical collaborative filtering methods, matrix factorization models, content-based approaches, graph neural networks, a hybrid matrix factorization model with side information, and a neural two-tower retrieval architecture. The benchmarking results highlight the importance of leveraging multi-relational structure and textual side information, with neural retrieval models achieving the strongest performance (NDCG@10 = 0.204). Overall, this work establishes a foundational benchmark and a publicly available resource for Bangla book recommendation research, enabling reproducible evaluation and future studies on recommendation in low-resource cultural domains. The dataset and code are publicly available at https://github.com/backlashblitz/Bangla-Book-Recommendation-Dataset

RokomariBG: Multi-Entity Heterogeneous Book Graph for Low-Resource Recommendations

Core Idea: A large-scale heterogeneous knowledge graph dataset for Bangla book recommendations that connects books, users, authors, categories, publishers, and reviews through eight relation types to enable graph-based and hybrid recommendation models in a low-resource language setting.

Problem It Solves: Personalized book recommendation in Bangla literature has been severely limited by the lack of structured, large-scale, publicly available datasets. Most recommendation research focuses on high-resource languages (English), leaving low-resource cultural domains without proper benchmarks or training data for modern neural and graph-based models.

How It Works:

  1. Data Collection: Aggregate 127,302 books, 63,723 users, 16,601 authors, 1,515 categories, 2,757 publishers, and 209,602 reviews from the Rokomari platform (major Bangla book retailer)
  2. Graph Construction: Structure entities and their interactions into a heterogeneous knowledge graph with eight relation types (e.g., user-rates-book, book-has-author, book-belongs-to-category, book-published-by-publisher)
  3. Multi-Modal Features: Incorporate textual side information (book titles, descriptions, review text) alongside relational structure
  4. Benchmarking: Provide systematic evaluation of collaborative filtering, matrix factorization, content-based, graph neural networks (GNN), hybrid models, and neural two-tower retrieval architectures on Top-N recommendation task

Where It’s Useful:

  • Low-resource language recommendation systems (cultural/regional content platforms)
  • Graph neural network research for heterogeneous recommendation graphs
  • Hybrid recommendation architectures that combine collaborative filtering with content features
  • Cross-lingual recommendation research and domain adaptation studies
  • Knowledge graph-based recommendation in specialized domains (books, academic papers, cultural products)

Limitations / Caveats:

  • Single-domain (Bangla literature only) - cross-domain generalization unclear
  • Language-specific textual features require Bangla NLP models; many pre-trained models are English-centric
  • Best model (neural two-tower retrieval) achieves NDCG@10 = 0.204, showing significant room for improvement
  • Dataset from single source (Rokomari platform) may have platform-specific biases
  • Review sparsity and cold-start problems still challenging despite large scale

Internal Use Cases:

  • Template for building multi-entity recommendation graphs in specialized domains (e.g., research papers with authors, venues, citations, topics)
  • Benchmarking framework for evaluating how well GNNs leverage heterogeneous relations vs. simpler collaborative filtering
  • Testing cross-lingual recommendation approaches: train on English book data, adapt to Bangla using this dataset
  • Hybrid retrieval architecture evaluation: comparing two-tower neural models against graph-based and matrix factorization baselines
  • Side information integration patterns: measuring impact of textual features (titles, reviews) on graph-based recommendations
